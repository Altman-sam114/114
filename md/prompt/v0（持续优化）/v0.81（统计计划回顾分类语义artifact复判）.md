# v0.81（统计计划回顾分类语义 artifact 复判）

## 版本号

v0.81

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；v0.80 已完成分类筛选反选清除 artifact 复判，并通过最新 `origin/main` 云端 artifact 验收。v0.81 继续在同一 v0 持续优化阶段递增。

## 背景

iOS 统计页“日程计划回顾”会展示 `store.pomodoroPlan` 中的计划项，但当前行内只显示颜色点、任务标题和开始时间，没有把 `PomodoroPlanItem.category` 转成可见分类 badge，也缺少整行分类可访问标签和 Voice Control input labels。日程页计划项和统计最近记录已有分类 badge/语义模式，可复用到统计页计划回顾。

## 目标

- iOS 统计页“日程计划回顾”每条计划项显示分类 badge，优先使用 `TaskCategoryPreset.matching(item.category)` 的图标和颜色，找不到预设时回退到 `item.accentHex` 和 `tag.fill`。
- 计划回顾行补齐整行可访问语义，读出任务名、分类、计划开始时间和轮次，并提供任务名、分类名、`分类`、`分类计划` 等 Voice Control input labels。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics plan review category accessibility contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics plan review category accessibility contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- README、测试规范、核心流程文档和更新日志同步 v0.81 的云端 artifact 复判范围。

## 非目标

- 不改 `PomodoroPlanItem`、`FocusStore`、`TimerEngine`、计划生成算法或统计计算。
- 不新增 macOS 统计页计划回顾列表；macOS 当前没有对应列表，本轮只覆盖 iOS 已存在的“日程计划回顾”。
- 不调整 Pro gating、报表范围、图表算法或最近记录 UI。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；人工明确要求本轮只去云端验证。

## 当前架构依据

- 计划数据来自 `FocusStore.pomodoroPlan`，单项模型 `PomodoroPlanItem` 已包含 `taskTitle`、`category`、`accentHex`、`roundNumber` 和 `scheduledStart`。
- 分类展示模式已有先例：`ScheduleView.PomodoroPlanRow` 使用 `TaskCategoryPreset.matching(item.category)`、`categoryPreset?.accentHex ?? item.accentHex` 和 `categoryPreset?.symbolName ?? "tag.fill"`。
- 统计最近记录已有 `RecentSessionCategoryBadge` 和整行 `.accessibilityElement(children: .ignore)` / `.accessibilityInputLabels` 模式。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 在 `ChronoFocus/Views/AnalyticsView.swift` 中抽取计划回顾行视图或内联补齐分类 badge，保证窄屏下任务名和分类不互相挤压，可把分类 badge 放在第二行。
2. 新增 `AnalyticsPlanReviewCategoryBadge` 或等价私有 View，使用 `TaskCategoryPreset.matching(item.category)` 计算图标和颜色，回退 `item.accentHex` 与 `tag.fill`。
3. 为计划回顾行新增 `private func planReviewAccessibilityLabel(for item: PomodoroPlanItem) -> String`，内容包含任务、分类、开始时间和第几轮；行上使用 `.accessibilityElement(children: .ignore)`、`.accessibilityLabel(...)` 和 `.accessibilityInputLabels(...)`。
4. 在 `scripts/verify_project.sh` 的统计源码检查区加入 iOS 计划回顾分类 badge/语义契约，并输出 `Analytics plan review category accessibility contracts verified.`。
5. 在 `scripts/validate_ci_artifact.rb` 增加 `verify_project analytics plan review category accessibility contracts` 检查。
6. 更新 `scripts/verify_project.sh` 的 validator 自检 grep、小型成功 fixture、负向 fixture 名称锁定和 marker 缺失负向 fixture。
7. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
