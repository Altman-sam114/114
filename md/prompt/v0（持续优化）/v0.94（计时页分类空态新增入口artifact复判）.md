## 版本分配依据

## 背景

总目标继续优化 UI 易用性、分类工作流和 CI 云端验收。v0.91-v0.93 已补齐 iOS/Mac 日程分类空态和 Mac 日历范围空态。iOS 计时页在选中分类但无可启动待办时，仍只有“清除筛选”，用户要切到日程页才能新增同分类待办。

## 目标

- `TimerTaskCategoryEmptyView` 增加“新增此分类”动作，与现有“清除”并列。
- 点击新增后，在计时页直接打开 `TaskEditorView`，并预填当前选中分类。
- 新增按钮具备 44pt 点击区、分类可访问标签和 Voice Control input labels。
- 空态总标签更新为可新增此分类或清除筛选。
- `scripts/verify_project.sh` 新增 marker：`Timer category empty state action contracts verified.`
- `scripts/validate_ci_artifact.rb` 新增 PASS：`verify_project timer category empty state action contracts`
- 增加 marker 缺失负向 fixture。

## 非目标

- 不改 Mac 计时页。
- 不改 `FocusStore`、`TimerEngine`、共享模型。
- 不把 `TaskEditorView` 做大范围跨文件重构，除非编译需要最小可见性调整。
- 不跑本地 Xcode/build/sim。

## 当前架构依据

- 计时页已有 `selectedTaskCategory` 和 `TimerTaskCategoryEmptyView`。
- 日程页已有 `TaskEditorView(initialDueDate:initialCategory:)`，可复用。
- 验证仍以 `origin/main` GitHub Actions artifact 为准。
