# v0.86（统计分类投入排序依据语义 artifact 复判）

## 版本号

v0.86

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；远端最新 v0.85 已完成统计分类投入排行位置语义，并通过最新 `origin/main` 云端 artifact 验收。v0.86 继续在同一 v0 持续优化阶段递增。

## 背景

统计页“分类投入”已显示分类、时长、占比、专注次数和排行位置。`FocusStore.categoryBreakdown()` 当前按分类投入秒数降序返回结果，但 UI 只显示“第 N 位”，没有明确告诉用户排行依据是投入时长。用户在分类复盘时可能把排行理解为按专注次数或占比单独排序；VoiceOver 用户也需要在行级语义里直接听到排序依据。

## 目标

- iOS `AnalyticsView` 分类投入区显示排序依据，例如“按投入时长排序”。
- macOS `MacAnalyticsDetailView` 分类投入区同步显示排序依据。
- 分类投入整行可访问标签包含排序依据、分类名、排行位置、时长、专注次数和占比。
- Voice Control input labels 包含排序依据相关说法。
- 不改变 `FocusStore.categoryBreakdown()` 的排序口径或统计聚合逻辑。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics category share sort context contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics category share sort context contracts` 复判。
- README、测试规范、核心流程文档、流程图和更新日志同步 v0.86 的云端 artifact 复判范围。

## 非目标

- 不新增统计筛选器、排序切换器或交互控件。
- 不改 `CategoryFocus`、`FocusStore.categoryBreakdown()`、session 聚合或计划生成逻辑。
- 不调整 Pro gating、StoreKit、EventKit、通知、Live Activity 或计时状态机。
- 不做大规模 SwiftUI 拆分、视觉重设计或无关文档重写。

## 当前架构依据

- `FocusStore.categoryBreakdown()` 已按 `seconds` 降序返回分类投入结果。
- iOS 统计分类投入 UI 位于 `ChronoFocus/Views/AnalyticsView.swift` 的 `categoryChart`。
- macOS 统计分类投入 UI 位于 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift` 的 `MacCategoryChartPanelView`。
- v0.84 已加入专注次数上下文；v0.85 已加入排行位置上下文。本轮只补排序依据上下文。
- 项目 CI 通过 `scripts/verify_project.sh` 输出项目专属 marker，再由 `scripts/validate_ci_artifact.rb` 下载 artifact 后结构化复判。

## GitHub Actions 和 Agent C 复判要求

- workflow：`.github/workflows/ci-results.yml`。
- Agent C 只验收最新 `origin/main` commit 对应 run。

- artifact 名称必须匹配 `chronofocus-ci-v0.10-main-<short-sha>-run<run_id>-attempt<attempt>`。
- validator 必须全 PASS；失败则退回 Agent B 在 `main` 追加修复 commit，不回滚。
