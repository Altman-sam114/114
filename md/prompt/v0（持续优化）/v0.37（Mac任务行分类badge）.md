# v0.37（Mac 任务行分类 badge）

## 版本号

v0.37

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.36，因此本轮自动递增为 v0.37。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 Mac 小窗已显示当前待办分类 badge，iOS 计时页任务行也同时展示分类和时间；但 Mac 详细窗口共用的 `MacTaskRowView` 使用 `task.dueDate?.scheduleTimeText ?? task.category`，有时间的任务会用时间替代分类，导致 Mac 计时队列和日程待办列表里的分类上下文不稳定。补齐常驻分类 badge 可以减少用户在分类筛选和任务列表之间来回确认。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `MacTaskRowView` 始终显示任务分类 badge。
2. 有 `dueDate` 的任务继续显示时间，但不再替代分类。
3. badge 使用分类代表色和 `TaskCategoryPreset.matching(task.category)` 的图标。
4. `scripts/verify_project.sh` 增加静态契约，锁定 Mac 任务行保留分类 badge 且不回退到 dueDate/category 二选一。
5. README、测试规范、核心流程文档和更新日志同步本轮 UI 覆盖范围。

## 非目标

- 不改变任务模型、分类模型、筛选状态或持久化。
- 不改变 iOS UI、Mac 小窗、Mac 分类筛选逻辑或快照脚本。
- 不改变 GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `MacTaskRowView` 定义在 `ChronoFocusMac/Views/MacTimerDetailView.swift`，被 Mac 计时队列和 Mac 日程待办列表复用。
- `MacMiniTaskCategoryBadgeView` 已使用 `TaskCategoryPreset.matching(task.category)?.symbolName ?? "tag.fill"`，可作为图标 fallback 参考。
- `scripts/verify_project.sh` 已检查 Mac 小窗分类 badge 和多个 Mac 日程分类标记，适合补充任务行分类 badge 契约。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 修改 `ChronoFocusMac/Views/MacTimerDetailView.swift`，在 `MacTaskRowView` 中新增 `categoryTint` 和 `categorySymbolName`。
3. 将原来的 `Text(task.dueDate?.scheduleTimeText ?? task.category)` 改为分类 badge + 可选时间文本。
4. 修改 `scripts/verify_project.sh`，检查 `MacTaskRowView` 包含分类 badge、分类可访问 label、可选 dueDate 文本，并拒绝 `task.dueDate?.scheduleTimeText ?? task.category` 回退模式。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.37: 增加 Mac 任务行分类 badge` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
