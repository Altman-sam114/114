# v0.45（日程摘要按钮分类语义）

## 版本号

v0.45

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.44，因此本轮自动递增为 v0.45。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.44 已为计时页选中分类摘要的“清除”按钮补齐分类名可访问标签和 Voice Control input labels。日程页 iOS 与 macOS 的选中分类摘要也提供“新增此分类”和“清除”按钮，但按钮本身仍是通用文案；辅助技术聚焦到具体按钮时，不能直接知道将新增或清除哪个分类。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 为 iOS `SelectedCategorySummaryView` 的“新增此分类”按钮增加分类名可访问标签和 Voice Control input labels。
2. 为 iOS `SelectedCategorySummaryView` 的“清除”按钮增加分类名可访问标签和 Voice Control input labels。
3. 为 macOS `MacSelectedCategorySummaryView` 的“新增此分类”和“清除”按钮增加同等分类语义。
4. 保留既有新增、清除、快照渲染、筛选状态和布局行为。
5. `scripts/verify_project.sh` 增加 iOS/Mac 日程摘要按钮静态契约，锁定按钮级 label 和 input labels。
6. README、测试规范、核心流程文档和更新日志同步本轮 UI 可访问范围。

## 非目标

- 不改变分类筛选状态、清除动作、空态触发条件、任务模型、持久化、计划生成或计时状态机。
- 不修改 CI workflow、artifact 文件列表、validator 结构或快照脚本。
- 不改变 Mac 快照静态替代视图的结构。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `SelectedCategorySummaryView` 和 `MacSelectedCategorySummaryView` 都是 SwiftUI 展示层，只调用外部传入的 `onAddTask` / `onClear`。
- `TaskCategoryPreset.matching(category)` 已提供分类图标与色彩，真实分类值仍来自 `FocusTask.category`，本轮只补按钮语义。
- v0.44 的 `TimerSelectedTaskCategorySummaryView` 已有可复用模式：按钮保持现有 tap target，并追加 `.accessibilityLabel("清除\(category)分类筛选")` 与 `.accessibilityInputLabels(...)`。
- `scripts/verify_project.sh` 已有 `schedule_summary_source` 和 Mac 摘要动作接线检查，可在对应 source slice 中追加按钮级契约。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`；若权限导致 no-op pull 无法执行，必须记录 `HEAD == origin/main` 的证据。
2. 修改 `ChronoFocus/Views/ScheduleView.swift`：
   - “新增此分类”按钮追加 `.accessibilityLabel("新增\(category)分类待办")`。
   - 同一按钮追加 `.accessibilityInputLabels([Text("新增此分类"), Text("新增\(category)分类待办"), Text("新增\(category)分类")])`。
   - “清除”按钮追加 `.accessibilityLabel("清除\(category)分类筛选")`。
   - 同一按钮追加 `.accessibilityInputLabels([Text("清除筛选"), Text("清除\(category)分类")])`。
3. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的非快照按钮，补齐同样的分类化 label 和 input labels；快照静态视图不改动作逻辑。
4. 修改 `scripts/verify_project.sh`：
   - 在 `schedule_summary_source` 中锁定 iOS 新增/清除按钮、44pt 清除/新增 tap target、分类 label 和 Voice Control input labels。
   - 增加 `mac_schedule_summary_source` slice，锁定 Mac 新增/清除按钮、分类 label 和 Voice Control input labels。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.45: 增强日程摘要按钮分类语义` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
