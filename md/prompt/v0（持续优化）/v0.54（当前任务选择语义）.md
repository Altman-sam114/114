# v0.54（当前任务选择语义）

## 版本号

v0.54

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.53，因此本轮自动递增为 v0.54。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.53 已增强 CI artifact manifest/index 元数据复判，并通过最新 `origin/main` 云端 artifact 验收。

当前 iOS 计时页、macOS 详细计时页和 macOS 小窗都允许用户从待办列表选择当前任务。视觉上已通过 checkmark 或背景高亮表达选中，但按钮/任务行本身没有统一暴露 selected trait，也没有说明“已选中当前待办”或“选择此待办”。对 VoiceOver、键盘/辅助技术用户来说，当前任务选择状态不够明确。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 计时页当前日程任务行暴露选中语义：
   - 选中任务行添加 `.accessibilityAddTraits(.isSelected)`。
   - 任务行添加可访问标签，包含任务标题、分类和已选中/未选中状态。
   - 任务行添加提示，说明已选中时是当前待办，未选中时可选择为当前待办。
2. macOS 详细计时任务行暴露同等选中语义。
3. macOS 小窗任务按钮暴露同等选中语义。
4. `scripts/verify_project.sh` 增加源码契约，锁定 iOS/Mac 当前任务选择 selected trait 和 hint。
5. README、测试规范、核心流程文档和更新日志同步本轮 UI 可访问性变化。

## 非目标

- 不改变任务模型、分类清洗、筛选排序、持久化、计划生成或 `TimerEngine` 状态机。
- 不改变任务选择行为、禁用逻辑、运行中不可切换任务规则或按钮布局。
- 不修改 GitHub Actions workflow、artifact 文件列表、validator 结构或 Xcode 构建命令。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- iOS `TimerView.taskPicker` 使用 `Button { engine.selectTask(task) }` 包裹 `TaskRow(task:isSelected:)`。
- macOS 详细计时页使用 `MacTaskRowView(task:isSelected:)` 展示待办选择行。
- macOS 小窗 `MacMiniTaskPickerView` 内部按钮直接渲染任务标题、分类 badge 和时间/轮次上下文。
- 三处选择状态都来自 `engine.selectedTaskID == task.id`，不需要新增状态或绕过 `TimerEngine`。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`；若 `git pull` 审批超时但 `git fetch origin` 后 `HEAD == origin/main`，记录该事实并继续。
2. 修改 `ChronoFocus/Views/TimerView.swift`：
   - 在 `TaskRow` 中新增 `selectionStateText`、`selectionHintText` 等私有计算属性。
   - 对任务行根视图添加 `.accessibilityElement(children: .combine)`、`.accessibilityLabel(...)`、`.accessibilityHint(...)`。
   - 选中时添加 `.accessibilityAddTraits(.isSelected)`。
3. 修改 `ChronoFocusMac/Views/MacTimerDetailView.swift`：
   - 在 `MacTaskRowView` 中新增同等 selection 文案。
   - 对任务行根视图添加合并、label、hint 和选中 trait。
4. 修改 `ChronoFocusMac/Views/MacMiniTimerView.swift`：
   - 在小窗任务按钮 label 的根 `HStack` 根据 `engine.selectedTaskID == task.id` 添加 label、hint 和 selected trait。
   - 可新增小型 helper，避免重复拼接文案。
5. 修改 `scripts/verify_project.sh`：
   - 增加 iOS `TaskRow` 源码契约，检查 `.accessibilityAddTraits(.isSelected)` 和选择提示文案。
   - 增加 Mac `MacTaskRowView` 源码契约，检查同等 selected trait 和提示。
   - 增加 Mac 小窗任务按钮源码契约，检查 selected trait 和提示。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
7. 只检查 diff 和 status，不运行本地测试命令。
8. 提交 `v0.54: 增强当前任务选择语义` 并 push 到 `origin/main`。
9. Agent C 下载最新 run artifact 并按下方标准验收。
