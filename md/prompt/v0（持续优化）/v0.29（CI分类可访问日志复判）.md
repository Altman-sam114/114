# v0.29（CI分类可访问日志复判）

## 版本号

v0.29

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.28，因此本轮自动递增为 v0.29。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.28 已为 iOS 日程页、iOS 计时页和 macOS 日程详情的分类筛选 chip 增加可访问 label/hint，并在 `scripts/verify_project.sh` 中加入源码 contract。当前云端 `verify_project.log` 只显示项目验证整体通过，Agent C 只能间接确认分类可访问 contract 运行过；结果包结构化复判脚本也还没有单独检查该 contract 的日志 marker。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/verify_project.sh` 在三端分类 chip 可访问源码 contract 通过后输出稳定日志 marker。
2. `scripts/validate_ci_artifact.rb` 下载 artifact 后强制复判 `verify_project.log` 包含该 marker。
3. `verify_project.sh` 自身的 CI 标记检查要锁定 validator 已包含这条分类可访问复判。
4. README、测试规范、核心流程文档和更新日志同步 CI 结果包对分类可访问 contract marker 的复判行为。

## 非目标

- 不改变 SwiftUI UI 视觉或交互。
- 不改 `FocusStore`、`TimerEngine`、计划、统计、通知、StoreKit、EventKit 或持久化逻辑。
- 不改变 GitHub Actions artifact 文件列表和版本号。
- 不新增第三方依赖。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `.github/workflows/ci-results.yml` 会把 `bash scripts/verify_project.sh` 输出保存到 `ci-results/verify_project.log`。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载结果包后的结构化复判入口，已经检查 `verify_project.log` 中的 `Mac core tests passed.` 和 `Project structure verified.`。
- v0.28 的分类可访问源码 contract 已集中在 `verify_project.sh` 的 Ruby 切片检查中，适合在所有断言通过后输出稳定 marker。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `scripts/verify_project.sh` 的分类 chip 可访问 contract 断言通过后输出 `Category chip accessibility contracts verified.`。
3. 在 `scripts/validate_ci_artifact.rb` 增加 `verify_project category accessibility contracts` 检查，要求 `verify_project.log` 包含该 marker。
4. 在 `scripts/verify_project.sh` 的 CI result package marker 区域增加对 validator marker 复判字符串的检查。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.29: 复判分类可访问日志` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。

## 验收标准

- 最新 `origin/main` commit 的 GitHub Actions artifact 通过 Agent C 核对，manifest、run context、artifact index、run id、attempt 和 commit SHA 全部一致。
- 下载的 `verify_project.log` 明确包含分类可访问 contract marker。
- 下载的 artifact validator 复判可证明 marker 被检查，不只依赖整体项目验证成功。
