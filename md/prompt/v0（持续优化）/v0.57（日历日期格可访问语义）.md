# v0.57 日历日期格可访问语义

## 版本号

v0.57

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试。v0.56 已完成 CI artifact index 精确清单并通过云端验收；本轮回到 UI 易用性，聚焦日程日历日期格的可访问语义，因此递增到 v0.57。

## 背景

iOS 日程页 `CalendarDayButton` 目前只有简单的日期和待办数可访问 label，缺少已选中状态、选择提示、selected trait 和 Voice Control input labels。macOS 日程详情 `MacCalendarDayCell` 当前没有专门可访问 label/hint/trait/input labels。用户在周/月视图中按日期浏览待办时，辅助技术难以明确当前选中日期和可执行动作。

## 目标

- iOS `CalendarDayButton` 补齐日期、待办数、已选中状态、跨月状态、选择提示、selected trait 和 Voice Control input labels。
- macOS `MacCalendarDayCell` 补齐同等日期格可访问语义。
- `scripts/verify_project.sh` 增加 iOS/Mac 日期格源码契约。
- README、测试规范、核心流程文档、流程图和更新记录同步当前真实行为。

## 非目标

- 不改变 `FocusStore`、`TimerEngine`、任务筛选、分类排序、计划生成或持久化。
- 不改变日历布局、日期计算、周/月范围或任务计数逻辑。
- 不引入新依赖，不重构 Schedule View。
- 不实现空态新增按钮、任务行操作标签或 CI 快照时间复判。

## 当前架构依据

- 日期格只是 SwiftUI View 收集用户选择意图，点击后更新局部 `selectedDate`。
- 待办数量仍由现有 `taskCount(on:)` 计算。
- 可访问性源码契约继续由 `scripts/verify_project.sh` 在云端 CI 执行。

## 实现步骤

1. 修改 `ChronoFocus/Views/ScheduleView.swift`：
   - 在 `CalendarDayButton` 内新增 `accessibilityDateText`、`accessibilityStateText`、`accessibilityHintText`、`accessibilityTraits` 和 `voiceControlInputLabels`。
   - label 格式包含日期、待办数、已选中状态和非本月状态。
   - hint 在已选中时说明当前正在查看此日期，否则说明选择此日期查看待办。
   - 选中日期添加 `.isSelected` trait。
   - input labels 至少包含日期、选择日期和“某日”。
2. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 在 `MacCalendarDayCell` 内增加同等 helper 和修饰符。
3. 修改 `scripts/verify_project.sh`：
   - 增加 `assert_calendar_day_accessibility` 或等价 Ruby contract。
   - 分别检查 iOS `CalendarDayButton` 与 Mac `MacCalendarDayCell` 的 label、hint、selected trait、Voice Control input labels 和非本月文案。
4. 同步文档：
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`
