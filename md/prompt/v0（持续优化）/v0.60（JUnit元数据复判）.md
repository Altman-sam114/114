# v0.60（JUnit元数据复判）

## 版本号

v0.60

## 版本分配依据

人工总目标要求持续优化 CI 测试。v0.59 已完成日程任务操作语义并通过云端 artifact 验收；本轮继续按小版本递增到 v0.60，聚焦 CI 结果包中 JUnit 元数据的身份复判。

## 背景

当前 `scripts/validate_ci_artifact.rb` 已核对 JUnit 的 testcase 名称、日志和 outcome，但还没有显式锁定 `testsuite name="ChronoFocus CI Results"` 与 `testcase classname="ChronoFocusCI"`。如果下载目录中混入结构相似但不是本项目 workflow 的 JUnit，现有检查仍有可混淆空间。

## 目标

- validator 新增 JUnit suite/classname 元数据复判。
- `scripts/verify_project.sh` 增加负向 fixture，篡改 JUnit suite name 或 classname 后必须失败。
- 云端 artifact 复判输出新增 `PASS junit metadata`。
- 同步 README、测试规范、核心流程和更新记录。

## 非目标

- 不改变 GitHub Actions 生成 JUnit 的格式，当前 workflow 已输出正确 suite/classname。
- 不改动 Swift 业务代码、UI、模型、计时状态机或持久化。
- 不运行本地测试、build、lint、`git diff --check` 或脚本验证；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- `.github/workflows/ci-results.yml` 生成 `junit.xml`，suite name 为 `ChronoFocus CI Results`，testcase classname 为 `ChronoFocusCI`。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载云端 artifact 后的结构化复判脚本。
- `scripts/verify_project.sh` 已用小型 artifact fixture 覆盖 validator 正向与负向路径。
- Agent C 必须下载最新 `origin/main` artifact 并核对 manifest、run context、artifact 名称、JUnit、日志和产物。

## 实现步骤

1. 同步最新 `main`：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认没有无关 tracked diff；不要读取或提交用户未跟踪文件。
2. 修改 `scripts/validate_ci_artifact.rb`：
   - 定义或直接检查期望 suite name `ChronoFocus CI Results`。
   - 要求所有 testcase 的 `classname` 均为 `ChronoFocusCI`。
   - 新增检查名 `junit metadata`。
3. 修改 `scripts/verify_project.sh`：
   - 锁定 validator 包含 `junit metadata` marker。
   - 在小型成功 fixture 后增加 `negative_junit_metadata_fixture`，篡改 suite name 或 testcase classname，要求输出 `FAIL junit metadata`。
4. 同步文档：
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## 测试要求

- 本轮不运行本地测试、build、lint、`git diff --check` 或脚本验证，因为人工明确要求“不得在本地测试，都去云端”。
- 提交并 push 后以 `.github/workflows/ci-results.yml` 的云端结果为准。
- Agent C 下载最新 artifact 后运行：

```bash
ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> \
  --commit <origin-main-sha> \
  --run-id <run_id> \
  --attempt <run_attempt>
```

- 复判输出必须包含 `PASS junit metadata`。

## 文档更新要求

- README 和测试规范补充 JUnit suite/classname 元数据复判与负向 fixture。
- `md/flow/flow.md` 和 `md/flow/flowchart.md` 同步 CI 结果包复判范围。
- `update_log.md` 新增 v0.60 记录，注明未跑本地测试及云端验收要求。

## 验收标准

- validator 对正确云端 artifact 输出 `PASS junit metadata`。
- 篡改 JUnit suite/classname 的 fixture 在云端 `verify_project.sh` 中会失败并出现 `FAIL junit metadata`。
- 云端 CI 成功，artifact manifest 的 `branch=main`、`commitSha`、run id、attempt 与最新 `origin/main` 一致。
- Agent C 下载最新 artifact 后复判全 PASS。

## 风险和禁止项

- 禁止改 workflow 生成格式之外的无关逻辑。
- 禁止把旧 run、旧 artifact 或本地输出冒充云端验收。
- 禁止提交未跟踪用户文件。

## main 提交、push 和云端验收

- 本轮固定使用 `main`。
- 提交信息：`v0.60: 复判 JUnit 元数据`。
- push 到 `origin/main` 后查询 `ChronoFocus CI Results` 最新 run。
- Agent C 下载最新 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 必须核对 manifest、artifact index、run context、artifact 名称、JUnit、failure summary、`verify_project.log`、Mac/iOS build 日志、`.xcresult` 和 Mac 快照产物。
