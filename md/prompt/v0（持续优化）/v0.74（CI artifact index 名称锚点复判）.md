# v0.74（CI artifact index 名称锚点复判）

## 版本号

v0.74

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.73，本轮继续围绕“优化 CI 测试”和云端 artifact 可复判性做小步增强，因此自动递增为 v0.74。

## 背景

当前 CI artifact 已在 manifest 和 `ci-run-context.txt` 中记录并复判 `artifactName`，`ci-run-context.txt` 也已做精确键集检查。但 `ci-artifact-index.json` 作为 Agent C 复判文件清单和大小的核心依据，尚未记录 artifact 名称。若后续生成逻辑漂移或混包，index 只能依赖 branch、commit、run 和 attempt 身份字段，缺少与 manifest/run context 的 artifactName 交叉锚点。

## 目标

- `.github/workflows/ci-results.yml` 生成 `ci-artifact-index.json` 时写入顶层 `artifactName`。
- `scripts/validate_ci_artifact.rb` 新增 `index artifact name` 复判，要求 index、manifest、run context 和固定规则计算出的 artifact 名称一致。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐 index `artifactName`。
- `scripts/verify_project.sh` 新增篡改 index `artifactName` 的负向 fixture，要求 validator 输出 `FAIL index artifact name`。
- README、测试规范、核心流程文档、流程图和更新日志同步 artifact index artifactName 复判范围。

## 非目标

- 不修改 Swift 业务代码、UI、模型、计时状态、计划生成或持久化逻辑。
- 不提升 `CI_PROCESS_VERSION`。
- 不改变 artifact 文件清单、目录结构、上传策略或 workflow 触发条件。
- 不运行本地测试、构建、lint、`git diff --check`、`bash scripts/verify_project.sh` 或本地 fixture validator。

## 当前架构依据

- `.github/workflows/ci-results.yml` 的 `Prepare result metadata` 生成 `ARTIFACT_NAME`，`Create CI manifest and summaries` 生成 manifest、JUnit、failure summary 和 artifact index。
- `scripts/validate_ci_artifact.rb` 负责 Agent C 下载云端 artifact 后复判 manifest、run context、artifact index、JUnit、failure summary、日志和快照。
- `scripts/verify_project.sh` 包含 validator 小型成功 fixture 与负向 fixture，用于云端 `Project verification` 阶段证明 validator 不放行错包。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `.github/workflows/ci-results.yml`：
   - 在 `write_artifact_index()` 生成的 index 顶层加入 `"artifactName": os.environ["ARTIFACT_NAME"]`。

2. 更新 `scripts/validate_ci_artifact.rb`：
   - 在 manifest artifactName 复判后增加 `index artifact name` 检查。
   - 要求 `index["artifactName"] == expected_artifact_name`，并与 `manifest["artifactName"]`、`run_context["artifactName"]` 完全一致。

3. 更新 `scripts/verify_project.sh`：
   - 小型成功 fixture 的 `ci-artifact-index.json` 顶层加入 `artifactName`。
   - 自检 grep 增加 `index artifact name`、`negative_index_artifact_name_fixture` 和 `FAIL index artifact name`。
   - 新增负向 fixture：复制成功 fixture，篡改 `ci-artifact-index.json` 的 `artifactName`，运行 validator 应失败并输出 `FAIL index artifact name`。

4. 同步文档：
   - README 描述 artifact index 也记录并复判 artifactName。
   - `md/test/test.md` 更新 Agent C PASS 清单和负向 fixture 范围。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实验证流。
   - `update_log.md` 追加 v0.74 记录，并说明未运行本地测试的原因。

## 验收标准

- 云端 artifact 的 `ci-artifact-index.json` 顶层包含本轮 artifactName。
- 最新云端 artifact 的 validator 输出包含 `PASS index artifact name`。
- manifest、run context、artifact index 三处 artifactName 与固定规则计算结果完全一致。
- GitHub Actions 最新 `origin/main` run 成功。
- manifest、run context、artifact 名称、commit SHA、run id、attempt 与最新 `origin/main` 完全一致。
