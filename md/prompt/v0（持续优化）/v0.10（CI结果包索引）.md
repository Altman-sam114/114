# v0.10（CI 结果包索引）

## 版本号

v0.10

## 版本分配依据

人工总目标包含“优化 CI 测试”。v0.9 已完成 iOS 计时页分类筛选待办，本轮继续在 `v0（持续优化）` 阶段递增为 v0.10，聚焦云端结果包的可验收性和容量可见性。

## 背景

当前 CI artifact 已包含 manifest、failure summary、JUnit、Mac/iOS build 日志、Mac/iOS `.xcresult`、Mac 快照和快照 manifest。Agent C 需要逐个确认这些文件/目录是否存在且非空，目录大小也只能靠下载后手动 `du`。为减少验收成本，本轮给结果包新增结构化 artifact index，列出关键文件和目录的存在性、类型、字节数和目录递归大小。

## 目标

- `.github/workflows/ci-results.yml` 的 `CI_PROCESS_VERSION` 更新为 v0.10。
- CI 生成 `ci-results/ci-artifact-index.json`。
- artifact index 至少覆盖：
  - `ci-artifact-manifest.json`
  - `ci-artifact-index.json`
  - `ci-failure-summary.md`
  - `junit.xml`
  - `static-checks.log`
  - `verify_project.log`
  - `xcodebuild.log`
  - `ios-xcodebuild.log`
  - `xcode-version.log`
  - `ci-run-context.txt`
  - `ChronoFocusMac.xcresult`
  - `ChronoFocus-iOS.xcresult`
  - `project-reports/mac-snapshots`
  - `project-reports/mac-snapshots/manifest.json`
  - 五张 Mac 快照 PNG
- manifest 增加 `artifactIndexPath`，`projectSpecificReports` 增加 `artifact_index`。
- `scripts/verify_project.sh` 增加 workflow 标记检查。
- 文档同步更新 Agent C 验收要求。

## 非目标

- 不改 Swift 业务代码、模型、计时状态机、UI 或快照脚本。
- 不新增新的 GitHub Actions job。
- 不上传 DerivedData、build cache、历史 artifact、视频或额外大文件。
- 不把 artifact index 作为替代 manifest；manifest 仍是 run/commit/outcome 的主权威。

## 当前架构依据

- `.github/workflows/ci-results.yml` 的 `Create CI manifest and summaries` 步骤已经用 Python 生成 manifest、JUnit 和 failure summary，适合在同一位置生成 artifact index。
- `md/test/test.md` 已要求 Agent C 下载 artifact 后检查目录大小，本轮把关键文件大小内置到结果包。
- Agent C 仍只验收最新 `origin/main` 对应 run 的 artifact。

## 实现步骤

1. 将 `CI_PROCESS_VERSION` 更新为 `v0.10`。
2. 在 workflow Python 逻辑中新增：
   - `artifactIndexPath: "ci-results/ci-artifact-index.json"`。
   - `artifact_index` 项加入 `projectSpecificReports`。
   - `path_metadata(...)` helper，输出 path、required、exists、kind、byteCount 或 recursiveByteCount、fileCount。
   - `write_artifact_index()`，生成 JSON，包含 version、branch、commitSha、runId、runAttempt、createdAt、entries 和 totals。
3. 生成顺序必须保证 manifest、JUnit 和 failure summary 已写入后再生成 artifact index。
4. artifact index 必须只扫描 `ci-results` 内明确列出的必要路径，避免递归扫描 DerivedData 或无关缓存。
5. 更新 `scripts/verify_project.sh`，检查 workflow 中存在 `ci-artifact-index.json`、`artifactIndexPath`、`path_metadata`、`recursiveByteCount`。
6. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 验收标准

- workflow 生成 `ci-artifact-index.json` 并上传到 artifact。
- manifest 中 `artifactIndexPath` 为 `ci-results/ci-artifact-index.json`。
- artifact index 中每个 required entry 都有 `exists=true`，文件有正数 `byteCount`，目录有正数 `recursiveByteCount` 和 `fileCount`。
- artifact index 覆盖 manifest、summary、JUnit、主日志、Mac/iOS `.xcresult`、快照目录、快照 manifest 和五张快照。
- 本地轻量检查通过。
- push 到 `origin/main` 后，Agent C 下载最新 run artifact，核对 manifest、artifact index、JUnit、failure summary、日志、Mac/iOS `.xcresult`、Mac 快照和快照 manifest。

## 风险和禁止项

- 禁止扫描或上传 DerivedData、缓存、历史 artifact 或无关大文件。
- 禁止用 artifact index 取代 manifest 的 run/commit/outcome 核对。
- 禁止跳过云端 artifact 验收。
- 禁止读取、提交或删除无关未跟踪文件。

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 用 GitHub CLI 下载最新 `origin/main` commit 对应 run 的 artifact 到：

```text
/private/tmp/chronofocus-c-review-<run_id>/
```

- 必须核对 `ci-artifact-manifest.json`、`ci-artifact-index.json`、`ci-failure-summary.md`、`junit.xml`、`verify_project.log`、`xcodebuild.log`、`ios-xcodebuild.log`、`ChronoFocusMac.xcresult`、`ChronoFocus-iOS.xcresult`、Mac 快照和 Mac 快照 manifest。
- manifest 的 `branch`、`commitSha`、`runId`、`runAttempt` 必须与最新 `origin/main` 和下载 run 完全一致。
