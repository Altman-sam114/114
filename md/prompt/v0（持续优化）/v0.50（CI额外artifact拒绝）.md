# v0.50（CI 额外 artifact 拒绝）

## 版本号

v0.50

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.49，因此本轮自动递增为 v0.50。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.49 已补齐 iOS 日程任务行分类 badge 的 Voice Control 输入标签，并通过最新 `origin/main` 云端 artifact 验收。当前 `scripts/validate_ci_artifact.rb` 已核对 required entry 的存在性、大小和本地元数据，但对下载目录中的额外根文件或 `project-reports` / Mac 快照目录中的未声明文件缺少显式拒绝项。由于 workflow 上传整个 `ci-results` 目录，若未来混入未索引文件，现有 validator 可能只在部分目录大小变化时发现，不能形成清晰的“小数据 artifact / 不下载无关产物”契约。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 在 `scripts/validate_ci_artifact.rb` 中增加 `unexpected local artifacts` 检查。
2. 检查 artifact 根目录只包含 manifest、index、summary、JUnit、日志、Mac/iOS `.xcresult` 和 `project-reports`。
3. 检查 `project-reports` 只包含 `mac-snapshots`，Mac 快照目录只包含 `manifest.json` 和 5 张预期 PNG。
4. 保持 `.xcresult` 内部文件不做 allowlist 限制，避免误伤 Xcode 原生结果包结构。
5. 在 `scripts/verify_project.sh` 中新增额外文件负向 fixture，证明 validator 会输出 `FAIL unexpected local artifacts`。
6. README、测试规范、核心流程文档和更新日志同步本轮 CI artifact 拒绝范围。

## 非目标

- 不改变 SwiftUI UI、分类筛选状态、任务模型、持久化、计划生成或计时状态机。
- 不修改 GitHub Actions workflow 上传路径、artifact 名称、JUnit 结构或 Xcode 构建命令。
- 不删除既有 validator 检查、负向 fixture 或 Mac 快照 byteCount 复判。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `.github/workflows/ci-results.yml` 生成 `ci-results` 并上传整个目录。
- `scripts/validate_ci_artifact.rb` 已使用 `EXPECTED_INDEX_ENTRIES` 锁定 required entry、metadata 和 index totals，但没有独立检查未声明的根目录或报告目录文件。
- `scripts/verify_project.sh` 已维护小型成功 fixture 和多种负向 fixture，可在同一区域新增 `unexpected_local_artifact_fixture`。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`；若 `git pull` 审批超时但 `git fetch origin` 后 `HEAD == origin/main`，记录该事实并继续。
2. 修改 `scripts/validate_ci_artifact.rb`：
   - 新增 artifact 根目录 allowlist 常量。
   - 新增 `project-reports` allowlist 常量。
   - 新增 Mac 快照目录 allowlist 常量，内容为 `manifest.json` 和 `EXPECTED_SNAPSHOTS`。
   - 新增 helper 计算目录中未预期 child。
   - 在 index 本地元数据检查后增加 `check(checks, "unexpected local artifacts")`。
3. 修改 `scripts/verify_project.sh`：
   - CI marker grep 增加 `unexpected local artifacts`、`unexpected_local_artifact_fixture` 和 `FAIL unexpected local artifacts`。
   - 在小型成功 fixture 通过后新增负向 fixture：复制成功 fixture，写入 `unexpected-root.log`，运行 validator 要求失败，并 grep `FAIL unexpected local artifacts`。
4. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 只检查 diff 和 status，不运行本地测试命令。
6. 提交 `v0.50: 增强 CI 额外 artifact 拒绝` 并 push 到 `origin/main`。
7. Agent C 下载最新 run artifact 并按下方标准验收。

## 测试要求

本轮不运行本地测试、构建、`git diff --check`、`bash scripts/verify_project.sh`、YAML/plist lint 或 Xcode 命令，因为人工明确要求“不得在本地测试，都去云端”。

云端必须由 push 到 `origin/main` 触发 `.github/workflows/ci-results.yml`，并在最新 run artifact 中确认：

- `staticChecks`、`projectVerification`、`macBuild`、`iosBuild` 全部 success。
- `scripts/validate_ci_artifact.rb` 对下载目录输出全 PASS，且包含 `PASS unexpected local artifacts`。
- `verify_project.log` 包含 `Category summary action contracts verified.`、`Category chip accessibility contracts verified.` 和 `Project structure verified.`。
- Mac/iOS build 日志包含 `** BUILD SUCCEEDED **`。

## 文档更新要求

- README 说明 validator 会拒绝未声明的根目录、报告目录和快照目录额外文件。
- `md/test/test.md` 同步 `unexpected local artifacts` PASS 和负向 fixture。
- `md/flow/flow.md` 与 `md/flow/flowchart.md` 同步 artifact 额外文件拒绝范围。
- `update_log.md` 新增 v0.50 记录，说明未跑本地测试及原因。

## 验收标准

- 最新 `origin/main` commit 的 GitHub Actions artifact 通过 Agent C 核对，manifest、run context、artifact index、run id、attempt 和 commit SHA 全部一致。
- validator 对下载 artifact 输出 `PASS unexpected local artifacts`。
- 云端 `verify_project.sh` 已执行额外文件负向 fixture。
- 本轮不依赖旧 run、本地输出或文字汇报冒充云端验收。

## 风险和禁止项

- 禁止把 `.xcresult` 内部文件做固定 allowlist，避免破坏 Xcode 原生结果包兼容性。
- 禁止扩大为 workflow 重构或 artifact 上传策略迁移。
- 禁止触碰未跟踪用户文件 `md/1.md` 和 `md/通用云端迭代升级提示词.md`。
