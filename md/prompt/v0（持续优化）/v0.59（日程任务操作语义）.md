# v0.59（日程任务操作语义）

## 版本号

v0.59

## 版本分配依据

人工总目标要求持续优化 UI 易用性、分类体验和 CI 测试。v0.58 已完成快照 manifest `generatedAt` 复判；本轮按小版本递增到 v0.59，聚焦日程任务行操作控件的可访问语义。

## 背景

当前 iOS 日程任务行的完成、启用、编辑按钮，以及 macOS 日程任务行的完成、启用、删除控件，在 VoiceOver / Voice Control 中多读成通用动作。用户在分类筛选或长列表中操作时，不容易确认动作会作用到哪一个待办。

## 目标

- iOS 日程任务行操作按钮读出具体任务名：完成/标记未完成、启用/停用、编辑。
- macOS 日程任务行操作控件读出具体任务名：完成/标记未完成、启用/停用、删除；快照静态控件也要保持同等语义。
- 为上述操作补齐 Voice Control input labels，方便按任务名直接触发。
- 在项目验证脚本中加入源码契约，确保云端 CI 能复判本轮行为。
- 同步 README、测试规范、核心流程和更新记录。

## 非目标

- 不改变可见布局、按钮尺寸、列表排序、筛选逻辑或任务数据模型。
- 不修改 `FocusStore`、`TimerEngine`、计划生成、统计或持久化逻辑。
- 不引入第三方依赖。
- 不做完整本机构建或本地测试；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- `FocusTask.title` 是任务操作语义的真实上下文，任务仍由 `FocusStore` 统一管理。
- iOS 日程 UI 位于 `ChronoFocus/Views/ScheduleView.swift`，任务行由 `ScheduleTaskCell` 渲染。
- macOS 日程 UI 位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift`，任务列表由 `MacTaskListPanelView` 渲染。
- `scripts/verify_project.sh` 是项目专属验证入口，云端 workflow 会运行它并把日志纳入 artifact。
- Agent C 必须下载最新 `origin/main` artifact，核对 manifest、日志、JUnit、`.xcresult` 和项目专属产物。

   - `ScheduleTaskCell` 的完成按钮 label 加入 `task.title`。
   - 启用/停用按钮 label 加入 `task.title`。

## 验收标准

- iOS 日程任务行完成/启用/编辑按钮包含任务名级别 accessibility label 和 input labels。
- macOS 日程任务行完成/启用/删除控件及快照静态控件包含任务名级别 accessibility label 和 input labels 或等价语义。
- 云端 CI 成功，artifact manifest 的 `branch=main`、`commitSha`、run id、attempt 与最新 `origin/main` 一致。
- `scripts/validate_ci_artifact.rb` 对下载 artifact 全部 PASS。
- `verify_project.log` 包含 `Schedule task action accessibility contracts verified.`。

## 风险和禁止项

- 禁止改动任务数据模型、计时状态机、计划生成或分类排序。
- 禁止改变可见按钮文案造成布局变化；动态语义应通过 accessibility API 提供。
- 禁止把未跟踪文件混入提交。
- 禁止用旧 run、旧 artifact 或本地输出冒充云端验收。
