# v0.39（CI artifact index 本地元数据复算）

## 版本号

v0.39

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.38，因此本轮自动递增为 v0.39。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 validator 已核对 artifact index 身份、required paths、entries totals 以及下载目录中 required artifact 是否非空，但对单个 required entry 的 `byteCount`、目录 `fileCount` 和 `recursiveByteCount` 仍主要依赖 index 自身记录。补齐本地元数据复算后，Agent C 能发现下载目录中文件被替换、截断、追加或目录内容不一致但 index 未更新的结果包。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/validate_ci_artifact.rb` 新增 required artifact 本地元数据复算。
2. file entry 必须验证下载目录实际 `byteCount` 与 `ci-artifact-index.json` 一致。
3. directory entry 必须验证下载目录实际 `fileCount` 和 `recursiveByteCount` 与 `ci-artifact-index.json` 一致。
4. `scripts/verify_project.sh` 增加 marker 检查，锁定新增 validator 规则。
5. `scripts/verify_project.sh` 增加本地文件大小篡改负向 fixture，确认 validator 会拒绝错包。
6. README、测试规范、核心流程文档和更新日志同步本轮 CI 覆盖范围。

## 非目标

- 不改变 GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不改变 SwiftUI UI、业务逻辑、数据模型、计时状态机或平台服务。
- 不改变 artifact index 生成逻辑，只增强下载 artifact 后的复判。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `.github/workflows/ci-results.yml` 已在 `ci-artifact-index.json` 中记录 required 文件和目录的存在性、kind、大小与递归大小。
- `scripts/validate_ci_artifact.rb` 已有 `local_artifact_path` 和 `positive_local_artifact?`，适合扩展为本地元数据复算。
- `scripts/verify_project.sh` 已有小型成功 fixture 和多个负向 fixture，可在本地文件内容篡改后断言 validator 失败。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `scripts/validate_ci_artifact.rb` 中引入 `find`，新增 `local_artifact_metadata(...)`。
3. 新增 `index required local metadata` 复判。
4. 在 `scripts/verify_project.sh` 中增加 `index required local metadata`、`mismatched_local_artifact_fixture` 和失败 marker 检查。
5. 在 `scripts/verify_project.sh` 的 fixture 区新增本地文件大小篡改负向 fixture，并断言输出 `FAIL index required local metadata`。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
7. 只检查 diff 和 status，不运行本地测试命令。
8. 提交 `v0.39: 复算 artifact index 本地元数据` 并 push 到 `origin/main`。
9. Agent C 下载最新 run artifact 并按下方标准验收。

本轮不运行本地测试、构建、`git diff --check`、`bash scripts/verify_project.sh`、Ruby validator、YAML/plist lint 或 Xcode 命令，因为人工明确要求“不得在本地测试，都去云端”。

云端必须由 push 到 `origin/main` 触发 `.github/workflows/ci-results.yml`，并在最新 run artifact 中确认：

- `scripts/validate_ci_artifact.rb` 对下载目录输出全 PASS，且包含 `PASS index required local metadata`。

- `verify_project.log` 包含 `Category chip accessibility contracts verified.` 和 `Project structure verified.`。
- Mac/iOS build 日志包含 `** BUILD SUCCEEDED **`。

## 验收标准

- 最新 `origin/main` commit 的 GitHub Actions artifact 通过 Agent C 核对，manifest、run context、artifact index、run id、attempt 和 commit SHA 全部一致。
- 下载 artifact 的 validator 输出包含 `PASS index required local metadata`。
- 本轮不依赖旧 run、本地输出或文字汇报冒充云端验收。

## 风险和禁止项

- 禁止用本地输出或旧 artifact 代替最新云端验收。
- 禁止扩大到 workflow 重构、artifact 版本升级或 UI 改动。
- 禁止触碰未跟踪用户文件 `md/1.md` 和 `md/通用云端迭代升级提示词.md`。
