# v0.87（统计分类投入空态语义 artifact 复判）

## 版本号

v0.87

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化分类 UI 易用性和 CI 测试。远端最新 v0.86 已完成统计分类投入排序依据语义，并通过最新 `origin/main` 云端 artifact 验收。v0.87 继续在同一 v0 持续优化阶段递增。

## 背景

iOS 统计页“分类投入”在没有分类统计时只显示普通文本“完成番茄钟后会自动生成分类统计。”；macOS 已使用 `ContentUnavailableView`，但缺少独立 helper、空态可访问标签和 Voice Control input labels。用户首次进入统计页或没有完成分类专注时，需要更明确知道“分类统计”为什么为空、下一步要完成带分类的番茄钟，以及空态仍属于分类投入区域。云端结果包也应能独立证明该空态语义没有退化。

## 目标

- iOS `AnalyticsView` 分类投入空态改用 `ContentUnavailableView`，显示“暂无分类统计”和下一步说明。
- macOS `MacAnalyticsDetailView` 分类投入空态与 iOS 使用同名 helper 文案。
- iOS/macOS 分类投入空态都提供可访问标签，读出“暂无分类统计”和“完成带分类的番茄钟后，会按投入时长生成分类统计”。
- iOS/macOS 分类投入空态都提供 Voice Control input labels，例如“分类统计”“暂无分类统计”“分类投入空态”。
- 不改变 `FocusStore.categoryBreakdown()`、统计排序、Pro gating 或任何会话聚合逻辑。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics category share empty state contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics category share empty state contracts` 复判。
- README、测试规范、核心流程文档、流程图和更新日志同步 v0.87 的云端 artifact 复判范围。

## 非目标

- 不新增统计筛选器、排序切换器、引导按钮或购买流程。
- 不修改 `CategoryFocus`、`FocusStore.categoryBreakdown()`、`FocusSession` 或计划生成逻辑。
- 不调整统计页整体布局、卡片风格、Pro 权益、StoreKit、EventKit、通知、Live Activity 或计时状态机。
- 不做大规模 SwiftUI 拆分或无关文档重写。

## 当前架构依据

- iOS 统计分类投入 UI 位于 `ChronoFocus/Views/AnalyticsView.swift` 的 `categoryChart`。
- macOS 统计分类投入 UI 位于 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift` 的 `MacCategoryChartPanelView`。
- `FocusStore.categoryBreakdown()` 返回空数组时进入分类投入空态；本轮只改空态展示和辅助技术语义。
- 项目 CI 通过 `scripts/verify_project.sh` 输出项目专属 marker，再由 `scripts/validate_ci_artifact.rb` 下载 artifact 后结构化复判。

## 实现步骤

1. 修改 `ChronoFocus/Views/AnalyticsView.swift`：
   - 将 `store.categoryBreakdown().isEmpty` 分支中的普通 `Text` 改为 `ContentUnavailableView`。
   - 新增 `categoryShareEmptyTitle()`、`categoryShareEmptyDescriptionText()` 和 `categoryShareEmptyAccessibilityLabel()` helper。
   - 空态添加 `.accessibilityElement(children: .ignore)`、`.accessibilityLabel(categoryShareEmptyAccessibilityLabel())` 和 `.accessibilityInputLabels([Text("分类统计"), Text("暂无分类统计"), Text("分类投入空态")])`。
2. 修改 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift`：
   - 使用同名 helper 替换当前字面量空态。
   - 保留 Mac 现有 `ContentUnavailableView` 和最小高度。
   - 添加与 iOS 一致的可访问标签和 Voice Control input labels。
3. 修改 `scripts/verify_project.sh`：
   - 在分类投入排序依据契约后新增空态源码契约。
   - 检查 iOS/macOS 都包含 helper、`ContentUnavailableView(categoryShareEmptyTitle()`、空态可访问标签和 Voice Control input labels。
   - 输出 `Analytics category share empty state contracts verified.`。
   - 成功 fixture 补齐新 marker。
   - 新增 `negative_analytics_category_share_empty_state_marker_fixture`。
   - 增加自锁 grep，确保 validator 和负向 fixture 都覆盖新契约。
4. 修改 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project analytics category share empty state contracts` 检查。
5. 更新文档：
   - README 和 `md/test/test.md` 补充空态 marker、PASS 和负向 fixture。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 补充当前真实统计空态语义和 artifact 复判范围。
   - `update_log.md` 追加 v0.87 记录，说明本轮不跑本地测试、以云端 artifact 为准。
