# v0.76（CI manifest 总结果复判）

## 版本号

v0.76

## 版本分配依据

人工未指定版本号。当前最新云端通过版本为 v0.75，本轮继续围绕“优化 CI 测试”做小步增强，因此自动递增为 v0.76。

## 背景

当前 CI 结果包 manifest 已记录 static checks、项目验证、macOS build、iOS build 等分阶段 outcome，failure summary 也列出这些阶段状态。但 manifest 缺少一个顶层总结果字段，Agent C 需要从多个字段推断整个结果包是否全绿。为减少人工复判歧义，本轮给 manifest 增加 `overallOutcome`，并让 validator 与 fixture 复判它和四个阶段状态一致。

## 目标

- `.github/workflows/ci-results.yml` 在 `ci-artifact-manifest.json` 写入 `overallOutcome`。
- `overallOutcome` 规则：static checks、project verification、Mac build、iOS build 全为 `success` 时为 `success`，否则为 `failure`。
- `ci-failure-summary.md` 增加 `Overall outcome` 行，和 manifest 保持一致。
- `scripts/validate_ci_artifact.rb` 新增 `manifest overall outcome` 复判，并复判 failure summary 中的总结果行。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐 `overallOutcome`，并新增总结果篡改负向 fixture。
- README、测试规范、核心流程文档、流程图和更新日志同步 CI manifest 总结果复判范围。

## 非目标

- 不修改 Swift UI、模型、Store、TimerEngine、计划生成或持久化逻辑。
- 不修改 artifact 名称规则、run context 精确字段集合或 CI process version。
- 不新增大体积 artifact，不上传 DerivedData 或无关缓存。
- 不运行本地测试、构建、lint、`git diff --check`、`bash scripts/verify_project.sh` 或本地 fixture validator。

## 当前架构依据

- CI 结果包由 `.github/workflows/ci-results.yml` 的 `Create CI manifest and summaries` 步骤生成。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载云端 artifact 后的结构化复判脚本。
- `scripts/verify_project.sh` 通过小型成功 fixture 和负向 fixture 覆盖 validator 放行/拒绝路径。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `.github/workflows/ci-results.yml`：
   - 在 Python 生成逻辑中基于四个阶段 outcome 计算 `overall_outcome`。
   - manifest 写入 `"overallOutcome": overall_outcome`。
   - failure summary 在 run 身份信息后加入 `- Overall outcome: ...`。

2. 更新 `scripts/validate_ci_artifact.rb`：
   - 增加四个阶段 outcome 到总结果的推导。
   - 新增 `PASS/FAIL manifest overall outcome`。
   - 将 `Overall outcome` 纳入 failure summary outcomes 复判。

3. 更新 `scripts/verify_project.sh`：
   - fixture manifest 和 summary 补齐 `overallOutcome`。
   - 增加 manifest 总结果篡改负向 fixture，要求 validator 输出 `FAIL manifest overall outcome`。
   - 增加源码 grep，确保 workflow、validator 和 fixture 都覆盖该字段。

4. 同步文档：
   - README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 写明 manifest 总结果复判。
   - `update_log.md` 追加 v0.76，并将 v0.75 云端结果更新为已通过。

## 验收标准

- 最新云端 artifact 的 manifest 包含 `overallOutcome=success`。
- failure summary 包含 `Overall outcome: success`。
- validator 输出包含 `PASS manifest overall outcome`。
- validator 仍输出既有 `PASS index artifact name`、`PASS run context exact keys` 和 UI contract marker 复判。
- GitHub Actions 最新 `origin/main` run 成功，manifest/run context/artifact 名称/commit/run id/attempt 与最新 `origin/main` 完全一致。
