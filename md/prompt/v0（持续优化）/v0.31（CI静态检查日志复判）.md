# v0.31（CI静态检查日志复判）

## 版本号

v0.31

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.30，因此本轮自动递增为 v0.31。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 GitHub Actions 会生成 `static-checks.log`，其中包含 committed diff whitespace check、plist lint 和 workflow YAML parse 三段静态检查输出；`scripts/validate_ci_artifact.rb` 已核对 JUnit 的 `staticChecks` testcase 和 artifact 必需路径，但还没有复判 `static-checks.log` 的关键 marker。补齐这层检查可以让 Agent C 更直接确认静态检查产物不是空文件或错日志。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/validate_ci_artifact.rb` 新增 static-checks 日志 marker 复判。
2. 复判内容覆盖 `Running committed diff whitespace check...`、`Running project plist lint...`、`Running workflow YAML parse check...` 和 `yaml ok`。
3. `scripts/verify_project.sh` 的 validator 正向 fixture 同步补齐 static-checks 日志 marker。
4. `verify_project.sh` 自身锁定 validator 必须包含 `EXPECTED_STATIC_CHECK_MARKERS`。
5. README、测试规范、核心流程文档和更新日志同步 static-checks 日志 marker 复判范围。

## 非目标

- 不改变 GitHub Actions workflow 阶段顺序。
- 不改变 artifact 文件列表或 CI process version。
- 不改 SwiftUI UI、业务逻辑、数据模型、计时状态机或平台服务。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `.github/workflows/ci-results.yml` 的 `Static checks` step 会把输出保存到 `ci-results/static-checks.log`。
- `scripts/validate_ci_artifact.rb` 已经读取 artifact 目录中的 manifest、index、JUnit、failure summary、`verify_project.log`、Mac/iOS build 日志和快照 manifest。
- `scripts/verify_project.sh` 已经用小型成功 fixture 跑 validator 正向路径，适合同步补齐 static-checks marker。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `scripts/validate_ci_artifact.rb` 中增加 `EXPECTED_STATIC_CHECK_MARKERS` 常量。
3. 增加 `static_checks_log_path`，读取 `static-checks.log`，新增 `static checks log markers` 检查。
4. 更新 `scripts/verify_project.sh` 中的 validator marker 检查和小型成功 fixture 的 `static-checks.log` 内容。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.31: 复判静态检查日志` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。

## 验收标准

- 最新 `origin/main` commit 的 GitHub Actions artifact 通过 Agent C 核对，manifest、run context、artifact index、run id、attempt 和 commit SHA 全部一致。
- 下载的 `static-checks.log` 明确包含四个静态检查 marker。
- 下载的 artifact validator 复判可证明 static-checks marker 被检查，不只依赖 JUnit outcome。

- 禁止用本地输出或旧 artifact 代替最新云端验收。
- 禁止新增不稳定或依赖 runner 路径的 static-check marker。
- 禁止扩大到 workflow 重构或 artifact 版本升级。
- 禁止触碰未跟踪用户文件 `md/1.md` 和 `md/通用云端迭代升级提示词.md`。
