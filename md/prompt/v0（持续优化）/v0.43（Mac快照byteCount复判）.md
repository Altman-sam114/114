# v0.43（Mac 快照 byteCount 复判）

## 版本号

v0.43

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.42，因此本轮自动递增为 v0.43。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 GitHub Actions artifact 已包含 Mac 快照目录和 `project-reports/mac-snapshots/manifest.json`，`scripts/validate_ci_artifact.rb` 会核对快照名称、尺寸、manifest 中的 `byteCount` 为正，以及下载后的 PNG 文件非空。但 manifest 中每张 PNG 的 `byteCount` 尚未与下载目录中的实际 `File.size` 做一致性比较；如果 manifest 被篡改为错误大小，仍可能通过结构化复判。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/validate_ci_artifact.rb` 增加 `snapshot byte counts` 检查。
2. 对 `project-reports/mac-snapshots/manifest.json` 中每个 snapshot，要求 `byteCount` 等于 artifact 下载目录中同名 PNG 的实际 `File.size`。
3. 保留现有 `snapshot names` 和 `snapshot dimensions` 检查。
4. `scripts/verify_project.sh` 增加 validator marker，锁定 `snapshot byte counts` 检查存在。
5. `scripts/verify_project.sh` 增加篡改 snapshot manifest `byteCount` 的负向 fixture，要求 validator 输出 `FAIL snapshot byte counts`。
6. README、测试规范、核心流程文档和更新日志同步本轮 CI artifact 复判范围。

## 非目标

- 不修改 `.github/workflows/ci-results.yml` artifact 文件列表或 CI process version。
- 不改变 Mac 快照渲染脚本、快照尺寸、PNG 内容或 Xcode build 命令。
- 不改变 iOS/macOS UI、计时状态机、模型、持久化或分类业务逻辑。
- 不运行任何本地测试、构建、格式检查、验证脚本或本地 validator fixture。

## 当前架构依据

- `.github/workflows/ci-results.yml` 已上传 Mac 快照 manifest 和 5 张 PNG。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载 artifact 后的结构化复判入口，适合增强下载结果包一致性检查。
- `scripts/verify_project.sh` 已用小型 artifact fixture 和多个负向 fixture 覆盖 validator 放行/拒绝路径，可追加一个只篡改快照 manifest `byteCount` 的负向 fixture。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `scripts/validate_ci_artifact.rb` 的快照检查附近新增 `snapshot byte counts` check。
3. 该 check 遍历 `snapshot_manifest.fetch("snapshots")`，定位下载目录下 `project-reports/mac-snapshots/<fileName>`，比较 `snapshot["byteCount"].to_i == File.size(snapshot_path)`。
4. 在 `scripts/verify_project.sh` 的 validator marker 区增加 `grep -q "snapshot byte counts" scripts/validate_ci_artifact.rb`。
5. 在现有 validator 负向 fixture 后追加 `mismatched_snapshot_manifest_fixture`：复制成功 fixture，读取 `project-reports/mac-snapshots/manifest.json`，把第一张图的 `byteCount` 加 1，运行 validator 并要求失败，输出包含 `FAIL snapshot byte counts`。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
7. 只检查 diff 和 status，不运行本地测试命令。
8. 提交 `v0.43: 增强 Mac 快照大小复判` 并 push 到 `origin/main`。
9. Agent C 下载最新 run artifact 并按下方标准验收。
