# v0.90（Mac 快速新增标题分类上下文 artifact 复判）

## 版本号

v0.90

## 版本分配依据

人工未指定版本号；当前 `md/prompt/v0（持续优化）/` 和 `update_log.md` 最新正式轮次为 v0.89，因此本轮自动递增为 v0.90。

## 背景

总目标仍是持续优化 ChronoFocus 的 UI 易用性、分类工作流和 CI 云端验收。v0.89 已完成 iOS/macOS 统计页分类投入占比可读性与 artifact 复判。本轮转向 macOS 日程页快速新增：当前分类输入框和“新增待办”按钮已有分类语义，但用户从分类筛选摘要点击“新增此分类”后，焦点进入的“任务名称”输入框仍未读出当前将新增到哪个分类，容易在第一输入点丢失预填上下文。

## 目标

- 为 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的快速新增“任务名称”输入框补充当前分类上下文。
- 输入框可访问标签必须读出“当前将新增到 X 分类”。
- Voice Control input labels 必须包含通用任务名称入口、按当前分类新增待办入口和“X 分类任务名称”入口。
- `scripts/verify_project.sh` 新增独立源码契约 marker：`Mac quick add title field category context contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 Agent C artifact 复判项：`PASS verify_project mac quick add title field category context contracts`。
- 增加 marker 缺失负向 fixture，防止旧 artifact 或缺少该契约的结果包被放行。

## 非目标

- 不改 `FocusStore`、`TimerEngine`、共享模型、持久化格式或计划生成逻辑。
- 不重构 macOS 日程页布局，不拆分大文件。
- 不同时实现 iOS 日程分类空态、CI 运行身份增强或统计页其它改动。
- 不引入第三方依赖。

## 当前架构依据

- `FocusStore` 仍是任务、分类、计划和统计的唯一核心数据仓库。
- `TimerEngine` 仍是唯一计时状态机，本轮不触碰计时状态。
- macOS UI 只能依赖 macOS 平台层和共享模型，不能引入 UIKit。
- `MacScheduleDetailView` 已有 `quickAddCategoryName`、`quickAddCategoryInputAccessibilityLabel`、`quickAddInputLabels`、`prepareQuickAdd(_:)` 和 `isTaskTitleFocused`，适合在同一边界内补齐标题输入框语义。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认 `HEAD == origin/main`，且只有人工未跟踪文件可忽略。
2. 在 `MacScheduleDetailView` 中新增 `quickAddTitleAccessibilityLabel` 和 `quickAddTitleInputLabels` helper。
3. 给 `TextField("任务名称", text: $taskTitle)` 添加 `.accessibilityLabel(quickAddTitleAccessibilityLabel)`、`.accessibilityHint("输入待办标题，保存后会归入当前分类")` 和 `.accessibilityInputLabels(quickAddTitleInputLabels)`。
4. 在 `scripts/verify_project.sh` 的 Mac quick add 源码契约中检查 helper、文案、input labels 和 TextField 绑定，并输出新 marker。
5. 在 `scripts/validate_ci_artifact.rb` 中复判 `verify_project.log` 是否包含新 marker。
6. 在 `scripts/verify_project.sh` 的成功 fixture 中加入新 marker，并新增删除该 marker 的负向 fixture，预期 validator 输出 `FAIL verify_project mac quick add title field category context contracts`。
7. 同步更新 `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
