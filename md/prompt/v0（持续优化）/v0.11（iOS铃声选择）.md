# v0.11（iOS 铃声选择）

## 版本号

v0.11

## 版本分配依据

人工总目标要求持续优化 UI 和分类，并优化 CI。v0.10 已完成 CI 结果包索引，本轮继续在 `v0（持续优化）` 阶段递增为 v0.11，聚焦 iOS 端已存在但缺少入口的 `CompletionSound` 铃声选择体验。

## 背景

共享模型已有 `CompletionSound`，`TimerSettings.completionSound` 也已向后兼容持久化。macOS 设置页已有 Pro 铃声选择、试听和非 Pro 回退逻辑，但 iOS 设置页目前只有“到点铃声”开关，用户无法选择音色。iOS `NotificationService.playCompletionAlert(...)` 接收 `CompletionSound`，但当前生成音色没有使用该参数，需要补齐实际效果。

## 目标

- iOS 设置页新增“铃声与音色”设置区域。
- 支持显示当前 `CompletionSound`，Pro 解锁后可选择全部音色。
- 非 Pro 用户保留默认 `.chime`，不能持久保留 Pro 音色；如果设置里已有 Pro 音色，应自动回退 `.chime`。
- 增加“试听”按钮，调用 iOS `NotificationService.playCompletionAlert(...)` 播放当前音色。
- iOS `NotificationService` 的 App 内生成提示音要根据 `CompletionSound` 变化，和 macOS 音色元数据保持一致；后台系统本地通知仍使用系统默认通知声。
- 不改变 `TimerEngine` 的调用路径；到点播放仍由 `TimerEngine` 传入 `store.settings.completionSound`。

## 非目标

- 不新增 `TimerSettings` 字段，不改变 Codable 结构。
- 不改 Pro 商品 ID 或 StoreKit 权益语义。
- 不改系统通知 `UNNotificationSound.default` 行为；本轮只覆盖 App 内播放/试听的生成音。
- 不重构 SettingsView 文件结构。

## 当前架构依据

- `CompletionSound` 定义在 `ChronoFocus/Models/AppModels.swift`，已包含 `.chime`、`.bell`、`.ripple`、`.softPulse` 和 `isPro`。
- iOS 根视图为 `ChronoFocus/Views/DashboardView.swift`，负责启动、前台恢复和设置变化的轻量同步。
- iOS 设置页为 `ChronoFocus/Views/SettingsView.swift`。
- iOS 提示音播放为 `ChronoFocus/Services/NotificationService.swift`。
- macOS 参考实现为 `ChronoFocusMac/Views/MacSettingsDetailView.swift` 和 `ChronoFocusMac/Services/MacNotificationService.swift`。
- Pro 权益服务为 `PremiumAccessService`，iOS 设置页已注入 `premium`。

## 实现步骤

1. 在 `SettingsView` 的主内容中加入 `soundPanel`，位置建议放在 `premiumPanel` 后、`systemPanel` 前。
2. `soundPanel` 显示：
   - 标题“铃声与音色”。
   - 当前 Pro 状态 pill。
   - `Picker("音色", selection: $store.settings.completionSound)`，`ForEach(CompletionSound.allCases)`。
   - 非 Pro 时禁用音色选择或只允许默认音色，并显示“解锁 Pro”按钮。
   - “试听”按钮调用 `notifications.playCompletionAlert(soundVolume:vibrationEnabled:completionSound:)`。
3. 增加非 Pro 回退逻辑：
   - `DashboardView.task` 和 scene active 路径刷新 Pro 权益并清洗持久化音色。
   - `DashboardView` 监听 `premium.isProUnlocked` 和 `store.settings.completionSound`，未解锁且当前音色 `isPro` 时回退 `.chime`。
   - `SettingsView.task` 中刷新通知权限和权益，并调用 helper。
   - `onChange(of: premium.isProUnlocked)` 和 `onChange(of: store.settings.completionSound)` 中，如果未解锁且当前音色 `isPro`，回退 `.chime`。
4. 更新 iOS `NotificationService`：
   - `playCompletionAlert` 调用 `playGeneratedTone(volume:completionSound:)`。
   - `makeToneWavData(for:)` 根据 `CompletionSound` 的频率、时长、attack、release 生成音色。
   - 私有 `CompletionSound` extension 与 macOS 保持同一组参数。
5. 更新 `scripts/verify_project.sh`，增加 iOS 设置页铃声选择和 iOS 通知服务音色生成标记。
6. 更新 README、`md/flow/flow.md`、`md/test/test.md`、`update_log.md`。
