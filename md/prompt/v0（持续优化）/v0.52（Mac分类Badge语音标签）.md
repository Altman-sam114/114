# v0.52（Mac 分类 Badge 语音标签）

## 版本号

v0.52

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.51，因此本轮自动递增为 v0.52。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.51 已让 macOS 日程详情快速新增在连续录入时保留刚使用的分类，并通过最新 `origin/main` 云端 artifact 验收。

当前 iOS 计时页和日程页任务行分类 badge 已提供分类名 Voice Control input labels，并优先使用 `TaskCategoryPreset` 的标准色和图标。macOS 详细计时任务行和菜单栏小窗当前任务分类 badge 已显示分类，但 Mac 任务行只设置了可访问标签，缺少分类名 Voice Control input labels；Mac 小窗 badge 也缺少可访问标签和 input labels。两处颜色也直接使用任务保存色，和 iOS 的预设色兜底不完全一致。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. macOS 详细计时任务行分类 badge 补齐分类名 Voice Control input labels。
2. macOS 菜单栏小窗当前任务分类 badge 补齐分类名可访问标签和 Voice Control input labels。
3. Mac 两处分类 badge 颜色优先使用 `TaskCategoryPreset.matching(task.category)?.accentHex`，没有预设时再回退到 `task.accentHex`。
4. 在 `scripts/verify_project.sh` 中新增或加强 Mac 分类 badge 源码契约，锁定详细任务行和小窗 badge 的 input labels 与预设色兜底。
5. README、测试规范、核心流程文档和更新日志同步本轮 Mac 分类语义变化。

## 当前架构依据

- `MacTaskRowView` 位于 `ChronoFocusMac/Views/MacTimerDetailView.swift`，负责 Mac 详细计时页任务行显示。
- `MacMiniTaskCategoryBadgeView` 位于 `ChronoFocusMac/Views/MacMiniTimerView.swift`，负责 Mac 菜单栏小窗当前任务分类上下文。
- iOS 的 `ScheduleTaskCell` 和 `TimerTaskCategoryBadge` 已使用 `preset?.accentHex ?? task.accentHex`、`.accessibilityLabel("\(task.category)分类")` 和 `.accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`，可作为本轮 Mac 对齐依据。
- `scripts/verify_project.sh` 已检查 Mac 任务行分类 badge 和 Mac 小窗分类上下文，可在相邻源码契约中加强 input labels 与预设色兜底检查。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`；若 `git pull` 审批超时但 `git fetch origin` 后 `HEAD == origin/main`，记录该事实并继续。
2. 修改 `ChronoFocusMac/Views/MacTimerDetailView.swift`：
   - 为 `MacTaskRowView` 增加 `categoryPreset` 计算属性或等价局部复用。
   - 将 `categoryTint` 改为 `Color(hex: categoryPreset?.accentHex ?? task.accentHex)`。
   - 将 `categorySymbolName` 改为复用 `categoryPreset?.symbolName ?? "tag.fill"`。
   - 在分类 `Label` 后补充 `.accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`。
3. 修改 `ChronoFocusMac/Views/MacMiniTimerView.swift`：
   - 为 `MacMiniTaskCategoryBadgeView` 增加 `categoryPreset` 计算属性或等价局部复用。
   - 将 `tint` 改为 `Color(hex: categoryPreset?.accentHex ?? task.accentHex)`。
   - 将 `symbolName` 改为复用 `categoryPreset?.symbolName ?? "tag.fill"`。
   - 为 badge 增加 `.accessibilityLabel("\(task.category)分类")`。
   - 增加 `.accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`。
4. 修改 `scripts/verify_project.sh`：
   - 加强 `MacTaskRowView` 源码切片断言，检查预设色兜底和分类 input labels。
   - 增加 `MacMiniTaskCategoryBadgeView` 源码切片断言，检查预设色兜底、可访问标签和分类 input labels。
   - 输出或复用清晰的失败文案，便于云端 `verify_project.log` 定位。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.52: 增强 Mac 分类 badge 语音标签` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
