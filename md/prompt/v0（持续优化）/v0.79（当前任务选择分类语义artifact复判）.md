# v0.79（当前任务选择分类语义 artifact 复判）

## 版本号

v0.79

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；v0.78 已完成统计最近记录分类上下文，并通过最新 `origin/main` 云端 artifact 复判。v0.79 继续在同一 v0 持续优化阶段递增。

## 背景

`scripts/verify_project.sh` 已检查 iOS 计时页、macOS 详细计时页和 macOS 小窗当前任务选择行的分类 badge、selected trait、运行中不可切换提示和 Voice Control input labels，但这些检查目前混在通用项目验证输出中，没有独立 marker，也没有进入 `scripts/validate_ci_artifact.rb` 的 Agent C 结果包复判清单。

## 目标

- 为当前任务选择行分类语义新增独立 `verify_project.log` marker：`Current task selection accessibility contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project current task selection accessibility contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- 文档同步说明当前任务选择行分类语义已成为云端 artifact 强制复判项。

## 非目标

- 不改 SwiftUI UI 行为、模型、计时状态机或持久化规则。
- 不扩大到其它计时按钮或日程行操作。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；人工明确要求本轮只去云端验证。

## 当前架构依据

- `TimerEngine` 仍是唯一计时状态机，本轮不触碰。
- 当前任务选择行属于平台 UI 层，只读 `FocusTask` 并发起选择意图，不写核心数据。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 在 `scripts/verify_project.sh` 已有当前任务选择行源码检查后输出 `Current task selection accessibility contracts verified.`。
