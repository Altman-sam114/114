# v0.7 / 分类筛选摘要与快照清单提示词

## 版本号

v0.7

## 版本分配依据

人工总目标要求 Agent X 持续优化 UI、分类和 CI。v0.6 已完成分类新建预填、筛选优先级和 Mac 快照稳定性修复，并由 Agent C 核对最新 `origin/main` run `28702651006` artifact 通过。本轮继续在 v0.x 持续优化阶段推进分类使用便利性和 CI artifact 可验收性，因此递增为 `v0.7`。

## 背景

当前 iOS/macOS 日程页已经支持分类筛选和选中分类后新建预填，但用户选中某个分类后，列表区缺少更明显的当前筛选摘要和一键清除入口。CI 侧已经上传 Mac 快照文件，但 artifact 中没有结构化快照清单，Agent C 需要通过文件列表间接确认快照产物。

## 目标

- iOS 日程待办列表在选中分类时显示当前分类摘要，包括分类名、当前范围数量和一键清除筛选入口。
- macOS 日程详情待办列表同样显示当前分类摘要，并保持快照安全渲染。
- 空列表文案在选中分类时明确提示可清除筛选或新增该分类待办。
- Mac 快照脚本生成 `manifest.json`，记录每张快照的文件名、像素尺寸、字节数和生成时间。
- `verify_project.sh` 检查快照 manifest 存在且包含 5 张预期快照；GitHub artifact 上传该 manifest，Agent C 可直接核对。

## 非目标

- 不新增持久化字段，不改变 `FocusTask.category`、`TaskCategoryPreset` 或 `TaskCategoryFilterOption` 语义。
- 不改 `TimerEngine`、计划生成、统计计算、通知、StoreKit 或 EventKit 业务逻辑。
- 不重构 `ScheduleView` / `MacScheduleDetailView` 整体结构。
- 不引入第三方依赖。
- 不提交未跟踪的 `md/1.md` 或其他无关文件。

## 当前架构依据

- iOS 分类筛选位于 `ScheduleView.swift` 的 `TaskCategoryFilterBar`，筛选状态为 `@State selectedCategory`。
- macOS 分类筛选位于 `MacScheduleDetailView.swift` 的 `MacTaskListPanelView` / `MacCategoryFilterBar`，筛选状态通过 binding 从 `MacScheduleDetailView` 上提。
- 分类计数已有 `taskCount(in:)`，可复用为摘要显示。
- Mac 快照由 `scripts/render_mac_snapshots.swift` 生成到 `/tmp/chronofocus-mac-snapshots/`，`scripts/verify_project.sh` 负责编译运行并检查快照。
- `.github/workflows/ci-results.yml` 会复制 `/tmp/chronofocus-mac-snapshots/` 到 `ci-results/project-reports/mac-snapshots/` 并上传 artifact。

## 实现步骤

1. iOS UI：
   - 新增小型 `SelectedCategorySummaryView` 或同等子视图，展示选中分类、当前范围数量和“清除”按钮。
   - 在 `ScheduleView.taskList` 的分类筛选条下方、空列表/任务列表上方显示摘要。
   - “清除”只把 `selectedCategory = nil`，不影响日期、范围或新建表单逻辑。
   - 空列表选中分类时文案改为可操作表达，提示清除筛选或点击右上角新增该分类待办。
2. macOS UI：
   - 新增 `MacSelectedCategorySummaryView` 或同等子视图，展示选中分类、未完成数量和“清除”按钮。
   - 在 `MacTaskListPanelView` 的分类筛选条下方显示摘要。
   - 快照渲染时避免 bordered Button，使用静态 chip 或 plain button 样式，保持 `ImageRenderer` 不出现黄色占位。
3. CI / 快照：
   - `scripts/render_mac_snapshots.swift` 在成功渲染全部快照后写入 `manifest.json`。
   - manifest 至少包含 `generatedAt`、`snapshots` 数组；每项包含 `fileName`、`width`、`height`、`byteCount`。
   - 渲染脚本内部验证 manifest 中包含 `mini-timer.png`、`detail-timer.png`、`detail-schedule.png`、`detail-analytics.png`、`detail-settings.png`。
   - `scripts/verify_project.sh` 检查 `/tmp/chronofocus-mac-snapshots/manifest.json` 存在，并用 Python/JSON 检查 5 张快照条目。
   - `.github/workflows/ci-results.yml` 的 `CI_PROCESS_VERSION` 更新为 v0.7，并在 manifest 的 `projectSpecificReports` 中记录 `mac_snapshot_manifest`。
4. 文档：
   - 更新 `README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 或 `md/test/test.md` 中受影响的分类 UI 和快照 manifest 描述。

## 验收标准

- iOS 选中分类后，待办列表显示当前分类、数量和清除入口；清除后回到全部分类。
- macOS 选中分类后，待办列表显示当前分类、数量和清除入口；清除后快速新增分类不被额外覆盖。
- 空分类/空时间范围文案更明确，不误导用户以为待办丢失。
- 本地 `bash scripts/verify_project.sh` 生成 5 张 Mac 快照和 `/tmp/chronofocus-mac-snapshots/manifest.json`，并输出 `Project structure verified.`。
- GitHub artifact 中包含 `project-reports/mac-snapshots/manifest.json`，manifest 的 `branch`、`commitSha`、run id、run attempt 与最新 `origin/main` 一致，且 static/project/Mac/iOS outcome 全为 `success`。

- 不要让清除分类影响日期范围、任务数据或新建表单保存逻辑。
- 不要在 macOS 快照路径引入新的 bordered Button 黄色占位。
- 不要上传大体积 DerivedData、历史 artifact 或无关图片。
- 不要提交 `md/1.md` 或其他未跟踪无关文件。
