# v0.63（计划面板操作语义）

## 版本号

v0.63

## 版本分配依据

人工总目标要求持续优化 UI 易用性、分类体验和 CI 测试。v0.62 已完成 Mac 计划分类上下文并通过云端 artifact 验收；本轮继续按小版本递增到 v0.63，聚焦自动计划面板的生成/清空操作语义。

## 背景

iOS 和 macOS 自动计划面板都有“按日程生成”和“清空”操作，但当前按钮语义较泛，未读出当前未完成计划轮数。用户在反复调整日程和分类后，不容易确认生成/清空动作会影响已有计划规模。

## 目标

- iOS 自动计划面板“按日程生成”按钮补齐当前未完成计划轮数语义和 Voice Control input labels。
- iOS 自动计划面板“清空”按钮补齐当前未完成计划轮数语义和 Voice Control input labels。
- macOS 自动计划面板真实按钮补齐同等语义。
- macOS 快照静态按钮保持短可见标题，但通过可访问覆盖文本保留计划轮数语义。
- 在 `scripts/verify_project.sh` 增加源码契约 marker，并让 `scripts/validate_ci_artifact.rb` 复判该 marker。
- 同步 README、测试规范、核心流程和更新记录。

## 非目标

- 不改计划生成、清空、排序、完成状态、`TimerEngine` 或 `FocusStore` 业务逻辑。
- 不改变可见按钮文案或布局密度。
- 不运行本地测试、build、lint、`git diff --check` 或脚本验证；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- iOS 自动计划面板位于 `ChronoFocus/Views/ScheduleView.swift` 的 `pomodoroPlanPanel`。
- macOS 自动计划面板位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的 `MacPlanPanelView`。
- 当前未完成计划轮数由 `store.pomodoroPlan.filter { !$0.isCompleted }.count` 计算，不需要新增模型字段。
- `scripts/verify_project.sh` 是项目专属验证入口；云端 workflow 会运行它并把日志纳入 artifact。

## 实现步骤

1. 同步最新 `main`，确认没有无关 tracked diff，不读取或提交用户未跟踪文件。
2. 修改 iOS `ScheduleView`：
   - 增加计划未完成轮数 helper，避免按钮语义重复计算散落。
   - “按日程生成”按钮 accessibility label 读出当前未完成计划轮数。
   - “清空”按钮 accessibility label/input labels 读出当前未完成计划轮数。
3. 修改 macOS `MacScheduleDetailView`：
   - `MacPlanPanelView` 增加计划未完成轮数 helper。
   - 真实“按日程生成”和“清空”按钮补齐 accessibility label/input labels。
   - `MacStaticScheduleActionChipView` 支持可选可访问标签覆盖；快照静态生成/清空按钮使用覆盖文本，不改变可见短标题。
4. 修改 `scripts/verify_project.sh`：
   - 增加 iOS/Mac 计划面板操作语义源码契约。
   - 输出稳定 marker，例如 `Plan panel action accessibility contracts verified.`。
   - 增加缺失 marker 负向 fixture。
5. 修改 `scripts/validate_ci_artifact.rb`：
   - 复判 `verify_project.log` 中的新增 marker。
6. 同步文档和 `update_log.md`。
