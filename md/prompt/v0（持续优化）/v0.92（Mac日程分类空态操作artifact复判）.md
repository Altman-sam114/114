# v0.92（Mac 日程分类空态操作 artifact 复判）

## 版本号

v0.92

## 版本分配依据

人工未指定版本号；当前 `md/prompt/v0（持续优化）/` 和 `update_log.md` 最新正式轮次为 v0.91，因此本轮自动递增为 v0.92。

## 背景

总目标仍是持续优化 ChronoFocus 的 UI 易用性、分类工作流和 CI 云端验收。v0.91 已完成 iOS 日程分类筛选无结果空态操作与 artifact 复判。macOS 详细窗口的待办列表在选中分类且无未完成待办时，仍只显示一段说明文字，用户需要回到上方摘要或左侧快速新增面板才能操作。空态本身应提供直接新增此分类和清除筛选的入口，形成 iOS/macOS 分类工作流的一致性。

## 目标

- 在 `MacScheduleDetailView` 的 `MacTaskListPanelView` 中，当 `visibleTasks.isEmpty` 且存在 `selectedCategory` 时，显示可操作的 Mac 分类空态组件。
- 新组件显示“暂无 X 分类待办”和说明文案，保留 Mac 玻璃面板内的紧凑视觉风格。
- 空态提供“新增此分类”和“清除筛选”两个动作。
- “新增此分类”调用现有 `onAddTaskInCategory(selectedCategoryName)`，让左侧快速新增表单预填并聚焦该分类。
- “清除筛选”设置 `selectedCategory = nil`。
- 两个按钮必须有稳定点击区、分类相关 `accessibilityLabel` 和 Voice Control input labels。
- `scripts/verify_project.sh` 新增独立源码契约 marker：`Mac schedule category empty state action contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 Agent C artifact 复判项：`PASS verify_project mac schedule category empty state action contracts`。
- 增加 marker 缺失负向 fixture，防止旧 artifact 或缺少该契约的结果包被放行。

## 非目标

- 不改 iOS `ScheduleView`，不重复 v0.91 的 iOS 空态。
- 不改 `FocusStore`、`TimerEngine`、共享模型、持久化格式或计划生成逻辑。
- 不改 macOS 日历面板的普通范围空态。
- 不重构 `MacScheduleDetailView` 大结构，不拆分文件。
- 不引入第三方依赖。

## 当前架构依据

- `FocusStore` 仍是任务、分类、计划和统计的唯一核心数据仓库。
- `TimerEngine` 仍是唯一计时状态机，本轮不触碰计时状态。
- `MacTaskListPanelView` 已有 `selectedCategory`、`visibleTasks`、`onAddTaskInCategory` 和 `MacSelectedCategorySummaryView`，适合在 View 层补齐空态操作。
- `MacSelectedCategorySummaryView` 已提供相同分类的新增/清除语义，本轮空态按钮应保持一致的动作和可访问文案。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认 `HEAD == origin/main`，且只有人工未跟踪文件可忽略。
2. 在 `MacTaskListPanelView` 的 `visibleTasks.isEmpty` 分支中：
   - 若 `selectedCategory` 有值，渲染 `MacScheduleCategoryEmptyStateView(category:isSnapshotRendering:onAddTask:onClear:)`。
   - 否则保留现有普通 `Text(emptyText)`。
3. 新增 `private struct MacScheduleCategoryEmptyStateView: View`，使用 `Label`、说明文本和两个按钮呈现分类空态；快照模式用静态 action view 保持渲染稳定。
4. 为“新增此分类”和“清除筛选”按钮加入稳定最小宽高、分类 `accessibilityLabel` 和 `accessibilityInputLabels`。
5. 在 `scripts/verify_project.sh` 中新增源码契约，绑定组件名、标题/说明、动作接线、按钮文案、可访问标签和 input labels，并输出新 marker。
6. 在 `scripts/validate_ci_artifact.rb` 中复判 `verify_project.log` 是否包含新 marker。
7. 在 `scripts/verify_project.sh` 的成功 fixture 中加入新 marker，并新增删除该 marker 的负向 fixture，预期 validator 输出 `FAIL verify_project mac schedule category empty state action contracts`。
8. 同步更新 `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
