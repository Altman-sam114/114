# v0.84 统计分类投入次数上下文 artifact 复判

## 版本号

v0.84

## 版本分配依据

人工未指定版本号。当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新正式轮次为 v0.83，本轮继续“优化 UI、优化分类、优化 CI 测试”的持续优化目标，因此递增为 v0.84。

## 背景

统计页“分类投入”当前只展示分类、累计时长和占比。用户在按分类复盘时无法区分某个分类是由一次长专注形成，还是多次短专注形成；这会降低分类复盘的可用性。项目已有 `Analytics category share accessibility contracts verified.`，但该契约只证明分类/时长/占比存在，尚未证明“专注次数”上下文会进入 iOS、macOS 和云端 artifact 复判。

## 目标

- 在共享统计聚合 `CategoryFocus` 中增加分类专注完成次数，不改变持久化格式。
- iOS `AnalyticsView` 的分类投入行展示“X 次专注”，并在可访问标签和 Voice Control 输入标签中包含分类次数上下文。
- macOS `MacAnalyticsDetailView` 的分类投入行同步展示和读出“X 次专注”。
- `scripts/verify_project.sh` 增加独立 marker：`Analytics category share session count contracts verified.`。
- `scripts/validate_ci_artifact.rb` 增加对应 artifact 复判：`PASS verify_project analytics category share session count contracts`。
- `scripts/verify_project.sh` 的小型成功 fixture 和负向 fixture 覆盖新 marker 缺失场景。
- 同步更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不新增统计筛选交互、图表类型或复杂报表。
- 不修改 `FocusSession`、`FocusTask` 的持久化字段。
- 不改 `TimerEngine`、通知、Live Activity、StoreKit、EventKit 或计划生成规则。
- 不跑本机完整 Xcode build，除非定位本轮失败必须这么做。

## 当前架构依据

- `FocusStore.categoryBreakdown()` 从已完成的专注会话聚合分类统计，是本轮新增次数的唯一数据来源。
- `CategoryFocus` 是非 Codable 的展示聚合模型，新增 `sessionCount` 不涉及历史 JSON 兼容解码。
- iOS 和 macOS 统计页都通过 `store.categoryBreakdown()` 渲染分类投入，适合共享同一聚合字段。
- CI 结果包由 `.github/workflows/ci-results.yml` 上传，Agent C 使用 `scripts/validate_ci_artifact.rb` 对 `verify_project.log` marker 做结构化复判。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认 `git status --short --branch` 无无关 diff。
2. 修改 `ChronoFocus/Models/AppModels.swift`：为 `CategoryFocus` 增加 `sessionCount: Int`。
3. 修改 `ChronoFocus/Services/FocusStore.swift`：`categoryBreakdown()` 按分类聚合时把 `sessions.count` 写入 `CategoryFocus`。
4. 修改 `scripts/test_mac_core.swift`：补充断言，确认完成一条专注会话后分类统计 `sessionCount == 1`。
5. 修改 `ChronoFocus/Views/AnalyticsView.swift`：分类投入行可见展示“X 次专注”；`categoryShareAccessibilityLabel(for:)` 和 input labels 包含次数上下文。
6. 修改 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift`：保持与 iOS 一致。
7. 修改 `scripts/verify_project.sh`：保留旧 `Analytics category share accessibility contracts verified.`，新增次数上下文源码契约和独立 marker；更新小型成功 fixture；新增 marker 缺失负向 fixture。
8. 修改 `scripts/validate_ci_artifact.rb`：新增 `verify_project analytics category share session count contracts` 检查。
9. 更新 README、测试规范、核心流程和流程图，说明 v0.84 起分类投入统计显示/读出专注次数，并由 artifact validator 复判。
10. 更新 `update_log.md`，记录版本、日期、核心变更、关键文件、验证结果和遗留事项。

## 验收标准

- iOS 和 macOS 分类投入行可见显示每个分类的专注次数。
- iOS 和 macOS 分类投入行可访问标签包含分类、时长、次数和占比。
- `FocusStore.categoryBreakdown()` 返回的每个 `CategoryFocus` 包含准确 `sessionCount`。
- `bash scripts/verify_project.sh` 输出 `Analytics category share session count contracts verified.` 和 `Project structure verified.`。
- `scripts/validate_ci_artifact.rb` 对云端 artifact 输出 `PASS verify_project analytics category share session count contracts`。
- 负向 fixture 删除该 marker 后 validator 必须失败，并出现 `FAIL verify_project analytics category share session count contracts`。

## 风险和禁止项

- 不要把 `CategoryFocus` 改成 Codable 持久化模型。
- 不要为了展示次数改动会话记录规则或统计口径；只统计 `mode == .focus && completed` 的已有会话集合。
- 不要移除旧 marker 或弱化既有 artifact 校验。
- 不要引入第三方依赖或跨平台 API 污染。
- 不要把 token、密钥或 gh 登录信息写入文件或日志。

## main 同步、提交和 push 要求

- 本轮固定使用 `main`。
- 开始前同步 `origin/main` 并确认无无关 diff。
- 提交前检查 `git diff --check` 和 `bash scripts/verify_project.sh` 结果。
- 只提交本轮相关文件。
- 提交信息：`v0.84: 增加统计分类投入次数语义`。
- push：`git push origin main`，触发 `.github/workflows/ci-results.yml`。

## GitHub Actions 与 Agent C 复判要求

- Agent B push 后记录 commit SHA、workflow run id、attempt 和 artifact 名称。
- Agent C 必须用 `gh auth login` 后下载最新 `origin/main` 对应 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 必须核对 manifest、artifact index、run context、JUnit、failure summary、主日志、Mac/iOS build 日志、`.xcresult`、Mac 快照和 `verify_project.log`。
- Agent C 必须运行：

```bash
ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> \
  --commit <origin-main-sha> \
  --run-id <run_id> \
  --attempt <run_attempt>
```

- Agent C 验收必须确认 artifact 中包含 `PASS verify_project analytics category share session count contracts`，且 manifest 的 `branch=main`、`commitSha`、run id、attempt 与最新 `origin/main` 完全一致。
