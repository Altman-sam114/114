# v0.47（分类摘要契约日志复判）

## 版本号

v0.47

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.46，因此本轮自动递增为 v0.47。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.45 和 v0.46 已连续增强 iOS/Mac 日程筛选摘要的按钮语义与点击区，`scripts/verify_project.sh` 已能在云端锁定这些源码契约。但当前 `verify_project.log` 只有 `Category chip accessibility contracts verified.` 这个统一 marker；Agent C 和 validator 只能间接知道分类摘要按钮契约已经跑过，不利于快速定位摘要动作契约是否被跳过或回退。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/verify_project.sh` 在分类摘要动作、按钮语义和点击区源码契约通过后输出稳定 marker：`Category summary action contracts verified.`。
2. `scripts/validate_ci_artifact.rb` 新增对该 marker 的复判，输出独立 PASS 项。
3. `scripts/verify_project.sh` 的小型成功 fixture 同步补齐该 marker，确保 validator 正向 fixture 与真实日志一致。
4. `scripts/verify_project.sh` 锁定 validator 必须包含新的摘要 marker 复判逻辑。
5. README、测试规范、核心流程文档和更新日志同步本轮 CI 复判范围。

## 非目标

- 不改变 SwiftUI UI、分类筛选状态、按钮动作、任务模型、持久化、计划生成或计时状态机。
- 不修改 GitHub Actions workflow、artifact 文件列表、JUnit 结构或 Xcode 构建命令。
- 不新增大体积 artifact、历史 artifact 下载或本地测试数据。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `scripts/verify_project.sh` 是项目专属验证入口，已有源码片段契约覆盖 iOS 日程摘要、计时摘要、空态清除、Mac 日程摘要、分类 chip 和预设按钮。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载 artifact 后的结构化复判脚本，已有 `verify_project category accessibility contracts` 检查。
- 新 marker 只提升日志可观测性和 artifact 复判强度，不改变产品运行时代码。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认当前 `HEAD == origin/main` 且只有用户未跟踪文件未触碰。
2. 修改 `scripts/verify_project.sh`：
   - 在分类摘要动作相关源码契约通过后输出 `Category summary action contracts verified.`。
   - 小型成功 fixture 的 `verify_project.log` 增加该 marker。
   - 增加 `grep -q "Category summary action contracts verified." scripts/validate_ci_artifact.rb`，锁定 validator 复判逻辑存在。
3. 修改 `scripts/validate_ci_artifact.rb`：
   - 新增独立检查项，例如 `verify_project category summary action contracts`。
   - 该检查读取 `verify_project.log` 并要求包含 `Category summary action contracts verified.`。
4. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 只检查 diff 和 status，不运行本地测试命令。
6. 提交 `v0.47: 增强分类摘要契约日志复判` 并 push 到 `origin/main`。
7. Agent C 下载最新 run artifact 并按下方标准验收。
