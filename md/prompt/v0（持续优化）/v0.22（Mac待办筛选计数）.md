# v0.22（Mac待办筛选计数）

## 版本号

v0.22

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.21 已收紧分类摘要测试，并通过最新 `origin/main` 云端 artifact 验收。本轮回到 UI 分类体验，递增为 v0.22。

## 背景

macOS 日程详情的待办列表已经支持分类筛选和选中分类摘要，但标题右侧仍只显示总未完成数。用户筛选后需要从下方摘要判断当前分类数量，不如 iOS 计时页的 `筛选数/总数` 反馈直接。本轮补齐 Mac 待办列表标题计数，让筛选状态更容易扫读。

## 目标

- `MacTaskListPanelView` 标题右侧无筛选时继续显示总未完成数。
- 选中分类时显示 `visibleTasks.count/总数 项未完成`。
- 当前未完成总数为 0 时显示 `0 项未完成`，避免旧筛选状态下出现 `0/0`。
- 计数文本保持紧凑、单行并支持轻微缩放，避免 Mac 详情窗口较窄时挤压。
- `scripts/verify_project.sh` 增加 Mac 待办筛选计数 marker。
- README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md` 同步当前行为。

## 非目标

- 不修改 `FocusStore`、`TimerEngine`、计划生成、统计或持久化逻辑。
- 不改变任务筛选规则、排序规则或分类摘要行为。
- 不改 iOS UI。
- 不新增第三方依赖。
- 不做大规模 View 拆分。

## 当前架构依据

- `MacTaskListPanelView` 已通过 `@Binding var selectedCategory: String?` 和 `visibleTasks` 渲染筛选后的任务列表。
- `visibleTasks` 已经基于 `store.upcomingTasks()` 和 `selectedCategory` 计算，不需要新增模型字段。
- `scripts/verify_project.sh` 是本地和云端的项目专属验证入口，可用 marker 锁定本轮 UI contract。

## 实现步骤

1. 更新 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 在 `MacTaskListPanelView` 新增 `private var taskListCountText: String`。
   - 总数为 0 时返回 `"0 项未完成"`。
   - 无选中分类时返回 `"\(totalCount) 项未完成"`。
   - 有选中分类且总数大于 0 时返回 `"\(visibleTasks.count)/\(totalCount) 项未完成"`。
   - 标题右侧 `Text` 使用 `taskListCountText`，并加 `.font(.caption)`、`.lineLimit(1)`、`.minimumScaleFactor(0.78)`。
2. 更新 `scripts/verify_project.sh`：
   - 增加 `taskListCountText`、`Text(taskListCountText)` 和属性片段 marker，覆盖 0 总数、筛选数和总数文案。
3. 更新文档：
   - README macOS 功能描述补充分类筛选后显示筛选/总数。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 同步 Mac 端筛选计数反馈。
   - `md/test/test.md` smoke 基线补充 Mac 待办筛选计数 marker。
   - `update_log.md` 追加 v0.22 记录。
