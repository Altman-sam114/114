# v0.35（分类预设按钮可访问语义）

## 版本号

v0.35

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.34，因此本轮自动递增为 v0.35。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。分类筛选 chip 已经具备已选中状态、操作提示、selected trait 和 Voice Control input labels；但新建/编辑待办时的常用分类预设按钮目前只有视觉选中态。补齐 iOS 与 macOS 预设按钮的可访问语义后，VoiceOver 用户可以听到当前预设是否已选中，Voice Control 用户也可以用分类名或“分类名分类”直接触发。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS `TaskCategoryPresetPicker` 的每个分类预设按钮补充可访问 label、hint、selected trait 和 Voice Control input labels。
2. macOS `MacCategoryPresetPicker` 的每个分类预设按钮补充同样语义。
3. `scripts/verify_project.sh` 增加静态契约，锁定 iOS/Mac 预设按钮可访问实现。
4. README、测试规范、核心流程文档和更新日志同步本轮 UI/CI 覆盖范围。

## 非目标

- 不改变分类预设数据、颜色、图标或排序。
- 不改变分类筛选状态机、任务持久化模型或 `FocusStore`。
- 不改变 GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `ScheduleView.swift` 的 `TaskCategoryPresetPicker` 负责 iOS 新建/编辑待办分类预设选择。
- `MacScheduleDetailView.swift` 的 `MacCategoryPresetPicker` 负责 macOS 快速新增分类预设选择。
- 两者已有 `isSelected(_:)` 视觉状态，可复用该状态生成可访问 label/hint/trait。
- `scripts/verify_project.sh` 已有分类 chip 和摘要动作可访问契约，适合扩展到分类预设按钮。
