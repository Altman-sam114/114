# v0.41（计时页分类 badge 可访问标签）

## 版本号

v0.41

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.40，因此本轮自动递增为 v0.41。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.40 已让 iOS 日程页任务行分类 badge 具备图标、颜色和可访问标签；但 iOS 计时页当前待办列表的 `TimerTaskCategoryBadge` 仍是 `HStack + Image + Text`，没有稳定可访问标签和 Voice Control input labels。补齐后，计时页任务行与日程页在分类 badge 语义上保持一致，也能通过云端静态契约防止回退。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 计时页 `TimerTaskCategoryBadge` 使用 `Label(task.category, systemImage: categorySymbolName)` 统一图标和文字语义。
2. 新增 `categorySymbolName` helper，优先使用 `TaskCategoryPreset.matching(task.category)?.symbolName`，未知分类回退 `tag.fill`。
3. 分类 badge 补充 `.accessibilityLabel("\(task.category)分类")`。
4. 分类 badge 补充 `.accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`，方便 Voice Control 直接说分类名。
5. `scripts/verify_project.sh` 增加计时页分类 badge 静态契约，锁定 preset、Label、可访问标签和 input labels。
6. README、测试规范、核心流程文档和更新日志同步本轮覆盖范围。

## 非目标

- 不改变任务筛选逻辑、选中状态、计时状态机、任务模型或持久化字段。
- 不改变 iOS 日程页、macOS UI、GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `TimerTaskCategoryBadge` 只属于 iOS `TimerView` 的展示层，当前被 `TaskRow` 复用，不持有业务状态。
- `TaskCategoryPreset.matching(_:)` 已作为分类图标和代表色的统一来源。
- `scripts/verify_project.sh` 已在 Ruby heredoc 中用 `source_slice` 锁定计时页分类摘要和筛选 chip，可在 `timer_summary_source` 检查后增加 `TimerTaskCategoryBadge` 切片。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `ChronoFocus/Views/TimerView.swift` 的 `TimerTaskCategoryBadge` 中新增 `categorySymbolName` helper。
3. 将 badge 主体改为 `Label(task.category, systemImage: categorySymbolName)`，保留现有 tint、padding 和 Capsule 背景。
4. 将字体提升为 `.caption.weight(.bold)`，保持与日程页任务行分类 badge 语义一致。
5. 添加 `.accessibilityLabel("\(task.category)分类")` 和 `.accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`。
6. 在 `scripts/verify_project.sh` 的计时页 Ruby 检查区新增 `timer_task_badge` slice 断言。
7. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
8. 只检查 diff 和 status，不运行本地测试命令。
9. 提交 `v0.41: 增强计时页分类 badge 可访问性` 并 push 到 `origin/main`。
10. Agent C 下载最新 run artifact 并按下方标准验收。
