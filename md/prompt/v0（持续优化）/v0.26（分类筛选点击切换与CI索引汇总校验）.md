# v0.26（分类筛选点击切换与 CI 索引汇总校验）

## 版本号

v0.26

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.25，因此本轮自动递增为 v0.26。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.25 已补齐 iOS 日程筛选计数，但三端分类筛选 chip 仍需要通过“全部”或摘要“清除”退出当前分类；CI artifact validator 已校验 index 身份、路径和本地文件存在性，但尚未复算 `ci-artifact-index.json` 的 `totals` 是否与 `entries` 自洽。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页、iOS 计时页和 macOS 日程详情页的分类筛选 chip 支持重复点击当前分类后取消筛选。
2. `scripts/validate_ci_artifact.rb` 复算 artifact index 的 `entryCount`、`missingRequiredCount`、`fileByteCount` 和 `directoryRecursiveByteCount`，确认 totals 与 entries 内部一致。
3. `scripts/verify_project.sh` 增加分类点击切换源码 contract，以及 index totals 篡改负向 fixture。
4. 同步 README、测试规范、核心流程图和更新日志。

## 非目标

- 不重构分类组件，不抽共享 UI。
- 不改变 `FocusTask.category`、`TaskCategoryPreset`、`TaskCategoryFilterOption` 或持久化模型。
- 不改变 `TimerEngine`、`FocusStore`、计划生成、统计或提醒逻辑。
- 不改 GitHub Actions workflow 的 artifact 生成逻辑。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- 分类筛选是 View 层 `@State` / `@Binding` 状态，真实任务分类仍由 `FocusStore` 管理。
- iOS 日程页使用 `TaskCategoryFilterBar`，iOS 计时页使用 `TimerTaskCategoryFilterBar`，macOS 日程详情使用 `MacCategoryFilterBar`。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载云端结果包后的结构化复判辅助脚本；它不能替代 Agent C 对最新 `origin/main` run 和 artifact 来源的一致性核对。
- `.github/workflows/ci-results.yml` 已在云端生成 `ci-artifact-index.json` 的 entries 和 totals。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在三个分类 filter bar 内新增 `toggleCategory(_:)`，当点击已选分类时把 `selectedCategory` 置为 `nil`，否则切到目标分类。
3. 在 `scripts/validate_ci_artifact.rb` 读取 index entries 后计算预期 totals，并新增 `index totals consistency` 检查。
4. 在 `scripts/verify_project.sh`：
   - 增加三端 filter bar 点击切换源码切片检查。
   - 正向 fixture 补齐 `fileByteCount` 和 `directoryRecursiveByteCount`。
   - 增加 `corrupt_index_totals_fixture`，篡改 index totals 后确认 validator 输出 `FAIL index totals consistency`。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.26: 增加分类筛选切换与 CI 汇总校验` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。

## 验收标准

- 三端分类筛选 chip 未选时点击进入筛选，已选时再次点击恢复全部。
- 原有“全部”、摘要清除、iOS 新增此分类、Mac 快速新增预填行为保持不变。
- `scripts/validate_ci_artifact.rb` 拒绝 totals 与 entries 不一致的 artifact index。
- `scripts/verify_project.sh` 的正向 fixture 和 totals 负向 fixture 被云端执行并通过。
- 最新 `origin/main` commit 的 GitHub Actions artifact 通过 Agent C 核对，manifest、run context、artifact index、run id、attempt 和 commit SHA 全部一致。

## 风险和禁止项

- 禁止把本地输出或旧 artifact 当作验收结果。
- 禁止扩大为分类组件重构或跨平台共享 UI 抽象。
- 禁止改动 `TimerEngine` 或 `FocusStore` 的核心业务语义。
- 禁止触碰未跟踪用户文件 `md/1.md` 和 `md/通用云端迭代升级提示词.md`。
- index totals 校验只复算 index 内部 entries，不比较 `ci-artifact-index.json` 自身的实际文件大小，避免自引用文件大小稳定化问题。
