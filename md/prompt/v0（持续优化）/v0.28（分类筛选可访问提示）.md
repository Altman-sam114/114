# v0.28（分类筛选可访问提示）

## 版本号

v0.28

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.27，因此本轮自动递增为 v0.28。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.26 已让三端分类 chip 支持再次点击已选分类退出筛选，v0.27 已补齐 Mac 选中分类摘要快捷新增。当前分类筛选 chip 的 VoiceOver label 只读出分类名和数量，没有明确“已选中”状态，也没有提示已选中时再次点击会清除筛选。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页分类 chip、iOS 计时页分类 chip、macOS 日程详情分类 chip 都补充可访问状态。
2. VoiceOver label 包含分类名、数量和已选中状态。
3. VoiceOver hint 区分“点击筛选此分类”“再次点击清除筛选”和“显示全部分类/当前显示全部分类”。
4. `scripts/verify_project.sh` 增加三端分类 chip 可访问 label/hint 的源码 contract。
5. 同步 README、测试规范、核心流程图和更新日志。

## 非目标

- 不改变分类筛选视觉布局。
- 不改变 v0.26 的点击切换逻辑。
- 不改 `FocusStore`、`TimerEngine`、计划、统计、提醒或持久化逻辑。
- 不抽共享 UI，不重构现有长 View 文件。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- 三端分类筛选 chip 已是 SwiftUI `Button`，满足可点击控件基本语义。
- iOS 日程页使用 `TaskCategoryFilterChip`，iOS 计时页使用 `TimerTaskCategoryFilterChip`，macOS 日程详情使用 `MacCategoryFilterChip`。
- v0.26 已在三端 chip 内建立 `toggleCategory(_:)`，因此可访问 hint 应准确描述当前行为。
- `scripts/verify_project.sh` 已有三端分类点击切换源码切片检查，适合继续补充可访问 label/hint contract。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在三个分类 chip struct 中增加 `accessibilityStateText` 和 `accessibilityHintText` 计算属性。
3. 将 `.accessibilityLabel` 更新为包含已选中状态。
4. 增加 `.accessibilityHint(accessibilityHintText)`，让用户知道当前点击会筛选、清除筛选或显示全部。
5. 在 `scripts/verify_project.sh` 增加三端 chip 的可访问源码 contract。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
7. 只检查 diff 和 status，不运行本地测试命令。
8. 提交 `v0.28: 增加分类筛选可访问提示` 并 push 到 `origin/main`。
9. Agent C 下载最新 run artifact 并按下方标准验收。
