# v0.70（统计分类占比语义）

## 版本号

v0.70

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.69，本轮继续围绕“优化 UI，更加方便使用，优化分类”做小步增强，因此自动递增为 v0.70。

## 背景

iOS `AnalyticsView` 和 macOS `MacAnalyticsDetailView` 的“分类投入”面板已列出分类名称、投入时长和进度条，但每行没有直接显示占比，也没有把“分类 + 时长 + 占比”组合成可访问语义。用户查看分类投入时，需要从进度条自行估算比例；VoiceOver/Voice Control 用户也缺少可读的分类投入摘要。

## 目标

- iOS “分类投入”每个分类行显示分类投入总量占比。
- macOS “分类投入”每个分类行显示分类投入总量占比。
- iOS/macOS 每个分类投入行补齐可访问标签和 Voice Control input labels，包含分类、时长和占比。
- `scripts/verify_project.sh` 增加源码契约检查和缺失 marker 负向 fixture。
- `scripts/validate_ci_artifact.rb` 增加云端 artifact 中 verify marker 的复判。
- 同步 README、测试规范、核心流程文档、流程图和更新日志。

## 非目标

- 不修改统计模型、`FocusStore`、`TimerEngine`、持久化格式或计划生成逻辑。
- 不修改 Pro 权限逻辑、报表区间算法或分类统计数据来源。
- 不调整 CI artifact 版本号、上传清单或 workflow 阶段。
- 不引入第三方依赖。

## 当前架构依据

- iOS 分类统计 UI 位于 `ChronoFocus/Views/AnalyticsView.swift` 的 `categoryChart`。
- macOS 分类统计 UI 位于 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift` 的 `MacCategoryChartPanelView`。
- 分类统计数据来自共享 `FocusStore.categoryBreakdown()`，本轮只在展示层基于现有分类汇总计算占比。
- `scripts/verify_project.sh` 负责源码契约和云端项目专属验证 marker。
- `scripts/validate_ci_artifact.rb` 负责 Agent C 下载 artifact 后复判 `verify_project.log` marker。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `ChronoFocus/Views/AnalyticsView.swift`：
   - 在分类投入行的时长旁显示 `xx%` 占比。
   - 新增本地 helper 计算分类占比，分母使用分类汇总总量并兜底为 1 避免除零。
   - 每个分类行增加 `.accessibilityElement(children: .ignore)`。
   - 增加 `.accessibilityLabel("\(item.category)分类投入，\(item.seconds.hourMinuteText)，占分类投入 \(percent)%")`。
   - 增加 `.accessibilityInputLabels([Text(item.category), Text("\(item.category)分类"), Text("\(item.category)分类投入")])`。

2. 更新 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift`：
   - 对 `MacCategoryChartPanelView` 做同等占比文本和可访问语义增强。
   - 保持现有 Mac 面板宽度和布局风格，不引入新视觉体系。

3. 更新 `scripts/verify_project.sh`：
   - 增加源码契约检查，确认 iOS/macOS 分类投入行包含占比文本、`accessibilityLabel`、`accessibilityInputLabels` 和 `accessibilityElement(children: .ignore)`。
   - 输出 marker：`Analytics category share accessibility contracts verified.`。
   - 小型成功 fixture 的 `verify_project.log` 增加该 marker。
   - 增加负向 fixture，移除该 marker 后要求 validator 输出 `FAIL verify_project analytics category share accessibility contracts`。

4. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project analytics category share accessibility contracts` 复判，要求 `verify_project.log` 包含本轮 marker。

5. 同步文档：
   - README 描述统计分类投入占比和可访问语义。
   - `md/test/test.md` 更新验证范围、Agent C PASS 清单和 JUnit/artifact 复判说明中新增 marker。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实核心验证流。
   - `update_log.md` 追加 v0.70 记录。
