# v0.17（分类筛选快捷新增）

## 版本号

v0.17

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.16 已收紧 CI 结果包校验并通过最新 `origin/main` 云端 artifact 验收。本轮继续在 `v0（持续优化）` 阶段递增为 v0.17，回到 UI 和分类使用体验，补强选中分类后的新增路径。

## 背景

iOS 日程页和 macOS 日程详情页都已支持分类筛选，并且选中分类后新增待办会预填该分类。但 iOS 选中分类摘要目前只有“清除”按钮，空态文案提示用户点右上角新增；Mac 左侧快速新增面板会被筛选分类预填，但界面没有明确提示“当前新增会使用筛选分类”。本轮把已有行为显性化，让用户在分类上下文里更容易新增任务。

## 目标

- iOS `SelectedCategorySummaryView` 增加“新增此分类”按钮。
- 点击 iOS “新增此分类”直接打开新增待办 sheet，并沿用当前 `selectedCategory` 作为 `TaskEditorView(initialCategory:)`。
- iOS 分类空态文案改为指向摘要内的新增按钮，而不是只提示右上角。
- macOS `MacScheduleDetailView` 的快速新增面板在选中分类时显示“已预填该分类”的上下文提示。
- `scripts/verify_project.sh` 增加本轮 UI marker，避免后续误删。
- README、`md/flow/flow.md`、`md/test/test.md`、`update_log.md` 同步受影响说明。

## 非目标

- 不修改 `FocusStore`、`TimerEngine`、计划生成、统计或持久化逻辑。
- 不新增分类模型字段。
- 不改默认分类列表和排序算法。
- 不做 SwiftUI View 大拆分。
- 不新增 UI 测试或完整本机 Xcode build 作为默认验证。

## 当前架构依据

- `ScheduleView` 已有 `selectedCategory`、`showingEditor` 和 `TaskEditorView(initialDueDate:initialCategory:)`。
- `SelectedCategorySummaryView` 当前展示分类、数量和清除入口，可扩展一个新增入口。
- `MacScheduleDetailView` 的 `.onChange(of: selectedCategory)` 已把快速新增面板的 `category` 和 `accentHex` 同步到筛选分类。
- `scripts/verify_project.sh` 已通过 grep marker 锁定分类筛选、摘要、预填和 Mac 分类 UI。

## 实现步骤

1. 更新 `ChronoFocus/Views/ScheduleView.swift`：
   - 给 `SelectedCategorySummaryView` 增加 `onAddTask` closure。
   - 调用处传入 `showingEditor = true`。
   - 在摘要中增加 `Button("新增此分类", systemImage: "plus.circle.fill", action: onAddTask)`。
   - 调整空态文案为“点击新增此分类”。
   - 保持按钮有文本 label、44pt 最小触控高度和清晰 accessibility。
2. 更新 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 在快速新增面板中，当 `selectedCategory != nil` 时显示一行带 `tag.fill` 的提示，说明会预填该分类。
   - 快照渲染路径也应有稳定静态文本，不使用会产生黄色占位的控件。
3. 更新 `scripts/verify_project.sh`：
   - 增加 `新增此分类`、`onAddTask`、Mac 预填提示等 marker 检查。
4. 更新文档：
   - README 功能描述补充“选中分类后可从摘要直接新增该分类待办”。
   - `md/flow/flow.md` / `md/flow/flowchart.md` / `md/test/test.md` 同步当前 UI 和 smoke marker。
   - `update_log.md` 追加 v0.17 记录。
