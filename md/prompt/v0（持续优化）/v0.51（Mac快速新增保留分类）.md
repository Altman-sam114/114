# v0.51（Mac 快速新增保留分类）

## 版本号

v0.51

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.50，因此本轮自动递增为 v0.51。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.50 已增强 CI artifact 额外文件拒绝，并通过最新 `origin/main` 云端 artifact 验收。当前 macOS 日程详情的快速新增面板支持手写分类、分类预设和从筛选摘要预填分类；但新增成功后，如果当前没有选中分类筛选，表单会把分类重置为“工作”。用户连续录入同一类任务时需要重复选择分类。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. macOS 日程详情快速新增成功后保留本次创建任务的分类，便于连续录入同类待办。
2. 如果当前有选中分类筛选，仍优先保留 `selectedCategory`。
3. 保留分类时使用 `store.addTask(...)` 返回的规范化任务分类和颜色，空白分类清洗后也能回填正确状态。
4. 在 `scripts/verify_project.sh` 中新增 Mac 快速新增连续录入源码契约。
5. README、测试规范、核心流程文档和更新日志同步本轮 Mac 分类录入体验变化。

- 不修改 iOS UI、通知、Live Activity、StoreKit、EventKit 或 Xcode 工程结构。

## 当前架构依据

- `MacScheduleDetailView.addTask()` 位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift`，通过 `store.addTask(...)` 创建任务，并在成功后清空标题、重置轮次和分类表单。
- `FocusStore.addTask(...)` 返回 `FocusTask?`，可用于读取清洗后的 `task.category` 与 `task.accentHex`。
- `scripts/verify_project.sh` 已在 Mac 快速新增附近锁定 `prepareQuickAdd(_:)` 的预填和聚焦行为，可在相邻区域新增 add 后保留分类契约。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`；若 `git pull` 审批超时但 `git fetch origin` 后 `HEAD == origin/main`，记录该事实并继续。
2. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 在 `addTask()` 中保存提交前的 `category` 和 `accentHex` 作为 fallback。
   - 若 `store.addTask(...)` 返回 `task`，同步提醒后设置 `category = selectedCategory ?? task.category`。
   - 同步设置 `accentHex = TaskCategoryPreset.matching(category)?.accentHex ?? task.accentHex`。
   - 若返回 nil，保留 `selectedCategory ?? submittedCategory` 和对应 fallback 颜色。
   - 继续清空 `taskTitle` 并将 `estimatedRounds` 重置为 2。
3. 修改 `scripts/verify_project.sh`：
   - 新增 `addTask()` 源码切片断言，检查 `category = selectedCategory ?? task.category`。
   - 检查 `TaskCategoryPreset.matching(category)?.accentHex ?? task.accentHex`。
   - 失败文案明确为 Mac quick add must retain submitted category after add。
4. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 只检查 diff 和 status，不运行本地测试命令。
6. 提交 `v0.51: 增强 Mac 快速新增分类保留` 并 push 到 `origin/main`。
7. Agent C 下载最新 run artifact 并按下方标准验收。
