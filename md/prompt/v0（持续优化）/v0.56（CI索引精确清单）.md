# v0.56 CI 索引精确清单

## 版本号

v0.56

## 版本分配依据

## 背景

`.github/workflows/ci-results.yml` 生成的 `ci-artifact-index.json` 使用固定路径清单，但 `scripts/validate_ci_artifact.rb` 此前只要求 entry 数量不少于预期并覆盖 required paths。这样如果 index 多声明未预期 entry，而下载目录没有对应额外文件，可能不会被 `unexpected local artifacts` 覆盖。

## 目标

- `scripts/validate_ci_artifact.rb` 增加 artifact index 路径集合精确复判。
- index 中的 entry path 必须与 `EXPECTED_INDEX_ENTRIES` 完全一致，不允许多声明或少声明。
- `scripts/verify_project.sh` 增加 `unexpected_index_entry_fixture` 负向 fixture，确认多声明 optional missing entry 会被拒绝。
- README、测试规范、核心流程文档、流程图和更新记录同步 CI index 精确清单能力。

## 非目标

- 不修改 GitHub Actions 上传的 artifact 路径列表。
- 不 bump `CI_PROCESS_VERSION`。
- 不改变 Mac/iOS build、快照生成或业务代码。
- 不扩大到 snapshot `generatedAt` 复判、failure summary clean-state 复判等后续候选项。

## 当前架构依据

- `ci-results.yml` 维护固定 `artifact_index_paths`。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载云端结果包后的结构化复判脚本。
- `scripts/verify_project.sh` 通过小型成功 fixture 和负向 fixture 锁定 validator 放行/拒绝路径。

## 实现步骤

1. 在 `scripts/validate_ci_artifact.rb` 中新增 `index unexpected entries` 检查：
   - 取 `index["entries"].map { |entry| entry["path"] }`。
   - 排序后与 `EXPECTED_INDEX_ENTRIES.keys` 精确相等。
2. 在 `scripts/verify_project.sh` 中：
   - 增加对 `index unexpected entries` 和 `FAIL index unexpected entries` 的源码 marker 检查。
   - 增加 `unexpected_index_entry_fixture`，复制成功 fixture 后向 index entries 添加 `ci-results/unexpected-index-only.log`，设为 `required: false`、`exists: false`、`kind: missing`，同步 totals。
   - 要求 validator 拒绝该 fixture 并输出 `FAIL index unexpected entries`。
3. 同步文档：
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## 验收标准

- validator 对正常云端 artifact 输出 `PASS index unexpected entries`。
- `verify_project.sh` 云端执行时负向 fixture 能锁定 `FAIL index unexpected entries`。
- GitHub Actions 最新 `main` run 成功。
- Agent C 下载的 artifact 中 manifest、artifact index、run context、JUnit、failure summary、主日志、Mac/iOS build 和项目专属验证产物均通过复判。
