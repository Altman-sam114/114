# v0.20（计时页分类筛选摘要）

## 版本号

v0.20

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.19 已增强 CI 结果包负向 fixture，并通过最新 `origin/main` 云端 artifact 验收。本轮回到 UI 和分类体验，递增为 v0.20，补齐 iOS 计时页选中分类后的筛选摘要。

## 背景

iOS 日程页在选中分类后已经显示分类摘要、当前数量、快捷新增和清除入口；但 iOS 计时页“当前日程”面板只有分类 chip 和右上角 `x/y 项待办` 计数。用户选中某个分类且该分类有可启动待办时，清除筛选入口只存在于横向 chip 的“全部”按钮里，不如日程页直接。空分类时已有 `TimerTaskCategoryEmptyView` 清除入口。本轮给计时页补一个一致的选中分类摘要，使分类上下文和清除动作始终可见。

## 目标

- iOS `TimerView.taskPicker` 在 `selectedTaskCategory != nil` 时显示选中分类摘要。
- 即使当前可启动待办变为 0 且 `selectedTaskCategory` 仍有旧值，也必须显示摘要和清除入口。
- 摘要显示分类图标、分类名称、当前可启动待办数量和“清除”按钮。
- 点击“清除”将 `selectedTaskCategory = nil`。
- 摘要使用 `TaskCategoryPreset.matching` 的图标和颜色；非预设分类 fallback 到 `tag.fill` 和默认 cyan。
- 保持 44pt iOS 最小触控高度和清晰 accessibility。
- `TimerTaskCategoryEmptyView` 继续保留；空分类时可以同时显示摘要和空态，清除动作一致。
- `scripts/verify_project.sh` 增加本轮 UI marker。
- README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md` 同步当前行为。

## 非目标

- 不修改 `FocusStore`、`TimerEngine`、任务排序、计划生成或持久化逻辑。
- 不改变计时页任务选择规则；计时运行中仍不能切换任务。
- 不新增任务创建入口或跨页导航。
- 不改 macOS UI。
- 不做大规模 View 拆分。

## 当前架构依据

- `TimerView` 已有 `@State private var selectedTaskCategory: String?`。
- `filteredUpcomingTasks` 和 `taskCount(in:)` 已基于 `selectedTaskCategory` 和 `store.upcomingTasks()` 计算当前可启动待办。
- `TimerTaskCategoryFilterBar` 已复用 `TaskCategoryPreset.prioritizedFilterOptions`，分类元数据不需要新增模型字段。
- `TimerTaskCategoryEmptyView` 已提供空分类清除入口，可复用相同清除 closure。

## 实现步骤

1. 更新 `ChronoFocus/Views/TimerView.swift`：
   - 在 `TimerTaskCategoryFilterBar` 下方，当 `selectedTaskCategory` 非空时插入 `TimerSelectedTaskCategorySummaryView`。
   - 摘要逻辑不要放在 `upcomingTasks.isEmpty` 的不可达分支里；有当前待办时显示筛选条，选中分类摘要独立显示，随后再进入空列表或任务列表分支。
   - 新增 `private struct TimerSelectedTaskCategorySummaryView: View`。
   - 传入 `category`、`count` 和 `onClear`。
   - 视图展示 `Label(category, systemImage:)`、`Text("\(count) 项可启动")` 和 `Button("清除", systemImage: "xmark.circle.fill", action: onClear)`。
   - 按现有 `AppTheme.panel`、8px 圆角和 tint stroke 风格实现，按钮 `frame(minHeight: 44)`。
2. 更新 `scripts/verify_project.sh`：
   - 增加 `TimerSelectedTaskCategorySummaryView`、`项可启动`、`当前筛选` 等 marker。
   - 增加源代码顺序检查，确认摘要插入点早于 `upcomingTasks.isEmpty` 空列表分支，并且 `onClear` 使用 `clearTaskCategoryFilter`。
3. 更新文档：
   - README 功能描述补充计时页选中分类时会显示筛选摘要和清除入口。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 同步计时页分类摘要。
   - `md/test/test.md` smoke 基线补充计时页分类摘要 marker。
   - `update_log.md` 追加 v0.20 记录。

## 验收标准

- iOS 计时页选中分类后，能看到该分类摘要、当前可启动待办数和“清除”按钮。
- 当前待办变为 0 时，如果仍有选中分类，也能看到摘要和清除按钮。
- 点击“清除”会清空 `selectedTaskCategory` 并恢复全部可启动待办。
- 空分类时摘要和空态清除入口都能使用，不产生第二套筛选状态。
- `scripts/verify_project.sh` 通过并覆盖本轮 marker。
- push 到 `origin/main` 后，Agent C 下载最新 v0.20 run artifact，用 `scripts/validate_ci_artifact.rb` 和手工核对共同确认结果包通过。

## 风险和禁止项

- 禁止在 View 中改变任务筛选规则以外的核心业务逻辑。
- 禁止影响 `TimerEngine` 开始/暂停/完成状态机。
- 禁止引入第三方依赖。
- 禁止读取、提交或删除无关未跟踪文件。
