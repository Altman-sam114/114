# v0.75（待办行操作分类语义）

## 版本号

v0.75

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.74，本轮继续围绕“优化 UI，更加方便使用，优化分类”做小步增强，因此自动递增为 v0.75。

## 背景

iOS/macOS 日程待办行已经显示分类 badge，但完成、启用、编辑和删除等操作的可访问语义仍主要只读出任务名。同名任务出现在不同分类时，VoiceOver 和 Voice Control 用户仍需要额外确认行内分类。计划项开始按钮已经读出分类，本轮将同样的分类上下文补到待办行操作语义中。

## 目标

- iOS `ScheduleTaskCell` 的完成/标记未完成、启用/停用、编辑操作读出任务名和分类。
- iOS 待办行 swipe 编辑/删除操作补齐任务名和分类可访问语义与 Voice Control input labels。
- macOS `MacTaskListPanelView` 的完成/标记未完成、启用/停用、删除操作读出任务名和分类。
- macOS 快照静态控件同步任务名和分类语义，避免真实按钮与快照安全控件不一致。
- `scripts/verify_project.sh` 升级现有 `Schedule task action accessibility contracts verified.` 源码契约，覆盖分类语义。
- README、测试规范、核心流程文档、流程图和更新日志同步待办行操作分类语义。

## 非目标

- 不修改 `FocusTask`、`FocusStore`、`TimerEngine`、计划生成或持久化逻辑。
- 不改可见按钮文案和布局，不重做任务行 UI。
- 不新增 validator marker；复用现有 `verify_project schedule task action accessibility contracts`。
- 不运行本地测试、构建、lint、`git diff --check`、`bash scripts/verify_project.sh` 或本地 fixture validator。

## 当前架构依据

- iOS 日程任务行位于 `ChronoFocus/Views/ScheduleView.swift` 的 `ScheduleTaskCell`，iOS swipe 编辑/删除位于 `taskList` 的 `ForEach(visibleTasks)`。
- macOS 日程任务行操作位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的 `MacTaskListPanelView`。
- macOS 快照模式使用 `MacStaticScheduleActionChipView` 和 `MacStaticTaskEnablePillView` 代替真实按钮。
- `scripts/verify_project.sh` 已有 `Schedule task action accessibility contracts verified.` marker，`scripts/validate_ci_artifact.rb` 已复判该 marker。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `ChronoFocus/Views/ScheduleView.swift`：
   - 在 `ScheduleTaskCell` 增加完成/启用/编辑操作的 accessibility label 和 Voice Control label helper。
   - 语义中包含 `\(task.title)` 和 `\(task.category)分类`。
   - 在 swipe 编辑/删除按钮上补齐 `.accessibilityLabel(...)` 和 `.accessibilityInputLabels(...)`。

2. 更新 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - macOS 完成/启用/删除真实控件的 accessibility label 和 Voice Control input labels 加入分类。
   - 快照静态控件的 title 或 accessibility label 加入分类。
   - `MacStaticTaskEnablePillView` 增加可选 `taskCategory`，有分类时读出 `\(taskTitle)待办已启用，\(taskCategory)分类` 或停用语义。

3. 更新 `scripts/verify_project.sh`：
   - iOS `ScheduleTaskCell` 契约改为检查任务名 + 分类语义。
   - 增加 iOS swipe 编辑/删除分类语义检查。
   - macOS 任务行真实按钮和静态控件契约改为检查任务名 + 分类语义。
   - 保留 `Schedule task action accessibility contracts verified.` marker。

4. 同步文档：
   - README 描述 iOS/macOS 待办行操作会读出分类。
   - `md/test/test.md` 更新验证范围。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实分类 UI 语义。
   - `update_log.md` 追加 v0.75 记录，并将 v0.74 云端结果补成已通过。
