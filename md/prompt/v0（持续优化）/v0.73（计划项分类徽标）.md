# v0.73（计划项分类徽标）

## 版本号

v0.73

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.72，本轮继续围绕“优化 UI，更加方便使用，优化分类”做小步增强，因此自动递增为 v0.73。

## 背景

iOS 和 macOS 的番茄钟计划项已经在开始按钮可访问标签中读出分类，但可见 UI 仍把分类混在 `时间段 · 第 N 轮 · 分类` 的 caption 文本里。分类是计划筛选、识别和启动前确认的重要上下文，应该像任务行一样以独立、可识别的分类 badge 呈现。

## 目标

- iOS `PomodoroPlanRow` 将计划项分类从纯文本升级为可见分类 badge。
- macOS `MacPlanPanelView` 的计划项同步显示分类 badge。
- badge 使用 `TaskCategoryPreset.matching(item.category)` 获取预设图标和颜色，fallback 到 `item.accentHex` 和 `tag.fill`。
- badge 补齐分类可访问标签和 Voice Control input labels。
- `scripts/verify_project.sh` 增加源码契约 marker 和缺失 marker 负向 fixture。
- `scripts/validate_ci_artifact.rb` 增加云端 artifact 中 verify marker 的复判。
- 同步 README、测试规范、核心流程文档、流程图和更新日志。

## 非目标

- 不修改 `PomodoroPlanItem`、`FocusTask`、`FocusStore`、`TimerEngine` 或计划生成逻辑。
- 不修改计划项开始行为、按钮语义或持久化格式。
- 不重做计划面板布局，不新增第三方依赖。
- 不运行本地测试、构建、lint 或本地 fixture validator。

## 当前架构依据

- iOS 计划项行位于 `ChronoFocus/Views/ScheduleView.swift` 的 `PomodoroPlanRow`。
- macOS 计划项行位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的 `MacPlanPanelView`。
- 常用分类元数据来自共享 `TaskCategoryPreset`；本轮只在展示层读取图标和颜色。
- `scripts/verify_project.sh` 负责源码契约和云端项目专属验证 marker。
- `scripts/validate_ci_artifact.rb` 负责 Agent C 下载 artifact 后复判 `verify_project.log` marker。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `ChronoFocus/Views/ScheduleView.swift`：
   - 在 `PomodoroPlanRow` 增加分类 preset、tint 和 symbol helper。
   - 将 metadata 拆为时间/轮次文本和分类 `Label` badge。
   - badge 显示 `item.category`，使用预设图标/颜色，fallback 到 `item.accentHex` / `tag.fill`。
   - badge 添加 `.accessibilityLabel("\(item.category)分类")` 和 `.accessibilityInputLabels([Text(item.category), Text("\(item.category)分类")])`。

2. 更新 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 在 Mac 计划项行加入同等分类 badge。
   - 维持计划行紧凑布局，避免挤压开始按钮。
   - 保留现有开始按钮 accessibility label 和 input labels。

3. 更新 `scripts/verify_project.sh`：
   - 新增源码契约检查，确认 iOS/macOS 计划项都包含 `TaskCategoryPreset.matching(item.category)`、分类 `Label` badge、fallback 颜色/图标、可访问标签和 Voice Control input labels。
   - 输出 marker：`Plan category badge contracts verified.`。
   - 小型成功 fixture 的 `verify_project.log` 增加该 marker。
   - 增加负向 fixture，移除该 marker 后要求 validator 输出 `FAIL verify_project plan category badge contracts`。

4. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project plan category badge contracts` 复判，要求 `verify_project.log` 包含本轮 marker。

5. 同步文档：
   - README 描述 iOS/macOS 计划项分类 badge。
   - `md/test/test.md` 更新验证范围、Agent C PASS 清单和 artifact marker 说明。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实核心验证流。
   - `update_log.md` 追加 v0.73 记录。
