# v0.66（Mac快速新增操作语义）

## 版本号

v0.66

## 版本分配依据

人工总目标要求持续优化 UI 易用性、分类体验和 CI 测试。v0.65 已完成 CI process version 固定复判并通过云端 artifact 验收；本轮继续按小版本递增到 v0.66，聚焦 macOS 日程详情页快速新增按钮的分类和预计轮次操作语义。

## 背景

macOS 日程详情页左侧“快速新增”表单已经支持分类预填、常用分类选择和预计轮次设置，但最终“新增待办”按钮本身只暴露通用标题。用户在选中分类或调整预计轮次后，辅助技术无法从提交动作确认当前将新增哪个分类、预计几轮。快照静态按钮也只保留短标题，缺少同等可访问覆盖语义。

## 目标

- macOS 快速新增真实按钮补齐当前分类和预计轮次可访问标签。
- macOS 快速新增真实按钮补齐 Voice Control input labels，支持“新增待办”“新增某分类分类待办”“新增某分类分类”。
- macOS 快照静态“新增待办”按钮保持短可见标题，同时通过可访问覆盖文本保留当前分类和预计轮次语义。
- 在 `scripts/verify_project.sh` 增加源码契约 marker，并让 `scripts/validate_ci_artifact.rb` 复判该 marker。
- 同步 README、测试规范、核心流程、流程图和更新记录。

## 非目标

- 不改 `addTask()`、`FocusStore.addTask`、分类清洗、提醒同步或计划生成行为。
- 不改可见按钮文案、表单布局或快照尺寸。
- 不修改 iOS 日程页；v0.64 已覆盖 iOS toolbar 新增入口分类语义。
- 不运行本地测试、build、lint、`git diff --check` 或脚本验证；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- macOS 日程详情页位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift`。
- 快速新增表单使用本地状态 `category`、`estimatedRounds` 和 `taskTitle`，提交仍通过 `addTask()` 进入 `FocusStore.addTask`。
- 快照模式使用 `MacStaticScheduleActionChipView` 替代真实按钮，该静态控件已支持 `accessibilityLabelText`。
- `scripts/verify_project.sh` 是项目专属验证入口；云端 workflow 会运行它并把 `verify_project.log` 纳入 artifact。

## 实现步骤

1. 同步最新 `main`，确认当前分支为 `main`，没有无关 tracked diff；不要读取或提交用户未跟踪文件。
2. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 增加快速新增分类名 helper，对空白分类给出稳定兜底。
   - 增加快速新增按钮可访问标签 helper，包含分类和预计轮次。
   - 增加快速新增按钮 Voice Control input labels helper。
   - 快照静态按钮传入 `accessibilityLabelText`，真实按钮添加 `.accessibilityLabel(...)` 和 `.accessibilityInputLabels(...)`。
3. 修改 `scripts/verify_project.sh`：
   - 增加 macOS 快速新增按钮操作语义源码契约。
   - 输出稳定 marker：`Mac quick add action accessibility contracts verified.`。
   - 在 validator 小型成功 fixture 中加入该 marker。
   - 增加缺失 marker 负向 fixture：`negative_mac_quick_add_action_marker_fixture`。
4. 修改 `scripts/validate_ci_artifact.rb`：
   - 复判 `verify_project.log` 中的新增 marker。
   - PASS 名称为 `verify_project mac quick add action accessibility contracts`。
5. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
