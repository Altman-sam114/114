# v0.44（计时页摘要清除 Voice Control）

## 版本号

v0.44

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.43，因此本轮自动递增为 v0.44。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.42 已让计时页分类空态清除按钮具备分类名可访问标签和 Voice Control input labels，但计时页选中分类摘要里的“清除”按钮仍只有通用文案。用户通过 VoiceOver 或 Voice Control 聚焦到按钮时，按钮本身没有明确说明要清除哪个分类筛选。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 在 `TimerSelectedTaskCategorySummaryView` 的“清除”按钮上增加 `.accessibilityLabel("清除\(category)分类筛选")`。
2. 为同一按钮增加 `.accessibilityInputLabels([Text("清除筛选"), Text("清除\(category)分类")])`。
3. 保留现有 72pt 最小宽度、44pt 最小高度和摘要容器可访问说明。
4. `scripts/verify_project.sh` 增加计时页摘要清除按钮静态契约。
5. README、测试规范、核心流程文档和更新日志同步本轮 UI 可访问范围。

## 非目标

- 不改变分类筛选状态、清除动作、空态触发条件、任务模型、持久化或计时状态机。
- 不修改日程页、macOS UI、CI workflow 或 artifact 文件列表。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `TimerSelectedTaskCategorySummaryView` 是 iOS `TimerView` 展示层，只调用外部传入的 `onClear`。
- 分类空态清除按钮已使用同类 `.accessibilityLabel` 和 `.accessibilityInputLabels`，摘要清除按钮应保持一致。
- `scripts/verify_project.sh` 已有 `timer_summary_source` slice，可在该 slice 中追加摘要清除按钮的 44pt、label 和 input labels 契约。
