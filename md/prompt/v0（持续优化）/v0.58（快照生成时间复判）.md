# v0.58 快照生成时间复判

## 版本号

v0.58

## 版本分配依据

人工总目标要求持续优化 UI 和 CI 测试。v0.57 已完成日程日期格可访问语义并通过云端验收；本轮继续增强 CI artifact 复判，锁定 Mac 快照 manifest 的生成时间字段，因此递增到 v0.58。

## 背景

`scripts/render_mac_snapshots.swift` 生成的 Mac 快照 manifest 已包含 `generatedAt` 字段，且使用 `ISO8601DateFormatter()` 写入。但 `scripts/validate_ci_artifact.rb` 当前只复判快照文件名、尺寸和 byteCount，没有检查 `generatedAt` 是否存在且为 ISO8601 时间。

## 目标

- `scripts/validate_ci_artifact.rb` 增加 `snapshot manifest generated at` 复判。
- `scripts/verify_project.sh` 增加无效 `generatedAt` 负向 fixture，要求 validator 输出 `FAIL snapshot manifest generated at`。
- README、测试规范、核心流程文档、流程图和更新记录同步快照 manifest 生成时间复判范围。

## 非目标

- 不修改 Mac 快照渲染内容或快照数量。
- 不修改 `.github/workflows/ci-results.yml` 上传路径。
- 不 bump `CI_PROCESS_VERSION`。
- 不改变 UI 或业务逻辑。

## 当前架构依据

- `scripts/render_mac_snapshots.swift` 的 `SnapshotManifest` 已有 `generatedAt: String`。
- `scripts/validate_ci_artifact.rb` 已有 `iso8601_timestamp?` helper，可复用。
- `scripts/verify_project.sh` 已有小型成功 fixture 和快照 manifest byteCount 负向 fixture，可在相邻位置追加 `generatedAt` 负向 fixture。

## 实现步骤

1. 在 `scripts/validate_ci_artifact.rb` 中新增：
   - `check(checks, "snapshot manifest generated at") { iso8601_timestamp?(snapshot_manifest["generatedAt"]) }`
2. 在 `scripts/verify_project.sh` 中：
   - 增加源码 marker：`snapshot manifest generated at` 和 `FAIL snapshot manifest generated at`。
   - 新增 `invalid_snapshot_generated_at_fixture`，复制成功 fixture 后把 `project-reports/mac-snapshots/manifest.json` 的 `generatedAt` 改为无效字符串。
   - 要求 validator 拒绝该 fixture 并输出 `FAIL snapshot manifest generated at`。
3. 同步文档：
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## 验收标准

- validator 对正常云端 artifact 输出 `PASS snapshot manifest generated at`。
- `verify_project.sh` 云端执行时负向 fixture 能锁定 `FAIL snapshot manifest generated at`。
- GitHub Actions 最新 `main` run 成功。
- Agent C 下载的最新 artifact 经 validator 全 PASS，manifest、run context、JUnit、failure summary、verify log、Mac/iOS build 和快照产物均与最新 `origin/main` 一致。
