# v0.18（CI运行上下文复判）

## 版本号

v0.18

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.17 已补强分类筛选快捷新增，并通过最新 `origin/main` 云端 artifact 验收。本轮回到 CI 测试可靠性，递增为 v0.18，增强 Agent C 下载结果包后的运行上下文复判。

## 背景

`.github/workflows/ci-results.yml` 会生成 `ci-results/ci-run-context.txt`，其中记录 artifact 名称、分支、提交 SHA、run id 和 run attempt。`scripts/validate_ci_artifact.rb` 当前已经核对 manifest、artifact index、JUnit、failure summary、Mac/iOS build 日志和快照 manifest，但没有解析 `ci-run-context.txt`，也没有确认 artifact 名称是否和当前分支、短 SHA、run id、attempt 保持一致。本轮把这一项纳入结构化 validator，减少 Agent C 误收旧包或错包的风险。

## 目标

- `scripts/validate_ci_artifact.rb` 解析 `ci-run-context.txt`。
- validator 核对 run context 的 `branch`、`commitSha`、`runId`、`runAttempt` 与命令参数一致。
- validator 核对 `artifactName` 符合 `chronofocus-ci-<version>-<branchSlug>-<shortSha>-run<runId>-attempt<attempt>`。
- validator 对 run context 缺失字段、错误字段和 artifact 名称不一致给出 FAIL。
- `scripts/verify_project.sh` 的小型 artifact fixture 同步覆盖新检查。
- README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 同步当前 CI 复判范围。

## 非目标

- 不改变 GitHub Actions workflow 产物格式，除非发现必须补齐的字段。
- 不改变 artifact 上传路径、保留期、CI job 名称或 Xcode build 命令。
- 不新增第三方依赖。
- 不修改 App UI、模型、计时、StoreKit、EventKit 或持久化逻辑。
- 不下载历史 artifact 或大体积数据。

## 当前架构依据

- workflow 的 `Prepare result metadata` 步骤已写入 `ci-results/ci-run-context.txt`。
- workflow artifact 名称由 `CI_PROCESS_VERSION`、分支 slug、短 SHA、run id 和 run attempt 组成。
- `scripts/validate_ci_artifact.rb` 已支持命令参数 `--commit`、`--run-id`、`--attempt`、`--branch`，并可解析下载目录或单一 artifact 子目录。
- `scripts/verify_project.sh` 已构造小型 artifact fixture，用来覆盖 validator 的 manifest/index/JUnit/summary/log/snapshot 检查。

## 实现步骤

1. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `context_path = File.join(artifact_dir, "ci-run-context.txt")`。
   - 增加解析 `key=value` 文本的 helper，忽略空行，必要时对重复 key 以后写为准。
   - 根据 manifest `version` 和命令参数计算 expected artifact name：
     - `branchSlug = branch.gsub("/", "-")`
     - `shortSha = commit[0, 7]`
     - `chronofocus-ci-#{manifest["version"]}-#{branchSlug}-#{shortSha}-run#{run_id}-attempt#{attempt}`
   - 新增 checks：`run context fields`、`run context identity`、`run context artifact name`。
   - 保持输出风格为 `PASS/FAIL <name>`。
2. 更新 `scripts/verify_project.sh` fixture：
   - fixture 的 `ci-run-context.txt` 必须包含 `artifactName=chronofocus-ci-v0.10-main-fixture-run12345-attempt1` 这类与 fixture commit/run/attempt 一致的名称。
   - 为了名称稳定，fixture commit 可继续用 `fixture-sha`，expected short SHA 为 `fixture`。
   - 增加 marker 检查：`run context identity`、`run context artifact name`、`ci-run-context.txt`。
3. 更新文档：
   - README 的 artifact validator 描述补充 run context 和 artifactName 核对。
   - `md/test/test.md` 的结果包复判说明和 smoke 当前基线补充 run context。
   - `md/flow/flow.md` / `md/flow/flowchart.md` 同步 Agent C 复判范围。
   - `update_log.md` 追加 v0.18 记录。
4. 不改 `.github/workflows/ci-results.yml`，除非实现时发现现有字段无法满足 validator。

## 文档更新要求

- README 面向用户，说明下载 artifact 后会同时核对 run context 和 artifact 名称。
- `md/test/test.md` 更新 Agent C 结果包复判范围和 smoke 基线。
- `md/flow/flow.md` 和 `md/flow/flowchart.md` 只描述当前真实 CI 验收流。
- `update_log.md` 记录版本、日期、核心变更、关键文件、验证结果和遗留事项。

## 验收标准

- validator 对真实 v0.17 artifact 运行全 PASS，并新增 run context 相关 PASS 行。
- `scripts/verify_project.sh` 中的小型 fixture 能覆盖新增 validator 检查，并输出 `Project structure verified.`。
- push 到 `origin/main` 后，Agent C 使用最新 v0.18 run artifact 复判通过，manifest、artifact index、run context、JUnit、failure summary、日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest 均一致。

## 风险和禁止项

- 禁止改变 workflow artifact contract 后不更新 validator 和文档。
- 禁止把旧 run、旧 artifact 或本地输出冒充最新云端结果。
- 禁止下载历史 artifact 或无关大文件。
- 禁止触碰 App 业务逻辑、UI 或平台服务。
- 禁止读取、提交或删除无关未跟踪文件。

## main 同步、提交和 push 要求

```bash
git switch main

git status --short --branch
```

提交前确认只包含本轮相关文件，提交信息使用：

```text
v0.18: 收紧 CI 运行上下文复判
```

```bash
git push origin main
```

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 下载 artifact 前必须先完成 `gh auth login`，并确认 GitHub 账号为项目授权账号。
- Agent C 下载最新 `origin/main` commit 对应 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 运行 `ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> --commit <sha> --run-id <run_id> --attempt <attempt>`。
- 同时继续人工核对 manifest、artifact index、run context、JUnit、failure summary、主日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest。
