# v0.46（Mac 日程摘要按钮点击区）

## 版本号

v0.46

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.45，因此本轮自动递增为 v0.46。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.45 已为 iOS 与 macOS 日程筛选摘要的“新增此分类”和“清除”按钮补齐分类名可访问标签和 Voice Control input labels。继续检查后发现 macOS 日程摘要真实按钮和快照静态按钮仍只有 `minHeight: 30`，且清除按钮没有稳定最小宽度；在详细窗口中会显得偏窄，不利于重复操作和快照回归判断。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 提升 `MacSelectedCategorySummaryView` 真实新增/清除按钮的稳定点击区。
2. 让新增按钮和清除按钮都有明确 `minWidth` 与 `minHeight`，避免随文案和系统渲染过度收缩。
3. 同步 `MacSummaryStaticActionView` 的快照静态按钮尺寸，确保快照与真实布局的按钮占位一致。
4. `scripts/verify_project.sh` 增加 Mac 日程摘要按钮点击区静态契约。
5. README、测试规范、核心流程文档和更新日志同步本轮 UI/CI 覆盖范围。

## 非目标

- 不改变新增/清除动作、筛选状态、任务模型、持久化、计划生成或计时状态机。
- 不修改 iOS 日程摘要按钮、CI workflow、artifact 文件列表、validator 结构或快照脚本。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `MacSelectedCategorySummaryView` 是 macOS 详情窗口展示层，只调用外部传入的 `onAddTask` / `onClear`。
- `MacSummaryStaticActionView` 只在 `macSnapshotRendering` 路径中替代真实按钮，避免快照中原生按钮缺失占位；它应保持与真实按钮相近的稳定尺寸。
- `scripts/verify_project.sh` 已有 `mac_schedule_summary_source` slice，可继续在该 slice 中锁定按钮尺寸和分类语义。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认当前 `HEAD == origin/main` 且只有用户未跟踪文件未触碰。
2. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 真实“新增此分类”按钮增加稳定最小宽度，例如 `.frame(minWidth: 104, minHeight: 36)`。
   - 真实“清除”按钮增加稳定最小宽度，例如 `.frame(minWidth: 72, minHeight: 36)`。
   - `MacSummaryStaticActionView` 使用同等最小宽度与高度：新增态 104，清除态 72，高度 36。
   - 保留 v0.45 的分类名可访问标签和 Voice Control input labels。
3. 修改 `scripts/verify_project.sh`：
   - 在 `mac_schedule_summary_add_button` 检查新增按钮 `.frame(minWidth: 104, minHeight: 36)`。
   - 在 `mac_schedule_summary_clear_button` 检查清除按钮 `.frame(minWidth: 72, minHeight: 36)`。
   - 增加 `MacSummaryStaticActionView` slice，锁定快照静态按钮使用相同 `minWidth` 和 `minHeight`。
4. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 只检查 diff 和 status，不运行本地测试命令。
6. 提交 `v0.46: 增强 Mac 日程摘要按钮点击区` 并 push 到 `origin/main`。
7. Agent C 下载最新 run artifact 并按下方标准验收。
