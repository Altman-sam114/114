# v0.83（待办编辑取消分类语义 artifact 复判）

## 版本号

v0.83

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；v0.82 已完成 iOS 待办新增/编辑保存按钮分类语义 artifact 复判，并通过最新 `origin/main` 云端 artifact 验收。v0.83 继续在同一 v0 持续优化阶段递增。

## 背景

iOS 日程页新增和编辑待办共用 `TaskEditorView`。v0.82 已让 toolbar 的“保存”按钮读出待办、分类和计划方式，但相邻的“取消”按钮仍只有 `Button("取消")`。用户在编辑多分类待办时，取消动作缺少当前任务和分类确认语义，Voice Control 也缺少任务名或分类名口令。

## 目标

- iOS `TaskEditorView` 的取消按钮补齐可访问标签，读出取消新增/取消编辑动作、待办标题和当前分类。
- 取消按钮 Voice Control input labels 支持取消、取消新增/取消编辑、任务标题、分类名和分类取消等自然口令。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Task editor cancel category accessibility contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project task editor cancel category accessibility contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- README、测试规范、核心流程文档和更新日志同步 v0.83 的云端 artifact 复判范围。

## 非目标

- 不改 `FocusTask`、`FocusStore`、`TimerEngine`、保存业务逻辑、提醒同步、计划生成或统计逻辑。
- 不改变取消按钮的现有行为；仍只 dismiss sheet。
- 不新增 macOS 待办编辑表单。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；人工明确要求本轮只去云端验证。

## 当前架构依据

- `TaskEditorView` 位于 `ChronoFocus/Views/ScheduleView.swift`，新增 sheet 和编辑 sheet 都复用它。
- v0.82 已新增 `taskTitleDisplayName`、`saveActionText`、`savePlanDescription`、`saveButtonAccessibilityLabel` 和 `saveButtonInputLabels`，取消按钮可复用同一任务名和分类 fallback。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 在 `ChronoFocus/Views/ScheduleView.swift` 的 `TaskEditorView` 中增加取消按钮 helper：
   - `cancelActionText`：新增时读“取消新增”，编辑时读“取消编辑”。
   - `cancelButtonAccessibilityLabel`：包含动作、任务标题和当前分类。
   - `cancelButtonInputLabels`：包含取消、取消新增/取消编辑、任务标题、分类名、分类取消口令。
2. 将 toolbar cancellation `Button("取消")` 加上 `.accessibilityLabel(...)` 和 `.accessibilityInputLabels(...)`，保持 dismiss 行为不变。
3. 在 `scripts/verify_project.sh` 的 `TaskEditorView` 分类输入上下文检查区加入取消按钮契约，并输出 `Task editor cancel category accessibility contracts verified.`。
4. 在 `scripts/validate_ci_artifact.rb` 增加 `verify_project task editor cancel category accessibility contracts` 检查。
5. 更新 `scripts/verify_project.sh` 的 validator 自检 grep、小型成功 fixture、负向 fixture 名称锁定和 marker 缺失负向 fixture。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
