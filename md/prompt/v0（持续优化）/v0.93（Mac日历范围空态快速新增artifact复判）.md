# v0.93（Mac 日历范围空态快速新增 artifact 复判）

## 背景

总目标是持续优化 ChronoFocus 的界面易用性、分类工作流和 CI 云端验收。v0.92 已补齐 macOS 日程分类空态操作，但 Mac 日历面板在当前日、周或月范围没有待办时仍只显示“当前范围暂无待办。”，用户必须回到左侧手动调整截止日期并聚焦标题。空态应提供直接为当前选中日期准备快速新增的入口。

## 目标

- `MacCalendarPanelView` 当前范围没有待办时显示紧凑的可操作空态。
- 空态显示当前选中日期，并提供“新增到此日期”动作。
- 点击动作后，左侧快速新增表单保留原截止时间的时分，只把日期替换为当前选中日期，并聚焦任务名称输入框。
- 快照渲染使用现有静态 action view，避免原生按钮占位。
- 真实按钮提供稳定点击区、日期相关 `accessibilityLabel`、操作提示和 Voice Control input labels。
- `scripts/verify_project.sh` 新增独立 marker：`Mac calendar range empty state quick add contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增复判项：`PASS verify_project mac calendar range empty state quick add contracts`。
- 增加 marker 缺失负向 fixture，防止旧 artifact 或缺少该契约的结果包被放行。

## 非目标

- 不改 iOS 日程页、分类筛选空态或计时页。
- 不改 `FocusStore`、`TimerEngine`、共享模型、持久化或计划生成逻辑。
- 不改变日/周/月范围筛选规则。
- 不引入第三方依赖，不拆分或重构整个 `MacScheduleDetailView`。

## 当前架构依据

- `MacCalendarPanelView` 自己维护 `selectedDate` 和 `calendarMode`，适合把选中日期通过闭包传回父视图。
- `MacScheduleDetailView` 已拥有快速新增表单的 `dueDate` 与 `@FocusState isTaskTitleFocused`，应由父视图完成日期预填和聚焦。
- `FocusStore` 仍是任务保存入口，本轮只准备表单状态，不新增第二套数据写入路径。
- Mac 快照路径已有 `MacStaticScheduleActionChipView`，可复用为稳定静态动作。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认 `HEAD == origin/main`，并忽略但不提交人工未跟踪文件。
2. 为 `MacCalendarPanelView` 增加 `(Date) -> Void` 的快速新增闭包，由父视图传入。
3. 在父视图新增日期预填方法：提取当前 `dueDate` 的时分，与日历面板传回的年月日组合，写回 `dueDate` 并聚焦标题。
4. 新增 `MacCalendarRangeEmptyStateView`，展示当前选中日期、说明和“新增到此日期”动作；快照模式使用静态 action view。
5. 为真实按钮加入稳定尺寸、日期可访问标签、操作提示和 Voice Control input labels。
6. 在 `scripts/verify_project.sh` 中绑定组件、父视图接线、日期组合、聚焦、空态分支、快照动作和可访问语义，并输出新 marker。
7. 在 `scripts/validate_ci_artifact.rb` 中复判新 marker。
8. 在成功 fixture 中加入新 marker，并新增删除 marker 的负向 fixture，预期输出 `FAIL verify_project mac calendar range empty state quick add contracts`。
9. 同步更新 `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

- 云端 `verify_project.log` 包含 `Mac calendar range empty state quick add contracts verified.`。
- Agent C validator 输出 `PASS verify_project mac calendar range empty state quick add contracts`。
- 最新 artifact 的 branch、commitSha、run id、attempt 和 artifact 名称与 `origin/main` 一致。
- 最新 artifact 同时通过 `PASS manifest overall outcome`、`PASS mac build succeeded` 和 `PASS ios build succeeded`。

## 风险和禁止项

- 周/月模式下动作必须明确使用当前选中日期，不能模糊地新增到整个范围。
- 日期组合必须保留现有时分，不能把截止时间静默重置为午夜。
- 禁止直接保存任务；动作只准备左侧表单，仍由现有 `addTask()` 进入 `FocusStore`。
- 禁止跳过 Agent C 对最新 artifact 的复判。
- 禁止把本地输出、旧 run 或旧 artifact 当作通过结论。
- 禁止触碰或提交 `md/1.md` 与 `md/通用云端迭代升级提示词.md`。
