# v0.89（统计分类投入占比可读性 artifact 复判）

## 版本号

v0.89

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化分类 UI 易用性和 CI 测试。远端最新 v0.88 已完成统计分类投入元信息可读性，并通过最新 `origin/main` 云端 artifact 验收。v0.89 继续在同一 v0 持续优化阶段递增。

## 背景

iOS/macOS 统计页“分类投入”已经具备占比、专注次数、排行位置、排序依据、空态和元信息可读性契约。v0.88 将次数/排行元信息从过小的 `.caption2` 提升为 `.caption`。当前右侧占比 pill 仍以内联 `.caption.bold()` 呈现，作为用户扫读分类投入的关键数值，可以进一步提升为更醒目的动态字体，并由独立 CI artifact marker 防止回退。

## 目标

- iOS 和 macOS 统计页“分类投入”右侧百分比 pill 使用独立 helper 管理字体。
- 百分比 pill 使用 `.subheadline.bold()` 动态字体，提升分类占比扫读可读性。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics category share percent readability contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics category share percent readability contracts` 复判。
- README、测试规范、核心流程文档、流程图和更新日志同步 v0.89 的云端 artifact 复判范围。

## 非目标

- 不修改 `FocusStore.categoryBreakdown()`、`CategoryFocus`、统计聚合、排序、Pro gating 或持久化模型。
- 不改变分类投入百分比计算、进度条、空态文案、可访问标签或 Voice Control input labels。
- 不引入第三方依赖，不重构统计页结构，不扩大到其他统计模块。
- 不运行本地测试、build、lint 或 `git diff --check`；人工明确要求“不得在本地测试，都去云端”。

## 当前架构依据

- 统计页展示属于平台 SwiftUI View 层：`ChronoFocus/Views/AnalyticsView.swift` 和 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift`。
- 统计聚合属于共享 `FocusStore`，本轮不得触碰。
- 项目专属结构验证由 `scripts/verify_project.sh` 输出 marker，Agent C 用 `scripts/validate_ci_artifact.rb` 对下载的云端 artifact 复判。
- v0.84-v0.88 已在同一统计分类投入区域建立一组独立 marker，本轮沿用同样模式新增占比可读性 marker。

## 实现步骤

1. 同步最新 main：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认只有用户未跟踪文件 `md/1.md`、`md/通用云端迭代升级提示词.md` 可被忽略，不得暂存。
2. 修改 iOS 统计页：
   - 在 `AnalyticsView` 的分类投入百分比 `Text("\(categorySharePercent(for: item.seconds))%")` 上，将内联 `.font(.caption.weight(.bold))` 改为 `.font(categorySharePercentFont())`。
   - 新增 `private func categorySharePercentFont() -> Font { .subheadline.bold() }`。
3. 修改 macOS 统计页：
   - 在 `MacCategoryChartPanelView` 的百分比 `Text` 上，将内联 `.font(.caption.bold())` 改为 `.font(categorySharePercentFont())`。
   - 新增同名 helper，返回 `.subheadline.bold()`。
4. 修改 `scripts/verify_project.sh`：
   - 在分类投入元信息可读性契约后新增占比可读性源码契约。
   - 检查 iOS/macOS 源码都包含 `private func categorySharePercentFont() -> Font`、`.subheadline.bold()` 和 `.font(categorySharePercentFont())`。
   - 拒绝旧内联 `.font(.caption.weight(.bold))` 和 `.font(.caption.bold())` 回退。
   - 输出 `Analytics category share percent readability contracts verified.`。
   - 成功 fixture 的 `verify_project.log` 加入该 marker。
   - validator 自检 grep 加入新 marker。
   - self-lock grep 加入 `negative_analytics_category_share_percent_readability_marker_fixture` 和 `FAIL verify_project analytics category share percent readability contracts`。
   - 新增负向 fixture：删除成功 fixture 中该 marker 后，validator 必须失败。
5. 修改 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project analytics category share percent readability contracts` 检查。
6. 更新文档：
   - README 和 `md/test/test.md` 补充占比可读性 marker、PASS 和负向 fixture。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 补充统计分类投入占比可读性与 artifact 复判。
   - `update_log.md` 追加 v0.89 记录，说明本轮不跑本地测试、以云端 artifact 为准。
