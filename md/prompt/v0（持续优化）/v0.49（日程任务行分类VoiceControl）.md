# v0.49（日程任务行分类 Voice Control）

## 版本号

v0.49

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.48，因此本轮自动递增为 v0.49。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.48 已增强分类摘要 marker 的云端负向 fixture。当前 iOS 计时页任务分类 badge 已提供 `accessibilityInputLabels([Text(task.category), Text("\(task.category)分类")])`，但 iOS 日程页任务行分类 badge 只有可访问标签，没有同等 Voice Control 输入标签；用户通过语音控制定位某个分类任务时，日程页与计时页语义不一致。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 为 iOS 日程任务行分类 badge 补齐 Voice Control input labels。
2. 复用计时页分类 badge 的输入标签语义：`Text(task.category)` 和 `Text("\(task.category)分类")`。
3. 在 `scripts/verify_project.sh` 的日程任务行分类 badge 源码契约中锁定该语义。
4. README、测试规范、核心流程文档和更新日志同步本轮分类 UI 与 CI 契约变化。

## 非目标

- 不改变任务筛选逻辑、分类排序、任务模型、持久化、计划生成或计时状态机。
- 不修改 Mac UI、Live Activity、通知、StoreKit、EventKit 或 Xcode 工程结构。
- 不修改 GitHub Actions workflow、artifact 文件列表、validator 结构或 Xcode 构建命令。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `ScheduleTaskCell` 位于 `ChronoFocus/Views/ScheduleView.swift`，负责 iOS 日程任务行展示，分类 badge 已使用 `TaskCategoryPreset.matching(task.category)` 决定图标和颜色，并已有 `.accessibilityLabel("\(task.category)分类")`。
- `TimerTaskCategoryBadge` 位于 `ChronoFocus/Views/TimerView.swift`，已提供同类 Voice Control input labels，可作为一致性参照。
- `scripts/verify_project.sh` 已通过 `schedule_task_cell` 源码切片锁定 iOS 日程任务行分类 badge、图标和可访问标签，可在同一位置增加 input labels 断言。
