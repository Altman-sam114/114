# v0.32（分类筛选 Voice Control 标签）

## 版本号

v0.32

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.31，因此本轮自动递增为 v0.32。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.28 已为三端分类筛选 chip 增加 VoiceOver label/hint，v0.30 已补齐 selected trait。当前分类 chip 的可访问 label 包含分类名、数量和已选状态，对 VoiceOver 有帮助，但对 Voice Control 用户来说命令可能过长；应提供更短的 input labels，让用户可直接说分类名触发筛选。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页 `TaskCategoryFilterChip` 增加分类名 Voice Control input labels。
2. iOS 计时页 `TimerTaskCategoryFilterChip` 增加分类名 Voice Control input labels。
3. macOS 日程详情 `MacCategoryFilterChip` 增加分类名 Voice Control input labels。
4. `scripts/verify_project.sh` 的三端分类 chip 可访问 contract 增加 input labels 源码检查。
5. README、测试规范、核心流程文档和更新日志同步 Voice Control 标签行为。

## 非目标

- 不改变分类筛选视觉样式。
- 不改变点击切换、清除筛选或 selected trait 行为。
- 不改变 `FocusStore`、`TimerEngine`、计划、统计、提醒或持久化逻辑。
- 不引入第三方依赖。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- 三端分类筛选 chip 已有 `title`、`count` 和 `isSelected`，并统一使用 SwiftUI `Button`。
- SwiftUI 可通过 `.accessibilityInputLabels(...)` 为 Voice Control 提供更短命令标签。
- `scripts/verify_project.sh` 的 `assert_chip_accessibility` 已集中检查三端分类 chip 的可访问 label、hint 和 selected trait，适合继续补充 input labels contract。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在三个分类 chip struct 中增加 `voiceControlInputLabels: [Text]`，内容为 `Text(title)` 和 `Text("\(title)分类")`。
3. 在三个 chip 的 modifier 链中增加 `.accessibilityInputLabels(voiceControlInputLabels)`。
4. 在 `scripts/verify_project.sh` 的 `assert_chip_accessibility` 中检查 `voiceControlInputLabels: [Text]`、`Text("\(title)分类")` 和 `.accessibilityInputLabels(voiceControlInputLabels)`。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.32: 增加分类语音控制标签` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
