# v0.16（CI 结果包校验收紧）

## 版本号

v0.16

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.15 已补齐 StoreKit / EventKit 本地配置说明并通过最新 `origin/main` 云端 artifact 验收。本轮继续在 `v0（持续优化）` 阶段递增为 v0.16，处理 v0.15 后续建议：收紧 `scripts/validate_ci_artifact.rb` 对 CI artifact index、failure summary 日志入口和 JUnit testcase 的结构化核对。

## 背景

`scripts/validate_ci_artifact.rb` 已能核对 manifest branch / commit / run / attempt、阶段 outcome、artifact index required entry size、JUnit 总数、failure summary 成功标记、主日志成功标记和 Mac 快照 manifest。但当前校验仍偏宽松：artifact index 只要求 `entryCount >= 19`，failure summary 只看 `All CI stages passed.`，JUnit 只看 `tests=4` 和 `failures=0`。Agent C 仍需要人工核对不少固定路径和日志入口。本轮不改变 workflow 产物格式，只让校验脚本显式锁定已有 artifact contract，降低用旧包、漏包或半完整结果包误判为通过的风险。

## 目标

- 在 `scripts/validate_ci_artifact.rb` 中显式定义并核对必需 artifact index 路径和 kind。
- 校验 index 中必需路径对应的下载后本地文件/目录真实存在且非空。
- 校验 manifest 中关键路径字段与当前 workflow contract 一致。
- 校验 `ci-failure-summary.md` 包含 static/project/Mac/iOS 日志入口、Mac/iOS `.xcresult` 入口、Mac snapshots 入口和 `All CI stages passed.`。
- 校验 `junit.xml` 四个 testcase 名称必须是 `staticChecks`、`projectVerification`、`macBuild`、`iosBuild`。
- 在 `scripts/verify_project.sh` 中加入一个小型 CI artifact fixture，运行 validator 覆盖新增检查。
- 更新 README、`md/test/test.md`、`md/flow/flow.md` 和 `update_log.md` 中受影响的说明。

## 非目标

- 不修改 `.github/workflows/ci-results.yml` 的 artifact 格式或上传内容。
- 不扩大 CI artifact 下载范围。
- 不引入第三方依赖。
- 不修改 Swift 业务逻辑、UI 或 Xcode 工程配置。
- 不把旧 run、旧 artifact 或本地 fixture 冒充云端验收结论。

## 当前架构依据

- `.github/workflows/ci-results.yml` 当前 artifact index 固定包含 19 个 required paths：manifest、index、failure summary、JUnit、static / verify / Mac / iOS logs、Xcode version、run context、Mac/iOS `.xcresult`、Mac snapshot 目录、snapshot manifest 和五张快照。
- workflow failure summary 固定列出 Static checks、Project verification、Mac build、Xcode result bundle、iOS build、iOS Xcode result bundle 和 Mac snapshots。
- `md/test/test.md` 已要求 Agent C 核对 artifact index、failure summary、JUnit、主日志、`.xcresult` 和项目专属产物。
- `scripts/verify_project.sh` 是本地/云端 smoke 入口，适合加入小 fixture 验证 validator 自身行为。

## 实现步骤

1. 更新 `scripts/validate_ci_artifact.rb`：
   - 增加 `EXPECTED_INDEX_ENTRIES`，记录每个必需 index path 的 expected kind。
   - 增加 `EXPECTED_MANIFEST_PATHS`，核对 `junitPath`、`buildLogPath`、`iosBuildLogPath`、`failureSummaryPath`、`artifactIndexPath`、Mac/iOS result bundle path 等关键字段。
   - 增加 `EXPECTED_SUMMARY_ENTRIES`，核对 `ci-failure-summary.md` 的日志和结果包入口。
   - 增加 `EXPECTED_JUNIT_TESTCASES`，核对 testcase 名称集合。
   - 保留现有 branch / commit / run / attempt、outcome、日志成功标记和快照检查。
   - 对 index entry path 先按 `ci-results/...` contract 检查，再映射到下载 artifact 根目录下的真实本地路径确认存在和非空。
2. 更新 `scripts/verify_project.sh`：
   - 增加 validator 新常量和检查项的 marker grep。
   - 构造临时最小 CI artifact fixture，包含 manifest、index、summary、JUnit、logs、Mac/iOS `.xcresult` 目录、Mac snapshots 和 snapshot manifest。
   - 运行 `ruby scripts/validate_ci_artifact.rb "$fixture" --commit <sha> --run-id <id> --attempt 1`，确保新增校验有本地覆盖。
3. 更新 README：
   - 说明 validator 现在会显式核对 artifact index 必需路径、summary 日志入口、JUnit testcase 和下载后本地文件/目录存在性。
4. 更新 `md/test/test.md`：
   - 更新 Agent C 结构化复判脚本覆盖范围。
   - 在 Smoke 当前基线中写明 validator fixture 已覆盖路径 contract。
5. 更新 `md/flow/flow.md`：
   - 同步 validator 当前职责描述。
6. 更新 `update_log.md`：
   - 追加 v0.16 记录，写明本地验证和云端结论待 push 后 Agent C 复判。

## 测试要求

本地轻量检查：

```bash
git diff --check
ruby -c scripts/validate_ci_artifact.rb
ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
bash scripts/verify_project.sh
```

如本地已有最新 artifact，可额外运行：

```bash
ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> \
  --commit <sha> \
  --run-id <run_id> \
  --attempt <attempt>
```

不默认运行完整本机 Xcode build；最终编译结论以 `origin/main` 云端 artifact 为准。

## 文档更新要求

- README 面向用户和 Agent C，说明 validator 结构化复判范围。
- `md/test/test.md` 面向 Agent B / Agent C，更新测试分层和验收要求。
- `md/flow/flow.md` 只同步当前真实核心验证职责。
- `update_log.md` 记录版本、日期、核心变更、关键文件、验证结果和遗留事项。

## 验收标准

- `scripts/validate_ci_artifact.rb` 仍能通过 v0.15 最新下载 artifact。
- `scripts/verify_project.sh` 通过，并覆盖 validator fixture。
- README / 测试规范 / 核心流程文档与脚本实际行为一致。
- push 到 `origin/main` 后，Agent C 下载最新 run artifact，用新版 `scripts/validate_ci_artifact.rb` 和手工核对共同确认结果包通过。

## 风险和禁止项

- 禁止写死与 `.github/workflows/ci-results.yml` 不一致的路径。
- 禁止把 fixture 当成真实云端验收。
- 禁止修改 artifact 上传格式导致 Agent C 下载规则扩大。
- 禁止引入第三方依赖或大体积测试数据。
- 禁止读取、提交或删除无关未跟踪文件。

## main 同步、提交和 push 要求

Agent B 开始前必须执行：

```bash
git fetch origin
git switch main
git pull --ff-only origin main
git status --short --branch
```

提交前确认只包含本轮相关文件，提交信息使用：

```text
v0.16: 收紧 CI 结果包校验
```

推送目标固定为：

```bash
git push origin main
```

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 下载最新 `origin/main` commit 对应 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 运行 `ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> --commit <sha> --run-id <run_id> --attempt <attempt>`。
- 同时继续人工核对 manifest、artifact index、JUnit、failure summary、主日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest。
