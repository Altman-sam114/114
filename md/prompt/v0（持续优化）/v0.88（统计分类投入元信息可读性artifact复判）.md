# v0.88（统计分类投入元信息可读性 artifact 复判）

## 版本号

v0.88

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化分类 UI 易用性和 CI 测试。远端最新 v0.87 已完成统计分类投入空态语义，并通过最新 `origin/main` 云端 artifact 验收。v0.88 继续在同一 v0 持续优化阶段递增。

## 背景

SwiftUI 设计参考明确指出 `.caption2` 字号过小，应尽量避免。当前 iOS/macOS 统计页“分类投入”有数据行中，专注次数和排行位置使用 `.font(.caption2.weight(.medium))`，这两项正是帮助用户扫描分类表现的关键元信息。把它们提升到 `.caption` 能改善小屏、动态字体和 macOS 详情页中的可读性，同时保持现有统计口径和布局结构。

## 目标

- iOS `AnalyticsView` 分类投入行的专注次数/排行元信息不再使用 `.caption2.weight(.medium)`。
- macOS `MacAnalyticsDetailView` 分类投入行同步不再使用 `.caption2.weight(.medium)`。
- 两端新增同名 helper `categoryShareMetadataFont()`，统一返回 `.caption`，并由元信息 `HStack` 使用。
- 保留现有分类、时长、占比、排行、排序依据、空态、accessibility label 和 Voice Control input labels。
- 不改变 `FocusStore.categoryBreakdown()`、统计排序、会话聚合、Pro gating 或布局层级。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics category share metadata readability contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics category share metadata readability contracts` 复判。
- README、测试规范、核心流程文档、流程图和更新日志同步 v0.88 的云端 artifact 复判范围。

## 非目标

- 不新增统计筛选器、排序切换器、引导按钮或交互控件。
- 不调整分类投入行的统计值、排序规则、颜色、进度条或百分比胶囊。
- 不修改 `CategoryFocus`、`FocusStore.categoryBreakdown()`、`FocusSession` 或计划生成逻辑。
- 不做 SwiftUI 大拆分或无关视觉重设计。

## 当前架构依据

- iOS 统计分类投入 UI 位于 `ChronoFocus/Views/AnalyticsView.swift` 的 `categoryChart`。
- macOS 统计分类投入 UI 位于 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift` 的 `MacCategoryChartPanelView`。
- v0.84-v0.87 已逐步锁定分类投入次数、排行、排序依据和空态语义。本轮只提升元信息字体可读性。
- 项目 CI 通过 `scripts/verify_project.sh` 输出项目专属 marker，再由 `scripts/validate_ci_artifact.rb` 下载 artifact 后结构化复判。

   - 检查两端不再包含 `.font(.caption2.weight(.medium))`。
   - 输出 `Analytics category share metadata readability contracts verified.`。
   - 成功 fixture 补齐新 marker。
   - 新增 `negative_analytics_category_share_metadata_readability_marker_fixture`。
   - 增加自锁 grep，确保 validator 和负向 fixture 覆盖新契约。
4. 修改 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project analytics category share metadata readability contracts` 检查。
5. 更新文档：
   - README 和 `md/test/test.md` 补充元信息可读性 marker、PASS 和负向 fixture。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 补充当前真实统计分类投入可读性和 artifact 复判范围。

## 验收标准

- iOS 和 macOS 分类投入行元信息使用 `categoryShareMetadataFont()`。
- iOS 和 macOS 均存在 `private func categoryShareMetadataFont() -> Font { .caption }`。
- iOS 和 macOS 分类投入行不再使用 `.font(.caption2.weight(.medium))`。
- 既有分类投入次数、排行、排序依据、空态、可访问标签和 Voice Control input labels 不退化。
- `FocusStore.categoryBreakdown()` 排序和统计口径未改。
- 最新 `origin/main` 云端 run 成功。
- Agent C 下载的最新 artifact 中 validator 全 PASS，包含 `PASS verify_project analytics category share metadata readability contracts`、`PASS verify_project analytics category share empty state contracts`、`PASS manifest overall outcome`、`PASS mac build succeeded` 和 `PASS ios build succeeded`。
