# v0.55 当前任务语音选择标签

## 版本号

v0.55

## 版本分配依据

人工总目标要求 Agent X 持续循环优化 UI 分类体验和 CI 测试。v0.54 已补齐当前任务选择行的已选中状态、提示和 selected trait，本轮在同一交互面继续小步增强 Voice Control 可用性，因此从 v0.54 递增到 v0.55。

## 背景

计时页、Mac 详细计时页和 Mac 菜单栏小窗都能从待办列表选择当前番茄钟任务。v0.54 已让辅助技术读出“已选中/未选中”和选择提示，但当前任务选择行本身尚未提供任务名、分类名语义的 Voice Control input labels；同时计时运行中按钮被禁用时，未选中行仍容易被理解为可立即切换。

## 目标

- iOS 计时页 `TaskRow` 增加当前任务选择行 Voice Control input labels。
- macOS 详细计时页 `MacTaskRowView` 增加同等 Voice Control input labels。
- macOS 菜单栏小窗 `MacMiniTaskPickerView` 增加同等 Voice Control input labels。
- 三端选择提示在计时运行中说明当前不可切换，避免禁用状态和提示语义冲突。
- `scripts/verify_project.sh` 增加三端源码契约，锁定 input labels 和运行中提示语义。
- 同步 README、测试规范、核心流程文档、流程图和更新记录。

## 非目标

- 不改变 `TimerEngine` 计时状态机。
- 不改变 `FocusStore` 数据模型、持久化或分类排序规则。
- 不调整任务筛选、计划生成、统计或通知行为。
- 不引入第三方依赖。
- 不重构长 View 文件。

## 当前架构依据

- 任务选择仍由 SwiftUI View 收集用户意图后调用 `TimerEngine.selectTask(_:)`。
- 计时运行中已有 `.disabled(engine.isRunning)` 禁止切换当前任务，本轮只让行组件知道该状态并同步可访问提示。
- 分类和任务名来自 `FocusTask`，不新增模型字段。
- 可访问性契约继续通过 `scripts/verify_project.sh` 的源码检查由云端 CI 执行。

## 实现步骤

1. 在 `ChronoFocus/Views/TimerView.swift` 中：
   - `TaskRow` 增加 `isTimerRunning` 参数。
   - 外层调用传入 `engine.isRunning`。
   - 增加 `selectionInputLabels`，包含任务名、任务名待办、分类待办等标签。
   - `.accessibilityInputLabels(selectionInputLabels)` 接到行根元素。
   - `selectionHintText` 在运行中且未选中时返回“计时运行中不可切换当前待办”。
2. 在 `ChronoFocusMac/Views/MacTimerDetailView.swift` 中：
   - `MacTaskRowView` 增加 `isTimerRunning` 参数。
   - 外层调用传入 `engine.isRunning`。
   - 增加同等 input labels 和运行中 hint。
3. 在 `ChronoFocusMac/Views/MacMiniTimerView.swift` 中：
   - `MacMiniTaskPickerView` 的行 label 增加 `.accessibilityInputLabels(selectionInputLabels(for: task))`。
   - `selectionHintText` 根据 `engine.isRunning` 返回运行中不可切换提示。
4. 在 `scripts/verify_project.sh` 中：
   - 增强 v0.54 的三条 Ruby contract，要求三端包含 `selectionInputLabels`、`.accessibilityInputLabels(...)`、任务名/分类待办标签和运行中不可切换提示。
5. 同步文档：
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`
