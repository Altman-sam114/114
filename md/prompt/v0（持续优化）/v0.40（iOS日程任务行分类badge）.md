# v0.40（iOS 日程任务行分类 badge）

## 版本号

v0.40

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.39，因此本轮自动递增为 v0.40。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 macOS 任务行和 iOS 计时页待办行已经能以分类 badge 保留任务分类上下文，但 iOS 日程页的 `ScheduleTaskCell` 仍只用纯文本展示 `task.category`。当任务同时有截止时间、循环和自动开始信息时，分类不够醒目，不利于在日程列表中快速识别分类。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页 `ScheduleTaskCell` 将纯文本分类升级为带图标、颜色和背景的分类 badge。
2. 分类 badge 优先使用 `TaskCategoryPreset.matching(task.category)` 的图标和代表色，未知分类回退 `task.accentHex` 和 `tag.fill`。
3. 保留截止时间、循环和自动开始等任务元数据，不能用截止时间替代分类。
4. 分类 badge 补充稳定可访问标签。
5. `scripts/verify_project.sh` 增加 iOS 日程任务行分类 badge 静态契约。
6. README、测试规范、核心流程文档和更新日志同步本轮覆盖范围。

## 非目标

- 不改变 `FocusTask`、`TaskCategoryPreset` 或任何 Codable 持久化字段。
- 不改变任务筛选、排序、计划生成、计时状态机或通知逻辑。
- 不改变 macOS UI。
- 不改变 GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `FocusTask.category` 是真实持久化分类字段，代表色保存在 `FocusTask.accentHex`。
- `TaskCategoryPreset.matching(_:)` 提供常用分类的图标和代表色，当前已被 iOS 计时页、iOS 编辑器、Mac 小窗和 Mac 任务行复用。
- `ScheduleTaskCell` 只属于 iOS SwiftUI View 层，适合做局部展示优化，不应新增业务状态。
- `scripts/verify_project.sh` 已用 Ruby slice 锁定多个分类 UI 契约，可新增针对 `ScheduleTaskCell` 的最小源码 contract。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `ChronoFocus/Views/ScheduleView.swift` 的 `ScheduleTaskCell` 中新增 `categoryPreset`、`categoryTint`、`categorySymbolName` helper。
3. 将元数据行里的 `Text(task.category)` 替换为 `Label(task.category, systemImage: categorySymbolName)` badge，并添加颜色、背景、单行限制和 `.accessibilityLabel("\(task.category)分类")`。
4. 保留 `if let dueDate = task.dueDate { Text(dueDate.scheduleTimeText) }`、循环和自动开始文案。
5. 在 `scripts/verify_project.sh` 中新增 `ScheduleTaskCell` slice 检查，断言 badge helper、`Label(...)`、可访问标签和 due date 旁路元数据仍存在。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
7. 只检查 diff 和 status，不运行本地测试命令。
8. 提交 `v0.40: 增加 iOS 日程任务行分类 badge` 并 push 到 `origin/main`。
9. Agent C 下载最新 run artifact 并按下方标准验收。
