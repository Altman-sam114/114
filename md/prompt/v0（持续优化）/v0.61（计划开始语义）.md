# v0.61（计划开始语义）

## 版本号

v0.61

## 版本分配依据

人工总目标要求持续优化 UI 易用性、分类体验和 CI 测试。v0.60 已完成 JUnit 元数据复判并通过云端 artifact 验收；本轮继续按小版本递增到 v0.61，聚焦从自动计划项进入专注的开始按钮语义。

## 背景

iOS 计划行开始按钮当前读作“开始计划番茄钟”，macOS 计划行开始按钮为 icon-only “开始”。这些标签缺少任务名、轮次和时间段，用户在多个计划项之间切换时难以确认即将启动哪一项。

## 目标

- iOS 自动计划项“开始”按钮读出任务名、时间段、轮次和分类。
- macOS 自动计划项“开始”按钮读出任务名、时间段和轮次，快照静态控件也保留任务名级别语义。
- 为真实按钮补齐 Voice Control input labels，支持按任务名启动计划项。
- 在 `scripts/verify_project.sh` 增加源码契约 marker，云端 artifact 可复判本轮行为。
- 同步 README、测试规范、核心流程和更新记录。

## 非目标

- 不改计划生成、排序、完成状态、`TimerEngine.startPlanItem(_:)` 或 `FocusStore`。
- 不改可见布局和按钮尺寸。
- 不运行本地测试、build、lint、`git diff --check` 或脚本验证；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- iOS 计划行由 `ChronoFocus/Views/ScheduleView.swift` 的 `PomodoroPlanRow` 渲染。
- macOS 计划行由 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 渲染，真实按钮调用 `engine.startPlanItem(item)`。
- `PomodoroPlanItem` 已含 `taskTitle`、`timeRangeText`、`roundNumber`、`category` 和 `accentHex`，无需改模型。
- `scripts/verify_project.sh` 是项目专属验证入口；云端 workflow 会运行它并把日志纳入 artifact。

## 实现步骤

1. 同步最新 `main`，确认没有无关 tracked diff，不读取或提交用户未跟踪文件。
2. 修改 iOS `PomodoroPlanRow`：
   - 开始按钮 accessibility label 包含 `item.taskTitle`、`item.timeRangeText`、`item.roundNumber` 和 `item.category`。
   - 开始按钮 input labels 包含“开始 + 任务名”“任务名 + 第 N 轮”等。
3. 修改 macOS 计划行：
   - 真实开始按钮 accessibility label/input labels 包含任务名、时间段和轮次。
   - 快照静态 `MacStaticScheduleActionChipView` title 改成带任务名/轮次的可访问语义，不改变可见 icon-only 布局。
4. 修改 `scripts/verify_project.sh`：
   - 增加 iOS/Mac 计划开始按钮语义源码契约。
   - 输出稳定 marker，例如 `Plan start action accessibility contracts verified.`。
5. 同步文档和 `update_log.md`。
