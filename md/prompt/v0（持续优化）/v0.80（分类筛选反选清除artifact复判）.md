# v0.80（分类筛选反选清除 artifact 复判）

## 版本号

v0.80

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；v0.79 已完成当前任务选择行分类语义 artifact 复判，并通过最新 `origin/main` 云端 artifact 验收。v0.80 继续在同一 v0 持续优化阶段递增。

## 背景

iOS 日程、iOS 计时和 macOS 日程分类筛选 chip 已支持“再次点击当前已选分类即可清除筛选”，这是分类使用体验中的核心效率细节。`scripts/verify_project.sh` 已有源码契约检查，但该行为目前只并入泛化的分类 chip accessibility marker，没有独立进入 Agent C artifact validator 的复判清单。

## 目标

- 为分类筛选反选清除行为新增独立 `verify_project.log` marker：`Category filter toggle contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project category filter toggle contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- 文档同步说明分类筛选反选清除已成为云端 artifact 强制复判项。

## 非目标

- 不改 SwiftUI UI 行为、分类筛选状态、模型、`FocusStore` 或 `TimerEngine`。
- 不扩大到分类 chip 视觉样式、分类排序或新增分类逻辑。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；人工明确要求本轮只去云端验证。

## 当前架构依据

- 分类筛选 option 来自 `TaskCategoryPreset` / `TaskCategoryFilterOption`，真实分类仍保存为 `FocusTask.category`。
- iOS 日程、iOS 计时、macOS 日程都已有 `selectedCategory == category ? nil : category` 的清除行为。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 在 `scripts/verify_project.sh` 现有 iOS 日程、iOS 计时、macOS 日程分类筛选 toggle-off 源码检查后输出 `Category filter toggle contracts verified.`。
2. 在 `scripts/validate_ci_artifact.rb` 增加 `verify_project category filter toggle contracts` 检查。
3. 更新 `scripts/verify_project.sh` 的自检 grep、小型成功 fixture 和 marker 缺失负向 fixture。
4. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
