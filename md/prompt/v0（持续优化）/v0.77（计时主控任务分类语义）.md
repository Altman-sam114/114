# v0.77（计时主控任务分类语义）

## 版本号

v0.77

## 版本分配依据

人工未指定版本号。当前最新云端通过版本为 v0.76，本轮继续围绕“优化 UI，更加方便使用，优化分类”做小步增强，因此自动递增为 v0.77。

## 背景

iOS/macOS 当前任务选择行、计划开始按钮和待办行操作已经能读出任务名与分类，但计时主控按钮仍多为“开始”“暂停”“停止”“跳过”这类泛化语义。用户在同名任务或跨分类任务之间切换后，VoiceOver 与 Voice Control 用户不容易确认操作将作用在哪个任务和分类。本轮给计时主控按钮补齐当前任务/分类上下文。

## 目标

- iOS `TimerView` 的开始/暂停/继续、停止、跳过和只设开始任务完成按钮读出当前任务名和分类。
- macOS `MacTimerDetailView` 的开始/暂停/继续、停止、跳过按钮读出当前任务名和分类。
- macOS `MacMiniTimerView` 的小窗开始/暂停/继续、停止、跳过按钮读出当前任务名和分类。
- 为上述真实按钮补齐 Voice Control input labels，保留原有短标签，追加任务名和分类指令。
- macOS 快照静态计时操作控件同步任务/分类语义，避免快照安全控件与真实控件不一致。
- `scripts/verify_project.sh` 新增 `Timer action accessibility contracts verified.` 源码契约 marker。
- `scripts/validate_ci_artifact.rb` 新增该 marker 的 artifact 复判。
- README、测试规范、核心流程文档、流程图和更新日志同步计时主控任务/分类语义。

## 非目标

- 不修改 `TimerEngine`、`FocusStore`、模型、计划生成、通知或持久化逻辑。
- 不改变可见按钮文案、图标、布局或禁用规则。
- 不修改 CI artifact 名称规则、run context 精确字段集合或 CI process version。
- 不运行本地测试、构建、lint、`git diff --check`、`bash scripts/verify_project.sh` 或本地 fixture validator。

## 当前架构依据

- iOS 计时主控位于 `ChronoFocus/Views/TimerView.swift` 的 `actionBar` 和 `openEndedFinishButton`。
- macOS 详细计时主控位于 `ChronoFocusMac/Views/MacTimerDetailView.swift` 的 `MacTimerActionRowView` 和 `MacStaticTimerActionRowView`。
- macOS 小窗计时主控位于 `ChronoFocusMac/Views/MacMiniTimerView.swift` 的 `MacMiniControlsView`。
- 当前任务可通过 `store.task(for: engine.selectedTaskID)` 获取；没有选中任务时回退到 `engine.currentTaskTitle`。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `ChronoFocus/Views/TimerView.swift`：
   - 新增当前任务/分类操作上下文 helper。
   - 开始/继续/暂停、停止、跳过、只设开始任务完成按钮使用上下文化 accessibility label。
   - 为这些按钮补齐 `.accessibilityInputLabels(...)`，保留短命令并追加任务名、分类名命令。

2. 更新 `ChronoFocusMac/Views/MacTimerDetailView.swift`：
   - `MacTimerActionRowView` 引入 `FocusStore` 环境对象。
   - 真实按钮和静态快照 chip 使用当前任务/分类上下文化 accessibility label 和 input labels。

3. 更新 `ChronoFocusMac/Views/MacMiniTimerView.swift`：
   - `MacMiniControlsView` 引入 `FocusStore` 环境对象。
   - 小窗 icon-only 控件补齐任务/分类语义与 Voice Control input labels。

4. 更新验证脚本：
   - `scripts/verify_project.sh` 新增源码契约和 `Timer action accessibility contracts verified.` marker。
   - 小型成功 fixture 补齐 marker，并新增 marker 缺失负向 fixture。
   - `scripts/validate_ci_artifact.rb` 新增 `verify_project timer action accessibility contracts` 复判。

5. 同步文档：
   - README 描述 iOS/macOS 计时主控按钮会读出当前任务和分类。
   - `md/test/test.md` 更新验证范围和 Agent C PASS 清单。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实分类 UI 语义。
