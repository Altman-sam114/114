# v0.14（iOS 模拟器构建基线）

## 版本号

v0.14

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试。v0.13 已新增 CI 结果包校验脚本并通过最新 `origin/main` 云端 artifact 验收。本轮继续在 `v0（持续优化）` 阶段递增为 v0.14，处理 `update_log.md` 中“本机模拟器构建命令和本机 destination 基线仍可继续补强”的遗留项。

## 背景

云端已固定使用 `generic/platform=iOS` 构建 `ChronoFocus` scheme；但本机诊断 iOS 构建时仍需要人工执行 `-showdestinations` 后手写 destination。不同机器的模拟器名称和系统版本可能不同，容易造成命令不稳定。本轮新增一个只读解析脚本，从 `xcrun simctl list devices available -j` 中选择可用 iOS 模拟器，输出稳定的 `platform=iOS Simulator,id=<UDID>` destination，并可打印完整本机 iOS build 命令。

## 目标

- 新增 `scripts/resolve_ios_simulator_destination.rb`。
- 默认从 `xcrun simctl list devices available -j` 读取可用设备。
- 支持 `--simctl-json PATH`，便于本地脚本用小 fixture 验证解析逻辑。
- 支持 `--name NAME` 优先选择指定模拟器名称。
- 支持 `--print-build-command` 输出完整 `xcodebuild` 命令。
- 无可用 iOS 模拟器时输出清晰错误并非 0 退出。
- 不改变默认本地验证策略；完整本机 iOS build 仍只在人工明确要求或定位问题时运行。

## 非目标

- 不在默认本地检查中运行完整 iOS simulator build。
- 不修改 GitHub Actions workflow。
- 不创建、启动或删除模拟器。
- 不依赖 `jq` 或第三方库。

## 当前架构依据

- `md/test/test.md` 已记录云端 iOS generic build 命令和本机 iOS build 需要根据当前机器 destination 选择。
- `scripts/verify_project.sh` 是本地/云端项目专属验证入口，可增加脚本语法和 fixture 解析检查。
- README 已提供本地验证和云端验证入口。

## 实现步骤

1. 新增 `scripts/resolve_ios_simulator_destination.rb`：
   - 使用 Ruby 标准库 `json`、`optparse`、`open3`、`shellwords`。
   - 在 `DEVELOPER_DIR` 未设置且 `/Applications/Xcode.app/Contents/Developer` 存在时自动使用完整 Xcode，避免当前机器 `xcode-select` 指向 Command Line Tools 时找不到 `simctl`；如果用户已经设置 `DEVELOPER_DIR`，打印的 build 命令应尊重该设置。
   - 解析 `simctl` devices JSON，只选择 runtime key 包含 `SimRuntime.iOS` 的设备。
   - 过滤 `isAvailable == false` 的设备。
   - 排序优先级：Booted、用户指定名称、常见 iPhone 名称、较新 runtime。
   - 默认输出 `platform=iOS Simulator,id=<UDID>`。
   - `--print-build-command` 输出带当前 `DEVELOPER_DIR` 或默认完整 Xcode、`ChronoFocus` scheme、`CODE_SIGNING_ALLOWED=NO` 的 build 命令。
2. 更新 `scripts/verify_project.sh`：
   - 把新脚本加入 required files。
   - `ruby -c scripts/resolve_ios_simulator_destination.rb`。
   - 用临时 fixture 验证脚本能选择 booted/指定名称 simulator，并输出 `platform=iOS Simulator,id=...`。
3. 更新 README：
   - 在本地验证 section 增加本机 iOS simulator destination 解析命令和可选 build 命令。
4. 更新 `md/test/test.md`：
   - 在 Full / iOS 构建基线中替换手工 `-showdestinations` 为脚本命令，同时保留手工 fallback。
5. 更新 `update_log.md`：
   - 追加 v0.14 记录。
   - 从遗留问题中移除“本机模拟器构建命令和本机 destination 基线仍可继续补强”。

## 测试要求

本地轻量检查：

```bash
git diff --check

ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'
plutil -lint ChronoFocus.xcodeproj/project.pbxproj
bash scripts/verify_project.sh
```

```bash

```

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 下载最新 `origin/main` commit 对应 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 运行 `ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> --commit <sha> --run-id <run_id> --attempt <attempt>`。
- 同时继续人工核对 manifest、artifact index、JUnit、failure summary、主日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest。
