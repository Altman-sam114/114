# v0.13（CI 结果包校验脚本）

## 版本号

v0.13

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试。v0.12 已完成 Mac 小窗分类上下文并通过最新 `origin/main` 云端 artifact 验收。本轮继续在 `v0（持续优化）` 阶段递增为 v0.13，聚焦 Agent C 反复手工核对 CI artifact 的流程，新增项目内可复用校验脚本。

## 背景

当前 `.github/workflows/ci-results.yml` 已生成 manifest、artifact index、JUnit、failure summary、Mac/iOS build 日志和 Mac 快照 manifest。Agent C 每轮需要手工核对 branch、commit、run id、run attempt、required entries、JUnit、build 成功标记和快照尺寸。把这些规则沉淀为脚本，可以减少人工漏检，也能让后续 Agent C 用同一命令复判下载后的结果包。

## 目标

- 新增 `scripts/validate_ci_artifact.rb`，用于校验已下载的单个 CI artifact 目录。
- 脚本支持参数：
  - artifact 目录路径。
  - `--commit <sha>`。
  - `--run-id <id>`。
  - `--attempt <n>`。
  - `--branch <name>`，默认 `main`。
- 校验内容覆盖：
  - `ci-artifact-manifest.json` 存在且 branch、commit、run id、attempt 匹配。
  - manifest 中 static/project/mac/iOS/test outcome 全部为 `success`。
  - `ci-artifact-index.json` 存在且 required entries 缺失数为 0、entry count 至少覆盖当前 19 项。
  - `junit.xml` 为 4 tests、0 failures。
  - `ci-failure-summary.md` 包含 `All CI stages passed.`。
  - `verify_project.log` 包含 `Mac core tests passed.` 和 `Project structure verified.`。
  - `xcodebuild.log` 与 `ios-xcodebuild.log` 包含 `** BUILD SUCCEEDED **`。
  - Mac snapshot manifest 包含 5 张指定截图，且宽高和字节数为正。
- 脚本输出 `PASS ...` / `FAIL ...` 行，任一失败时非 0 退出。
- 不改 GitHub Actions workflow 结构，不改变 artifact 格式。

## 非目标

- 不下载 artifact；下载仍由 `gh run download` 完成。
- 不修改 `.github/workflows/ci-results.yml` 的 artifact 格式或 `CI_PROCESS_VERSION`。
- 不运行 Xcode build。
- 不删除既有手工验收要求；脚本是 Agent C 的辅助证据，不替代必要人工判断。

## 当前架构依据

- 云端结果包格式由 `.github/workflows/ci-results.yml` 生成。
- Agent C 缓存目录规则为 `/private/tmp/chronofocus-c-review-<run_id>/`，下载后通常包含一个 `chronofocus-ci-...` artifact 子目录。
- 测试规范在 `md/test/test.md` 中定义 Agent C 必须核对 manifest、artifact index、JUnit、failure summary、日志、`.xcresult` 和 Mac 快照。
- README 和 `md/flow/flow.md` 已描述 artifact index 和 Agent C 复判流程。

## 实现步骤

1. 新增 `scripts/validate_ci_artifact.rb`：
   - 使用 Ruby 标准库 `json`、`optparse`、`rexml/document`。
   - 只读 artifact 目录，不访问网络。
   - 如果传入的是下载缓存根目录且里面只有一个 artifact 子目录，应自动进入该子目录；如果有多个子目录，要求用户传入具体 artifact 目录。
   - 所有检查收集后统一输出，最后根据是否有失败决定退出码。
2. 更新 `scripts/verify_project.sh`：
   - 增加脚本文件存在性检查。
   - 使用 `ruby -c scripts/validate_ci_artifact.rb` 做语法检查。
   - 增加关键校验标记 grep，例如 `ci-artifact-manifest.json`、`missingRequiredCount`、`BUILD SUCCEEDED`、`Mac core tests passed.`。
3. 更新 README：
   - 在云端验证部分增加 Agent C 可用脚本命令示例。
4. 更新 `md/test/test.md`：
   - 在 Agent C 结果包验收要求中加入脚本使用方式。
5. 更新 `md/flow/flow.md`：
   - 在协作验证主链路或测试层关系中说明结果包可用脚本辅助复判。
6. 更新 `update_log.md`：
   - 追加 v0.13 记录。

## 验收标准

- `scripts/validate_ci_artifact.rb` 能对 v0.12 下载结果包输出全 PASS 并以 0 退出。
- `bash scripts/verify_project.sh` 通过，并覆盖新脚本语法和关键标记。
- 本地轻量检查通过。
- push 到 `origin/main` 后，Agent C 下载最新 run artifact，并使用新脚本和手工核对共同确认结果包通过。

## 风险和禁止项

- 禁止脚本访问网络、下载 artifact 或修改文件。
- 禁止放宽 Agent C 现有 artifact 核对要求。
- 禁止改 workflow artifact 格式导致旧结果包无法解析。
- 禁止读取、提交或删除无关未跟踪文件。

## main 同步、提交和 push 要求

```bash
git fetch origin
git switch main
git pull --ff-only origin main
git status --short --branch
```

提交前确认只包含本轮相关文件，提交信息使用：

```text
v0.13: 增加 CI 结果包校验脚本
```

推送目标固定为：

```bash
git push origin main
```

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 用 GitHub CLI 下载最新 `origin/main` commit 对应 run 的 artifact 到：

```text
/private/tmp/chronofocus-c-review-<run_id>/
```

- Agent C 应运行：

```bash
ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> \
  --commit <origin-main-sha> \
  --run-id <run_id> \
  --attempt <run_attempt>
```

- 同时继续人工核对 `ci-artifact-manifest.json`、`ci-artifact-index.json`、`ci-failure-summary.md`、`junit.xml`、`verify_project.log`、`xcodebuild.log`、`ios-xcodebuild.log`、`ChronoFocusMac.xcresult`、`ChronoFocus-iOS.xcresult`、Mac 快照和 Mac 快照 manifest。
