# v0.27（Mac 分类摘要快捷新增）

## 版本号

v0.27

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.26，因此本轮自动递增为 v0.27。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.26 已让三端分类 chip 支持再次点击已选分类退出筛选，并收紧 artifact index totals 校验。当前 iOS 日程页的选中分类摘要已有“新增此分类”入口，但 macOS 日程详情的选中分类摘要仍只有“清除”，用户需要回到左侧快速新增区域确认分类预填后再输入任务名。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. macOS 日程详情的选中分类摘要新增“新增此分类”入口。
2. 点击“新增此分类”时，将左侧快速新增表单分类和颜色预填为当前筛选分类，并聚焦任务名称输入框。
3. 保留现有“清除”、分类 chip 切换、左侧快速新增预填提示和新增待办逻辑。
4. `scripts/verify_project.sh` 增加 Mac 分类摘要新增入口和动作接线的源码 contract。
5. 同步 README、测试规范、核心流程图和更新日志。

## 非目标

- 不新增任务编辑 sheet，不改变 Mac 窗口布局结构。
- 不改 `FocusStore`、`TimerEngine`、计划、统计、提醒或持久化逻辑。
- 不重构 iOS/Mac 分类组件，不抽共享 UI。
- 不改 GitHub Actions workflow。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- Mac 日程详情页的快速新增表单状态由 `MacScheduleDetailView` 持有：`taskTitle`、`category`、`dueDate`、`estimatedRounds`、`accentHex`。
- Mac 选中分类状态由 `selectedCategory` 持有，当前已通过 `onChange(of: selectedCategory)` 同步快速新增分类和预设色。
- `MacTaskListPanelView` 只展示筛选列表和摘要，可通过闭包把“新增此分类”的意图传回父视图，不直接写持久化。
- `scripts/verify_project.sh` 已有分类摘要插入点、清除动作和点击切换的源码切片检查，适合补充本轮 contract。
