# v0.23（CI 本地缺失产物负向 fixture）

## 版本号

v0.23

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.22 已补齐 Mac 待办筛选计数，并通过最新 `origin/main` 云端 artifact 验收。本轮回到 CI 测试覆盖，递增为 v0.23。

## 背景

`scripts/validate_ci_artifact.rb` 已检查 artifact index 必需路径、kind、大小和下载后本地文件/目录非空状态。本地 `scripts/verify_project.sh` 目前有小型成功 fixture 和 artifactName mismatch 负向 fixture，但还缺少“index 声称文件存在、下载目录实际缺文件”的负向覆盖。本轮补齐这个拒绝路径，避免下载残缺包时只依赖 index 元数据。

## 目标

- 在 `scripts/verify_project.sh` 的 validator fixture 中新增本地缺失产物负向 fixture。
- 负向 fixture 复制成功 fixture 后删除一个必需本地文件，例如 `static-checks.log`，但保留 artifact index 原样。
- 运行 `scripts/validate_ci_artifact.rb` 时必须失败。
- 显式匹配 `FAIL index required local artifacts`，确认失败来自下载后本地产物完整性检查。
- README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md` 同步当前覆盖范围。

## 非目标

- 不修改 GitHub Actions workflow 产物格式。
- 不修改 validator 的真实验收规则。
- 不下载历史 artifact 或大体积测试数据。
- 不改 UI、模型、计时状态机或平台服务。

## 当前架构依据

- `scripts/validate_ci_artifact.rb` 的 `positive_local_artifact?` 已检查本地文件存在且非空，目录存在且包含非空文件。
- `scripts/verify_project.sh` 已能生成小型 artifact fixture，并运行 validator 成功路径。
- v0.19 已新增 artifactName mismatch 负向 fixture，本轮沿用同样模式增加本地缺失文件负向路径。

## 实现步骤

1. 更新 `scripts/verify_project.sh`：
   - 在 artifactName mismatch 负向 fixture 后新增 `missing_local_artifact_fixture`。
   - 复制成功 fixture 到新临时目录。
   - 删除 `static-checks.log`。
   - 运行 validator，期望失败。
   - 匹配 `FAIL index required local artifacts`。
   - 清理临时目录和输出文件。
   - 增加 marker grep，锁定新负向 fixture 和失败项。
2. 更新文档：
   - README 本地验证说明补充本地缺失产物负向 fixture。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 同步 validator fixture 覆盖。
   - `md/test/test.md` smoke 基线和 validator 描述补充本地缺失产物负向 fixture。
   - `update_log.md` 追加 v0.23 记录。

## 验收标准

- `scripts/verify_project.sh` 能证明 validator 成功 fixture 放行。
- artifactName mismatch 负向 fixture 仍失败并匹配 `FAIL run context artifact name`。
- 新增本地缺失产物负向 fixture 失败并匹配 `FAIL index required local artifacts`。
- 本地四项轻量检查通过。
- push 到 `origin/main` 后，Agent C 下载最新 v0.23 run artifact，用 `scripts/validate_ci_artifact.rb` 和手工核对共同确认结果包通过。

## 风险和禁止项

- 禁止把负向 fixture 写成只要 validator 失败就通过；必须匹配目标失败项。
- 禁止上传或下载大体积测试数据。
- 禁止修改生产 CI artifact 格式。
- 禁止读取、提交或删除无关未跟踪文件。

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 下载 artifact 前必须先完成 `gh auth login`，并确认 GitHub 账号为项目授权账号。
- Agent C 下载最新 `origin/main` commit 对应 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 运行 `ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> --commit <sha> --run-id <run_id> --attempt <attempt>`。
- 同时继续人工核对 manifest、artifact index、run context、JUnit、failure summary、主日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest。
