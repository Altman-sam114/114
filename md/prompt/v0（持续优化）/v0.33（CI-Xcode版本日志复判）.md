# v0.33（CI Xcode 版本日志复判）

## 版本号

v0.33

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.32，因此本轮自动递增为 v0.33。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前 GitHub Actions 会在 `ci-results/xcode-version.log` 中记录 runner 使用的 Xcode 版本；artifact index 已检查该文件存在且非空，但结构化复判还没有确认内容确实是 `xcodebuild -version` 的输出。补齐这层检查能让 Agent C 更明确地区分真实 Xcode 版本日志和空/错日志。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. `scripts/validate_ci_artifact.rb` 新增 `xcode version log` 复判。
2. 复判内容要求 `xcode-version.log` 包含 `Xcode` 和 `Build version`。
3. `scripts/verify_project.sh` 的 validator 正向 fixture 同步补齐 `Build version`。
4. `verify_project.sh` 自身锁定 validator 必须包含 Xcode 版本日志复判。
5. README、测试规范、核心流程文档和更新日志同步 Xcode 版本日志复判范围。

## 非目标

- 不改变 GitHub Actions workflow 阶段顺序。
- 不改变 artifact 文件列表或 CI process version。
- 不改 SwiftUI UI、业务逻辑、数据模型、计时状态机或平台服务。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `.github/workflows/ci-results.yml` 的 `Select Xcode` step 已执行 `xcodebuild -version | tee ci-results/xcode-version.log`。
- `scripts/validate_ci_artifact.rb` 已经要求 artifact index 包含 `ci-results/xcode-version.log`，但尚未复判内容 marker。
- `scripts/verify_project.sh` 已经用小型成功 fixture 跑 validator 正向路径，适合同步补齐 Xcode 版本日志内容。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在 `scripts/validate_ci_artifact.rb` 中增加 `xcode_version_log_path`。
3. 新增 `xcode version log` 检查，要求日志包含 `Xcode` 和 `Build version`。
4. 更新 `scripts/verify_project.sh` 中的 validator marker 检查和小型成功 fixture 的 `xcode-version.log` 内容。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.33: 复判 Xcode 版本日志` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
