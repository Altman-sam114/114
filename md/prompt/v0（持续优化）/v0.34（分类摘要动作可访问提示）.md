## 版本分配依据

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。当前分类 chip 已有选中状态、操作提示、selected trait 和 Voice Control input labels；Mac 分类摘要的可访问标签也会说明可新增或清除动作。但 iOS 日程页和计时页的选中分类摘要只读出当前筛选和数量，没有把摘要内的“新增此分类”或“清除”动作纳入整体可访问语义。补齐后，VoiceOver 用户能更快理解选中分类摘要不仅是状态，也能执行后续操作。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页 `SelectedCategorySummaryView` 的可访问标签说明可新增此分类待办或清除筛选。
2. iOS 计时页 `TimerSelectedTaskCategorySummaryView` 的可访问标签说明可清除筛选。
3. `scripts/verify_project.sh` 增加静态契约，锁定两个摘要动作可访问提示。
4. README、测试规范、核心流程文档和更新日志同步本轮 UI/CI 覆盖范围。

## 非目标

- 不改变分类筛选状态机、分类排序、持久化模型或 `FocusStore`。
- 不改变按钮布局、视觉主题、Mac 分类摘要或快照渲染逻辑。
- 不改变 GitHub Actions workflow、artifact 文件列表或 CI process version。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- `ScheduleView.swift` 的 `SelectedCategorySummaryView` 已显示分类、数量、“新增此分类”和“清除”按钮，并由 `showingEditor = true` 与 `selectedCategory = nil` 接线。
- `TimerView.swift` 的 `TimerSelectedTaskCategorySummaryView` 已显示分类、可启动数量和“清除”按钮，并通过 `clearTaskCategoryFilter` 接线。
- `MacScheduleDetailView.swift` 的 `MacSelectedCategorySummaryView` 已在可访问标签中说明“可新增此分类待办或清除筛选”，可作为 iOS 一致性参考。
- `scripts/verify_project.sh` 已用 Ruby source slice 锁定分类 chip 和摘要接线，适合补充摘要动作提示契约。
