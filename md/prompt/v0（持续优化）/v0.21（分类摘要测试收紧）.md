# v0.21（分类摘要测试收紧）

## 版本号

v0.21

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.20 已补齐 iOS 计时页分类筛选摘要，并通过最新 `origin/main` 云端 artifact 验收。本轮聚焦 CI 测试收紧，递增为 v0.21。

## 背景

分类摘要已经覆盖 iOS 日程页、iOS 计时页和 macOS 日程详情，但过去 `scripts/verify_project.sh` 主要依赖字符串 marker。v0.20 子 agent 审阅发现：仅检查类型名和文案无法证明摘要插入点没有落入不可达分支，也无法证明清除动作接到正确状态。因此本轮把分类摘要的 UI contract 纳入项目专属验证。

## 目标

- `scripts/verify_project.sh` 增加分类摘要源码 contract 检查。
- 检查 iOS `ScheduleView` 的 `SelectedCategorySummaryView` 出现在 `visibleTasks.isEmpty` 空态分支之前。
- 在 iOS `ScheduleView` 摘要调用点到空态分支之前的源码片段内，检查 `onAddTask` 设置 `showingEditor = true`，`onClear` 设置 `selectedCategory = nil`。
- 检查 iOS `TimerView` 的 `TimerSelectedTaskCategorySummaryView` 出现在 `upcomingTasks.isEmpty` 空态分支之前。
- 在 iOS `TimerView` 摘要调用点到空态分支之前的源码片段内，检查 `onClear` 使用 `clearTaskCategoryFilter`。
- 在 macOS `MacScheduleDetailView` 摘要调用点到空态分支之前的源码片段内，检查 `MacSelectedCategorySummaryView` clear closure 设置 `selectedCategory = nil`。
- README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md` 同步当前测试覆盖。

## 非目标

- 不修改 iOS 或 macOS 业务 UI 行为。
- 不修改 `FocusStore`、`TimerEngine`、计划生成、统计或持久化逻辑。
- 不引入第三方依赖。
- 不新增大型 UI 自动化框架。
- 不改变 GitHub Actions artifact 格式。

## 当前架构依据

- `scripts/verify_project.sh` 是项目本地和云端的专属验证入口，已负责检查分类 UI marker、Mac core tests 和快照。
- `ScheduleView.taskList`、`TimerView.taskPicker` 和 `MacTaskListPanelView` 都在同一文件内组合分类摘要与空态分支，适合用小型 Ruby 源码顺序检查锁定 contract。
- 云端 `.github/workflows/ci-results.yml` 会运行 `bash scripts/verify_project.sh`，因此本轮测试收紧会自动进入 `origin/main` 结果包。

## 验收标准

- `scripts/verify_project.sh` 在三个分类摘要插入点或动作接线被破坏时会失败。
- 本地四项轻量检查通过。
- push 到 `origin/main` 后，Agent C 下载最新 v0.21 run artifact，用 `scripts/validate_ci_artifact.rb` 和手工核对共同确认结果包通过。

## 风险和禁止项

- 禁止把测试收紧写成只匹配无关字符串的空检查。
- 禁止扩大为大规模 UI 自动化框架或引入依赖。
- 禁止改动核心业务逻辑。
- 禁止读取、提交或删除无关未跟踪文件。

## GitHub Actions 和 Agent C 复判要求

- 触发 workflow：`.github/workflows/ci-results.yml`
- Agent C 下载 artifact 前必须先完成 `gh auth login`，并确认 GitHub 账号为项目授权账号。
- Agent C 下载最新 `origin/main` commit 对应 run 的 artifact 到 `/private/tmp/chronofocus-c-review-<run_id>/`。
- Agent C 运行 `ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> --commit <sha> --run-id <run_id> --attempt <attempt>`。
- 同时继续人工核对 manifest、artifact index、run context、JUnit、failure summary、主日志、Mac/iOS `.xcresult` 和 Mac 快照 manifest。
