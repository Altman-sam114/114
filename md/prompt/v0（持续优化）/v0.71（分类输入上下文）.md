# v0.71（分类输入上下文）

## 版本号

v0.71

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.70，本轮继续围绕“优化 UI，更加方便使用，优化分类”做小步增强，因此自动递增为 v0.71。

## 背景

iOS 新增/编辑待办表单和 macOS 快速新增表单都支持手输分类和常用分类预设，但分类输入框本身只是普通 `TextField("分类")`。用户手输或点预设后，需要靠输入框文字自行判断当前分类；辅助技术用户也缺少“当前分类 + 可输入自定义分类或选择常用分类”的输入语义。

## 目标

- iOS 新增/编辑待办表单在分类输入区域显示当前分类上下文。
- macOS 快速新增表单在分类输入区域显示当前分类上下文；选中列表分类时仍保留“已预填”语义。
- iOS/macOS 分类输入框补齐可访问标签、提示和 Voice Control input labels。
- `scripts/verify_project.sh` 增加源码契约检查和缺失 marker 负向 fixture。
- `scripts/validate_ci_artifact.rb` 增加云端 artifact 中 verify marker 的复判。
- 同步 README、测试规范、核心流程文档、流程图和更新日志。

## 非目标

- 不修改 `FocusStore`、`TimerEngine`、任务模型、持久化格式或计划生成逻辑。
- 不修改分类预设列表、分类排序算法或筛选逻辑。
- 不重做任务编辑表单布局，不新增第三方依赖。
- 不调整 CI artifact 版本号、上传清单或 workflow 阶段。

## 当前架构依据

- iOS 新增/编辑待办表单位于 `ChronoFocus/Views/ScheduleView.swift` 的 `TaskEditorView`。
- macOS 快速新增表单位于 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的 `MacScheduleDetailView`。
- 常用分类元数据来自共享 `TaskCategoryPreset`；本轮只在展示层读 preset 图标和颜色。
- `scripts/verify_project.sh` 负责源码契约和云端项目专属验证 marker。
- `scripts/validate_ci_artifact.rb` 负责 Agent C 下载 artifact 后复判 `verify_project.log` marker。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `ChronoFocus/Views/ScheduleView.swift`：
   - 为 `TaskEditorView` 增加 `categoryDisplayName`、`categoryPreset`、`categoryTint`、`categorySymbolName` 等展示 helper。
   - `TextField("分类", text: $category)` 增加 `.accessibilityLabel`、`.accessibilityHint` 和 `.accessibilityInputLabels`。
   - 在分类输入附近新增紧凑 `TaskEditorCategoryContextView`，显示当前分类、颜色/图标上下文。
   - Context view 可访问标签包含当前分类，并为 Voice Control 暴露分类名。

2. 更新 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - 为快速新增分类输入补齐 `.accessibilityLabel`、`.accessibilityHint` 和 `.accessibilityInputLabels`。
   - 扩展或新增分类上下文视图，显示当前分类；当来自选中筛选分类时保留“已预填”语义。
   - 保持现有 Mac 快速新增布局宽度和视觉风格。

3. 更新 `scripts/verify_project.sh`：
   - 增加源码契约检查，确认 iOS/macOS 分类输入框包含当前分类可访问标签、提示、Voice Control input labels，以及分类上下文视图。
   - 输出 marker：`Category input context contracts verified.`。
   - 小型成功 fixture 的 `verify_project.log` 增加该 marker。
   - 增加负向 fixture，移除该 marker 后要求 validator 输出 `FAIL verify_project category input context contracts`。

4. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project category input context contracts` 复判，要求 `verify_project.log` 包含本轮 marker。

5. 同步文档：
   - README 描述分类输入上下文与可访问语义。
   - `md/test/test.md` 更新验证范围、Agent C PASS 清单和 artifact marker 说明。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新当前真实核心验证流。
   - `update_log.md` 追加 v0.71 记录。
