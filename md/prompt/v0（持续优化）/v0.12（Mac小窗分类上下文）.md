# v0.12（Mac 小窗分类上下文）

## 版本号

v0.12

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI。v0.11 已补齐 iOS 铃声选择并通过最新 `origin/main` 云端 artifact 验收；`update_log.md` 仍保留“Mac 小窗待办行可继续补强分类 badge 或更多上下文”的遗留问题。本轮在 `v0（持续优化）` 阶段递增为 v0.12，聚焦 Mac 小窗待办行的信息可扫读性。

## 背景

`MacMiniTimerView` 的“当前待办”区域目前每行显示任务标题，右侧显示 `dueDate?.shortTimeText ?? task.category`。当任务有时间时，分类被隐藏；用户在状态栏小窗里快速选任务时，无法同时看到任务所属分类和时间上下文。iOS 计时页已经有任务行分类 badge，本轮让 Mac 小窗补齐同类上下文。

## 目标

- Mac 小窗“当前待办”每个任务行同时显示任务标题、分类 badge 和时间上下文。
- 分类 badge 使用任务 `category`、`accentHex` 和 `TaskCategoryPreset.matching` 的图标信息。
- 如果任务有 `dueDate`，显示时间；如果没有时间，仍显示预计轮次或简短上下文，避免右侧空白。
- 保持小窗宽高和三条任务列表结构，不新增详细窗口或第二套任务选择逻辑。
- 保持快照安全：只使用 `Text`、`Image`、`HStack/VStack` 等 SwiftUI 基础绘制，不引入原生 `Picker/Toggle/Slider/Button` 新占位风险。

## 非目标

- 不改 `FocusTask` 模型字段、分类持久化或排序逻辑。
- 不改 `FocusStore.upcomingTasks()` 的数据规则。
- 不改 `TimerEngine.selectTask(_:)` 或计时状态机。
- 不改 iOS 计时页分类筛选。
- 不改 CI workflow 结构或 artifact 格式。

## 当前架构依据

- Mac 小窗 UI 在 `ChronoFocusMac/Views/MacMiniTimerView.swift`。
- 待办数据来自 `FocusStore.upcomingTasks()`，选择任务仍调用 `TimerEngine.selectTask(_:)`。
- 分类预设和图标信息在共享模型 `TaskCategoryPreset.matching(_:)`。
- Mac 快照由 `scripts/render_mac_snapshots.swift` 渲染，`scripts/verify_project.sh` 会检查实现标记并生成快照 manifest。
- 核心流程文档中，Mac 左键 popover 是用户快速选任务和进入详情的入口。

## 实现步骤

1. 在 `MacMiniTaskPickerView` 的任务行中，把单行 `HStack` 改为更清晰的结构：
   - 左侧保留任务颜色点。
   - 中间用 `VStack` 显示任务标题和分类 badge。
   - 右侧显示时间或轮次上下文。
2. 新增小型私有视图，例如 `MacMiniTaskCategoryBadgeView`：
   - 输入 `FocusTask`。
   - 用 `TaskCategoryPreset.matching(task.category)?.symbolName ?? "tag.fill"` 显示图标。
   - 显示 `task.category`，颜色取 `task.accentHex`。
   - 文字限一行、压缩时不撑破小窗。
3. 新增小型 helper，例如 `taskContextText(for:)`：
   - 有 `dueDate` 时返回 `shortTimeText`。
   - 无 `dueDate` 时返回 `\(task.remainingRounds) 轮` 或 `只设开始` 的简短文案。
4. 更新 `scripts/verify_project.sh`，增加 Mac 小窗分类 badge 和上下文 helper 的实现标记。
5. 更新 README、`md/flow/flow.md`、`md/test/test.md`、`update_log.md`。
6. 不需要改 `md/flow/flowchart.md`，除非实现改变核心数据流；本轮只是 Mac 小窗展示层优化。

## 验收标准

- Mac 小窗任务行同时显示任务标题、分类 badge 和时间/轮次上下文。
- 任务选择仍通过 `TimerEngine.selectTask(_:)`，运行中仍禁用选择。
- 不新增平台边界污染，不复制 `FocusStore` 或 `TimerEngine` 逻辑。
- 本地轻量检查通过。
- push 到 `origin/main` 后，Agent C 下载最新 run artifact，核对 manifest、artifact index、JUnit、failure summary、日志、Mac/iOS `.xcresult`、Mac 快照和快照 manifest。

## 风险和禁止项

- 禁止改动共享模型持久化字段。
- 禁止绕过 `TimerEngine` 新建任务选择状态。
- 禁止为了样式重构整个 Mac 小窗或详细窗口。
- 禁止引入会在 Mac 快照里出现黄色缺失控件占位的原生控件。
- 禁止跳过云端 artifact 验收。
- 禁止读取、提交或删除无关未跟踪文件。
