# v0.72（CI运行上下文精确复判）

## 版本号

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.71，本轮继续围绕“优化 CI 测试”和云端 artifact 复判做小步增强，因此自动递增为 v0.72。

## 背景

GitHub Actions 的 `ci-run-context.txt` 目前由 workflow 固定写入 `artifactName`、`branch`、`commitSha`、`runId`、`runAttempt` 五个字段。`scripts/validate_ci_artifact.rb` 已检查这些字段非空、身份匹配和 artifact 名称匹配，但读取时使用 Hash，重复 key 会被后写值覆盖，也没有拒绝额外字段。Agent C 复判需要更严格地确认 run context 是精确契约，而不是带有重复或额外身份字段的自洽错包。

## 目标

- `scripts/validate_ci_artifact.rb` 增加 `ci-run-context.txt` 精确键集复判。
- 拒绝重复 key、额外 key 或缺失 key 的 run context。
- `scripts/verify_project.sh` 增加对应负向 fixture，确认 validator 会输出 `FAIL run context exact keys`。
- README、测试规范、核心流程文档、流程图和更新日志同步 run context 精确键集复判范围。

## 非目标

- 不修改 `.github/workflows/ci-results.yml` 的输出字段和 artifact 命名规则。
- 不修改 SwiftUI、`FocusStore`、`TimerEngine`、模型或持久化。
- 不新增第三方依赖。
- 不运行本地测试、构建、lint 或本地 fixture validator。

## 当前架构依据

- `.github/workflows/ci-results.yml` 的 `Prepare result metadata` 阶段写入固定五个 run context 字段。
- `scripts/validate_ci_artifact.rb` 是 Agent C 下载 artifact 后的结构化复判脚本。
- `scripts/verify_project.sh` 维护 validator 的小型成功 fixture 和负向 fixture，自身由云端 `Project verification` 阶段执行。
- 用户明确要求“不得在本地测试，都去云端”；本轮本地只允许文件读取、diff/status、commit/push、云端查询、下载云端 artifact 和对下载 artifact 运行 validator。

## 实现步骤

1. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `EXPECTED_RUN_CONTEXT_KEYS = %w[artifactName branch commitSha runId runAttempt].freeze`。
   - 保留 `read_key_values` 对现有调用的 Hash 行为，或拆出 entries 读取函数。
   - 对 `ci-run-context.txt` 保留原始 key entries。
   - 新增 check：`run context exact keys`。
   - 该 check 要求 key 列表排序后等于 `EXPECTED_RUN_CONTEXT_KEYS`，并且 entries 数量等于 expected 数量，从而拒绝重复 key 和额外 key。
   - 保留现有 `run context fields`、`run context identity`、`run context artifact name`。

2. 更新 `scripts/verify_project.sh`：
   - 自检 grep 加入 `EXPECTED_RUN_CONTEXT_KEYS` 和 `FAIL run context exact keys`。
   - 在小型成功 fixture 保持固定五个 run context 字段。
   - 增加负向 fixture，例如向 `ci-run-context.txt` 追加 `source=stale`，运行 validator 必须失败并输出 `FAIL run context exact keys`。
   - 不使用 heredoc 内 `ruby -e` 这类易错写法，延续现有 Python/Ruby fixture 风格。

3. 同步文档：
   - README 描述 run context 字段集合精确、无重复、无额外字段复判。
   - `md/test/test.md` 更新 validator 范围和负向 fixture 描述。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 更新云端 artifact 复判链路。
   - `update_log.md` 追加 v0.72 记录。

## 测试要求

本轮不得运行本地测试、构建、lint、`git diff --check`、`bash scripts/verify_project.sh` 或本地 fixture validator。

```bash
ruby scripts/validate_ci_artifact.rb /private/tmp/chronofocus-c-review-<run_id> \
  --commit <origin-main-sha> \
  --run-id <run_id> \
  --attempt <run_attempt>
```

## 文档更新要求

- 文档必须说明 `ci-run-context.txt` 不只核对身份字段和值，还核对字段集合精确、无重复、无额外字段。

## 验收标准

- `scripts/validate_ci_artifact.rb` 输出包含 `PASS run context exact keys`。
- `scripts/verify_project.sh` 包含负向 fixture，并要求错包输出 `FAIL run context exact keys`。
- GitHub Actions 最新 `origin/main` run 成功。
- 最新云端 artifact 的 validator 输出包含 `PASS run context exact keys`，并保留既有 `PASS run context identity`、`PASS run context artifact name` 和关键 verify_project marker PASS。
- manifest、run context、artifact 名称、commit SHA、run id、attempt 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 若未来 workflow 合法新增 run context 字段，validator 会失败；这符合契约变更必须同步 validator 和文档的目标。
- 不要修改 workflow 输出格式或 artifact 名称。
- 不要运行本地测试或本地构建。
- 不要触碰用户未跟踪文件。
