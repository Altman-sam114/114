# v0.25（iOS 日程筛选计数）

## 版本号

v0.25

## 版本分配依据

人工总目标要求持续优化 UI、分类和 CI 测试，并由 Agent X 自行寻找下一轮优化目标。v0.24 已补齐 CI artifact index 身份错包负向 fixture，并通过最新 `origin/main` 云端 artifact 验收。本轮转回 UI/分类体验，递增为 v0.25。

## 背景

iOS 计时页和 macOS 日程详情已经在选中分类时显示筛选数量与总数，例如 `筛选数/总数`。iOS 日程页当前只在标题右侧显示 `visibleTasks.count 项`，选中分类后用户无法从列表标题直接判断这是筛选后的数量还是当前时间范围总数。本轮补齐 iOS 日程页待办列表标题的筛选计数反馈，让分类筛选状态更清晰。

## 目标

- 在 `ChronoFocus/Views/ScheduleView.swift` 中新增 `taskListCountText`。
- 未选中分类时继续显示当前日/周/月范围内总待办数，例如 `3 项`。
- 选中分类且当前时间范围有待办时显示 `筛选数/总数 项`。
- 当前时间范围总数为 0 时显示 `0 项`，避免旧筛选状态出现 `0/0`。
- 标题右侧计数文本增加单行和缩放约束，降低较窄屏幕挤压风险。
- `scripts/verify_project.sh` 增加 iOS 日程筛选计数源码 marker 和属性片段检查。
- README、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md` 同步当前行为。

## 非目标

- 不修改任务筛选规则、排序规则、分类预设或持久化模型。
- 不修改 macOS 日程详情已有计数行为。
- 不修改计时页分类筛选摘要。
- 不修改 GitHub Actions workflow 或 CI artifact 格式。
- 不引入新依赖，不拆分 SwiftUI 大文件。

## 当前架构依据

- iOS 日程页的可见任务由 `visibleTasks` 统一计算，筛选条件来自 `selectedCategory` 和 `isTaskInSelectedRange(_:)`。
- `taskCount(in:)` 已能返回当前时间范围内全部任务数量或指定分类数量。
- macOS `MacTaskListPanelView.taskListCountText` 已提供同类实现，可作为 UI 行为参考，但不要复制 macOS 平台 API。
- `scripts/verify_project.sh` 已有 Mac 待办筛选计数 marker，本轮增加 iOS 对应 marker。

## 实现步骤

1. 更新 `ChronoFocus/Views/ScheduleView.swift`：
   - 在 `visibleTasks` 附近新增 `private var taskListCountText: String`。
   - 使用 `taskCount(in: nil)` 得到当前时间范围总数。
   - `totalCount == 0` 时返回 `0 项`。
   - 未选分类时返回 `\(totalCount) 项`。
   - 已选分类时返回 `\(visibleTasks.count)/\(totalCount) 项`。
   - 将待办列表标题右侧 `Text("\(visibleTasks.count) 项")` 改为 `Text(taskListCountText)`。
   - 给计数文本加 `.lineLimit(1)` 和 `.minimumScaleFactor(0.78)`。
2. 更新 `scripts/verify_project.sh`：
   - 增加 `grep -q "taskListCountText" ChronoFocus/Views/ScheduleView.swift`。
   - 增加 `grep -q "Text(taskListCountText)" ChronoFocus/Views/ScheduleView.swift`。
   - 增加 Ruby 片段检查 `taskListCountText` 包含 `totalCount > 0`、`0 项`、`visibleTasks.count`、`totalCount` 和 `项`。
3. 更新文档：
   - README 功能和验证说明补充 iOS 日程筛选计数。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 同步 iOS 日程筛选计数反馈。
   - `md/test/test.md` smoke 基线补充 iOS 日程筛选计数 marker。
   - `update_log.md` 追加 v0.25 记录。
