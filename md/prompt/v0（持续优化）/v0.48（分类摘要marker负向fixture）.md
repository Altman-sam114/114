# v0.48（分类摘要 marker 负向 fixture）

## 版本号

v0.48

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.47，因此本轮自动递增为 v0.48。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.47 已让 `verify_project.log` 输出 `Category summary action contracts verified.`，并让 `scripts/validate_ci_artifact.rb` 独立复判该 marker。当前正向 fixture 已覆盖有 marker 时可以通过，但缺少一个负向 fixture 来证明下载 artifact 如果遗漏该 marker 会被 validator 拒绝。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. 在 `scripts/verify_project.sh` 中新增 `negative_summary_marker_fixture`。
2. 该 fixture 复制小型成功 artifact 后移除 `verify_project.log` 中的 `Category summary action contracts verified.`。
3. 运行 validator 时必须失败，并检查输出包含 `FAIL verify_project category summary action contracts`。
4. `scripts/verify_project.sh` 自身增加 grep marker，锁定该负向 fixture 和失败文案存在。
5. README、测试规范、核心流程文档和更新日志同步本轮 CI 负向覆盖范围。

## 非目标

- 不改变 SwiftUI UI、分类筛选状态、按钮动作、任务模型、持久化、计划生成或计时状态机。
- 不修改 GitHub Actions workflow、artifact 文件列表、JUnit 结构或 Xcode 构建命令。
- 不新增大体积 artifact、历史 artifact 下载或真实云端 artifact 下载逻辑。
- 不运行任何本地测试、构建、格式检查或验证脚本。

## 当前架构依据

- `scripts/verify_project.sh` 已在小型 artifact fixture 后维护多个负向 fixture，例如 JUnit outcome 错包、artifactName 错包、index 错包、index totals 篡改、本地缺失产物和快照 manifest 篡改。
- `scripts/validate_ci_artifact.rb` 已有 `verify_project category summary action contracts` 检查项；缺失 marker 应输出 `FAIL verify_project category summary action contracts`。
- 新负向 fixture 只增强 CI 覆盖，不影响产品运行时代码。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认当前 `HEAD == origin/main` 且只有用户未跟踪文件未触碰。
2. 修改 `scripts/verify_project.sh`：
   - 在正向 fixture 通过后、现有负向 fixture 附近新增 `negative_summary_marker_fixture` 和输出文件变量。
   - 复制 `artifact_fixture` 到负向目录。
   - 用 Python 或 Ruby 删除 `verify_project.log` 中的 `Category summary action contracts verified.\n`。
   - 运行 `ruby scripts/validate_ci_artifact.rb ...`，要求失败；若通过则输出错误并退出。
   - `grep -q "FAIL verify_project category summary action contracts"` 检查失败输出。
   - 清理该 fixture 目录和输出文件。
   - 在脚本自检 grep 区增加 `negative_summary_marker_fixture` 与失败文案 marker。
3. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
4. 只检查 diff 和 status，不运行本地测试命令。
5. 提交 `v0.48: 增强分类摘要 marker 负向 fixture` 并 push 到 `origin/main`。
6. Agent C 下载最新 run artifact 并按下方标准验收。
