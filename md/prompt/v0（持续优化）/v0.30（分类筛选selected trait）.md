# v0.30（分类筛选 selected trait）

## 版本号

v0.30

## 版本分配依据

人工未指定小版本号；当前 `update_log.md` 和 `md/prompt/v0（持续优化）/` 最新版本为 v0.29，因此本轮自动递增为 v0.30。

## 背景

总目标要求持续优化 UI、分类体验和 CI 测试。v0.28 已为三端分类筛选 chip 增加可访问 label/hint，v0.29 已让云端 artifact validator 复判分类可访问 contract marker。当前 chip 虽然会读出“已选中”，但还没有向辅助技术暴露 `.isSelected` trait；补齐 trait 能让选中状态不只依赖朗读文本，也能被系统可访问语义识别。

人工明确要求：“不得在本地测试，都去云端”。本轮不得运行本地测试、构建、格式检查或验证脚本，最终验收只以 `origin/main` 最新 GitHub Actions artifact 为准。

## 目标

1. iOS 日程页 `TaskCategoryFilterChip` 在选中时暴露 `.isSelected` 可访问 trait。
2. iOS 计时页 `TimerTaskCategoryFilterChip` 在选中时暴露 `.isSelected` 可访问 trait。
3. macOS 日程详情 `MacCategoryFilterChip` 在选中时暴露 `.isSelected` 可访问 trait。
4. `scripts/verify_project.sh` 的三端分类 chip 可访问 contract 增加 selected trait 源码检查。
5. README、测试规范、核心流程文档和更新日志同步 selected trait 行为。

## 非目标

- 不改变分类筛选视觉样式。
- 不改变 v0.26 的重复点击清除筛选逻辑。
- 不改变 v0.28 的可访问 label/hint 文案。
- 不改 `FocusStore`、`TimerEngine`、计划、统计、提醒或持久化逻辑。
- 不运行任何本地测试、构建或验证命令。

## 当前架构依据

- 三端分类筛选 chip 已是 SwiftUI `Button`，并已有 `isSelected` 状态、可访问 label 和 hint。
- 选中状态只来自现有 `selectedCategory` 绑定，不需要新增状态。
- `scripts/verify_project.sh` 的 `assert_chip_accessibility` 已集中检查三端 chip 的可访问源码 contract，适合继续补充 selected trait 检查。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`。
2. 在三个分类 chip struct 中增加 `accessibilityTraits: AccessibilityTraits` 计算属性，选中时返回 `.isSelected`，未选中时返回空 trait。
3. 在三个 chip 的 modifier 链中增加 `.accessibilityAddTraits(accessibilityTraits)`。
4. 在 `scripts/verify_project.sh` 的 `assert_chip_accessibility` 中检查 `accessibilityTraits: AccessibilityTraits`、`.isSelected` 和 `.accessibilityAddTraits(accessibilityTraits)`。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 只检查 diff 和 status，不运行本地测试命令。
7. 提交 `v0.30: 增加分类筛选选中语义` 并 push 到 `origin/main`。
8. Agent C 下载最新 run artifact 并按下方标准验收。
