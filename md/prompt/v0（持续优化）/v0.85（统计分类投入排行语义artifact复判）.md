# v0.85（统计分类投入排行语义 artifact 复判）

## 版本号

v0.85

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；远端最新 v0.84 已完成统计分类投入专注次数上下文，并通过最新 `origin/main` 云端 artifact 验收。v0.85 继续在同一 v0 持续优化阶段递增。

## 背景

统计页“分类投入”已经显示分类、时长、占比和专注次数。`FocusStore.categoryBreakdown()` 当前按分类投入秒数降序返回结果，但 iOS/macOS UI 和辅助技术语义没有说明每个分类在投入排行中的位置。用户复盘多分类投入时，需要知道当前分类是第几位，尤其 VoiceOver 或 Voice Control 场景下无法直接从视觉顺序判断排序。

## 目标

- iOS `AnalyticsView` 分类投入行显示或读出每个分类的排行位置，例如“第 1 位”。
- macOS `MacAnalyticsDetailView` 分类投入行同步显示或读出排行位置。
- 排行位置必须来自 `ForEach(Array(store.categoryBreakdown().enumerated()), id: \.element.id)` 或等价方式，不改 `FocusStore.categoryBreakdown()` 的排序口径。
- 可访问标签和 Voice Control input labels 包含分类名和排行位置。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Analytics category share ranking contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project analytics category share ranking contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- README、测试规范、核心流程文档和更新日志同步 v0.85 的云端 artifact 复判范围。

## 非目标

- 不新增统计筛选交互、排序开关、图表类型或报表。
- 不修改 `FocusSession`、`FocusTask`、`CategoryFocus` 的持久化或聚合口径。
- 不改 `TimerEngine`、通知、Live Activity、StoreKit、EventKit 或计划生成规则。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；本轮按人工要求只去云端验证。

## 当前架构依据

- `FocusStore.categoryBreakdown()` 已返回分类投入统计，并按投入秒数降序排序。
- iOS `AnalyticsView` 和 macOS `MacAnalyticsDetailView` 都在分类投入面板里 `ForEach(store.categoryBreakdown())` 渲染分类行。
- v0.84 已新增 `sessionCount`、可见次数、可访问标签和 Voice Control input labels，本轮只补排行上下文。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 修改 `ChronoFocus/Views/AnalyticsView.swift`：
   - 分类投入 `ForEach` 改为枚举 `store.categoryBreakdown()`，得到 `rank = index + 1`。
   - 分类行可见显示“第 N 位”或等价短标签。
   - `categoryShareAccessibilityLabel` 增加 `rank` 参数并读出排行位置。
   - Voice Control input labels 增加 `第 N 位分类`、`分类名第 N 位` 或等价自然口令。
2. 修改 `ChronoFocusMac/Views/MacAnalyticsDetailView.swift`，保持与 iOS 一致。
3. 修改 `scripts/verify_project.sh`：
   - 在分类投入检查区新增排行源码契约，检查枚举、rank、可见标签、可访问标签和 Voice Control input labels。
   - 输出 `Analytics category share ranking contracts verified.`。
   - 更新 validator 自检 grep、小型成功 fixture 和 marker 缺失负向 fixture。
4. 修改 `scripts/validate_ci_artifact.rb`：新增 `verify_project analytics category share ranking contracts` 检查。
5. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
