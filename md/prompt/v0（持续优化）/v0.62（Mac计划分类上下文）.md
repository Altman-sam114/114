# v0.62（Mac计划分类上下文）

## 版本号

v0.62

## 版本分配依据

人工总目标要求持续优化 UI 易用性、分类体验和 CI 测试。v0.61 已完成计划项开始按钮语义并通过云端 artifact 验收；本轮继续按小版本递增到 v0.62，聚焦 macOS 自动计划列表的分类上下文。

## 背景

iOS 计划项副标题已显示 `时间段 · 第 N 轮 · 分类`，但 macOS 详细日程页的自动计划列表只显示 `时间段 · 第 N 轮`。在多分类任务同时生成计划时，Mac 用户无法直接从计划列表判断计划项所属分类；开始按钮语义也缺少分类。

## 目标

- macOS 自动计划项副标题补齐分类文本。
- macOS 计划开始按钮可访问标签和 Voice Control input labels 补齐分类上下文。
- 快照静态开始按钮同步保留分类语义。
- 在 `scripts/verify_project.sh` 增加源码契约 marker，并让 `scripts/validate_ci_artifact.rb` 复判该 marker。
- 同步 README、测试规范、核心流程和更新记录。

## 非目标

- 不改 iOS 行为；iOS 已有分类文本和分类语义。
- 不改计划生成、排序、完成状态、`TimerEngine.startPlanItem(_:)` 或 `FocusStore`。
- 不改 Mac 计划行结构为复杂 badge，不引入新组件或大重构。
- 不运行本地测试、build、lint、`git diff --check` 或脚本验证；人工已要求“不得在本地测试，都去云端”。

## 当前架构依据

- macOS 计划行由 `ChronoFocusMac/Views/MacScheduleDetailView.swift` 的 `MacPlanPanelView` 渲染。
- `PomodoroPlanItem` 已含 `category`，无需新增模型字段。
- `scripts/verify_project.sh` 是项目专属验证入口；云端 workflow 会运行它并把日志纳入 artifact。
- Agent C 必须下载最新 `origin/main` artifact，核对 manifest、run context、artifact 名称、JUnit、日志和项目产物。

## 实现步骤

1. 同步最新 `main`，确认没有无关 tracked diff，不读取或提交用户未跟踪文件。
2. 修改 `ChronoFocusMac/Views/MacScheduleDetailView.swift`：
   - Mac 计划项副标题改为 `\(item.timeRangeText) · 第 \(item.roundNumber) 轮 · \(item.category)`。
   - 真实开始按钮 accessibility label 增加 `\(item.category)分类`。
   - 真实开始按钮 input labels 增加 `\(item.category)分类开始`。
   - 快照静态 `MacStaticScheduleActionChipView` title 同步增加分类语义。
3. 修改 `scripts/verify_project.sh`：
   - 增加 Mac 计划行分类上下文源码契约。
   - 输出稳定 marker，例如 `Mac plan category context contracts verified.`。
   - 增加缺失 marker 负向 fixture。
4. 修改 `scripts/validate_ci_artifact.rb`：
   - 复判 `verify_project.log` 中的新增 marker。
5. 同步文档和 `update_log.md`。
