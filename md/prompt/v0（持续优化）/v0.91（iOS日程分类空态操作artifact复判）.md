# v0.91（iOS 日程分类空态操作 artifact 复判）

## 版本号

v0.91

## 版本分配依据

人工未指定版本号；当前 `md/prompt/v0（持续优化）/` 和 `update_log.md` 最新正式轮次为 v0.90，因此本轮自动递增为 v0.91。

## 背景

总目标仍是持续优化 ChronoFocus 的 UI 易用性、分类工作流和 CI 云端验收。v0.90 已完成 macOS 日程快速新增“任务名称”输入框分类上下文与 artifact 复判。本轮转向 iOS 日程页：当前选中分类筛选但当前时间范围无待办时，列表区域只显示说明文字 `Text(emptyTaskListText)`，用户需要回到上方摘要或 toolbar 才能新增此分类或清除筛选。空态本身应提供直接操作。

## 目标

- 在 iOS `ScheduleView` 的任务列表空态中，当 `visibleTasks.isEmpty` 且存在 `selectedCategory` 时，显示可操作的分类空态组件。
- 新组件使用 `ContentUnavailableView` 显示“暂无 X 分类待办”，说明可新增此分类或清除筛选。
- 空态提供“新增此分类”和“清除筛选”两个动作。
- “新增此分类”继续触发 `showingEditor = true`，复用现有 `TaskEditorView(initialDueDate: defaultNewTaskDate, initialCategory: selectedCategory)` 预填分类。
- “清除筛选”设置 `selectedCategory = nil`。
- 两个按钮必须有 44pt 点击区、分类相关 `accessibilityLabel` 和 Voice Control input labels。
- `scripts/verify_project.sh` 新增独立源码契约 marker：`Schedule category empty state action contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 Agent C artifact 复判项：`PASS verify_project schedule category empty state action contracts`。
- 增加 marker 缺失负向 fixture，防止旧 artifact 或缺少该契约的结果包被放行。

## 非目标

- 不改 `FocusStore`、`TimerEngine`、共享模型、持久化格式或计划生成逻辑。
- 不改未选分类时普通空态的 Pro 日历同步提示。
- 不重构 `ScheduleView` 大结构，不拆分文件。
- 不同时实现 CI 运行身份增强、macOS 空态或统计页其它改动。
- 不引入第三方依赖。

## 当前架构依据

- `FocusStore` 仍是任务、分类、计划和统计的唯一核心数据仓库。
- `TimerEngine` 仍是唯一计时状态机，本轮不触碰计时状态。
- `ScheduleView` 已有 `selectedCategory`、`visibleTasks`、`showingEditor` 和 `TaskEditorView(initialCategory: selectedCategory)`，适合在 View 层补齐空态操作。
- `SelectedCategorySummaryView` 已提供相同分类的新增/清除语义，本轮空态按钮应保持一致的动作和可访问文案。

## 实现步骤

1. 同步 `main`：执行 `git fetch origin`、`git switch main`、`git pull --ff-only origin main`，确认 `HEAD == origin/main`，且只有人工未跟踪文件可忽略。
2. 在 `ScheduleView.taskList` 的 `visibleTasks.isEmpty` 分支中：
   - 若 `selectedCategory` 有值，渲染 `ScheduleCategoryEmptyStateView(category:onAddTask:onClear:)`。
   - 否则保留现有普通 `Text(emptyTaskListText)`。
3. 新增 `private struct ScheduleCategoryEmptyStateView: View`，使用 `ContentUnavailableView` 呈现分类空态和两个按钮。
4. 为“新增此分类”和“清除筛选”按钮加入 44pt 最小高度、分类 `accessibilityLabel` 和 `accessibilityInputLabels`。
5. 在 `scripts/verify_project.sh` 中新增源码契约，绑定组件名、`ContentUnavailableView`、动作接线、按钮文案、可访问标签和 input labels，并输出新 marker。
6. 在 `scripts/validate_ci_artifact.rb` 中复判 `verify_project.log` 是否包含新 marker。
7. 在 `scripts/verify_project.sh` 的成功 fixture 中加入新 marker，并新增删除该 marker 的负向 fixture，预期 validator 输出 `FAIL verify_project schedule category empty state action contracts`。
8. 同步更新 `README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
