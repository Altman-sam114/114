# v0.82（待办编辑保存分类语义 artifact 复判）

## 版本号

v0.82

## 版本分配依据

人工总目标仍是 Agent X 持续循环优化 UI 分类可用性和 CI 测试；v0.81 已完成 iOS 统计页日程计划回顾分类语义 artifact 复判，并通过最新 `origin/main` 云端 artifact 验收。v0.82 继续在同一 v0 持续优化阶段递增。

## 背景

iOS 日程页新增和编辑待办共用 `TaskEditorView`。表单内分类输入框和当前分类上下文已经具备分类语义，但 toolbar 的“保存”按钮仍只有 `Button("保存")`，没有读出待办标题、分类、预计轮次或“只设开始”模式，也没有 Voice Control input labels。用户在新增或编辑多分类待办时，最后提交动作缺少关键确认语义。

## 目标

- iOS `TaskEditorView` 的保存按钮补齐可访问标签，读出新增/保存动作、待办标题、当前分类和计划方式。
- 保存按钮 Voice Control input labels 支持任务标题、分类名、`分类`、`保存/新增/更新` 等自然口令。
- 对 `plannedRounds` 读出 `预计 N 轮`；对 `openEnded` 读出 `只设开始`，不能误读为预计 1 轮。
- `scripts/verify_project.sh` 新增源码契约和 marker：`Task editor save category accessibility contracts verified.`。
- `scripts/validate_ci_artifact.rb` 新增 `verify_project task editor save category accessibility contracts` 复判。
- `scripts/verify_project.sh` 的小型成功 fixture 补齐新 marker，并新增 marker 缺失负向 fixture。
- README、测试规范、核心流程文档和更新日志同步 v0.82 的云端 artifact 复判范围。

## 非目标

- 不改 `FocusTask`、`FocusStore`、`TimerEngine`、计划生成、提醒同步或保存业务逻辑。
- 不新增 macOS 待办编辑表单；macOS 当前快速新增按钮已具备分类和预计轮次语义，本轮只覆盖 iOS 已存在的保存按钮缺口。
- 不调整分类预设、筛选排序、表单字段布局或 Pro/日历逻辑。
- 不运行本地测试、构建、lint、`git diff --check` 或本地 `verify_project.sh`；人工明确要求本轮只去云端验证。

## 当前架构依据

- `TaskEditorView` 位于 `ChronoFocus/Views/ScheduleView.swift`，新增 sheet 和编辑 sheet 都复用它。
- `categoryDisplayName` 已处理空分类 fallback；`startMode == .openEnded` 保存时会把轮次清为 1，但 UI 语义应读作“只设开始”。
- 现有可复用模式包括：日程 toolbar 新增入口分类 label/input labels、待办行编辑按钮任务名/分类 input labels、计划开始按钮任务/时间/轮次/分类语义。
- 云端 workflow 已上传 `verify_project.log`，Agent C 可用 validator 复判 marker。

## 实现步骤

1. 在 `ChronoFocus/Views/ScheduleView.swift` 的 `TaskEditorView` 中增加保存按钮 helper：
   - `taskTitleDisplayName`：空标题时提供稳定 fallback。
   - `saveActionText` 或等价 helper：新增时读“新增”，编辑时读“保存/更新”。
   - `savePlanDescription`：`plannedRounds` 读 `预计 \(estimatedRounds) 轮`，`openEnded` 读 `只设开始`。
   - `saveButtonAccessibilityLabel`：包含动作、任务、分类和计划方式。
   - `saveButtonInputLabels`：包含保存、新增/更新、任务标题、分类名和分类保存口令。
2. 将 toolbar confirmation `Button("保存")` 加上 `.accessibilityLabel(...)` 和 `.accessibilityInputLabels(...)`，保持现有 disabled 条件不变。
3. 在 `scripts/verify_project.sh` 的 `TaskEditorView` 分类输入上下文检查区加入保存按钮契约，并输出 `Task editor save category accessibility contracts verified.`。
4. 在 `scripts/validate_ci_artifact.rb` 增加 `verify_project task editor save category accessibility contracts` 检查。
5. 更新 `scripts/verify_project.sh` 的 validator 自检 grep、小型成功 fixture、负向 fixture 名称锁定和 marker 缺失负向 fixture。
6. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
