# v0.67（Mac 小窗快捷面板语义）

## 版本号

v0.67

## 版本分配依据

人工未指定版本号。当前最新完成版本为 v0.66，本轮继续围绕“优化 UI，更方便使用，优化分类；优化 CI 测试；自行寻找目标不断优化”的总目标做小步增强，因此自动递增为 v0.67。

## 背景

v0.66 已补齐 macOS 日程详情页快速新增提交按钮的分类和预计轮次语义，并通过最新 `origin/main` 云端 artifact 验收。后续遗留方向包括 Mac 小窗快捷面板按钮语义。

当前 `MacMiniQuickPanelView` 已提供模式切换、专注时长、铃声、试听、日程、统计和设置入口，但部分按钮只暴露短可见标题。对 VoiceOver 和 Voice Control 用户来说，“日程”“统计”“设置”“铃声”“15”这类短标签缺少动作上下文，也不能稳定表达当前选中状态。

## 目标

- 为 Mac 小窗快捷面板的模式按钮补齐“当前模式 / 切换到某模式”的可访问标签、提示、Voice Control input labels 和 selected trait。
- 为 Mac 小窗快捷面板的 15/25/45 分钟按钮补齐“设置专注时长为某分钟 / 当前已选”的可访问标签、提示、Voice Control input labels 和 selected trait。
- 为 Mac 小窗快捷面板的铃声、试听、日程、统计、设置按钮补齐动作语义和 Voice Control input labels。
- 新增 `verify_project.sh` 源码契约 marker，并让 `validate_ci_artifact.rb` 复判云端 artifact 中的该 marker。

## 非目标

- 不修改 `TimerEngine`、`FocusStore`、计时状态机、持久化、计划生成或统计逻辑。
- 不改变小窗快捷面板的可见布局、按钮标题、排序、禁用规则或业务行为。
- 不扩展到状态栏右键菜单、iOS 页面、Mac 日程快速新增提交按钮、计划面板或 CI process version。
- 不引入第三方依赖。

## 当前架构依据

- Mac 小窗 UI 位于 `ChronoFocusMac/Views/MacMiniTimerView.swift`。
- 小窗快捷面板通过 `MacMiniQuickPanelView` 渲染，通用按钮为 `MacMiniQuickButton`。
- 平台边界要求 macOS target 不依赖 UIKit，iOS target 不依赖 AppKit。
- 云端验证入口是 `.github/workflows/ci-results.yml`，会运行 `bash scripts/verify_project.sh` 并上传未加密 artifact。
- Agent C 下载 artifact 后用 `scripts/validate_ci_artifact.rb` 辅助核对 manifest、run context、JUnit、failure summary、日志和项目专属 marker。

## 实现步骤

1. 更新 `ChronoFocusMac/Views/MacMiniTimerView.swift`：
   - 扩展 `MacMiniQuickButton`，支持传入 `accessibilityLabelText`、`accessibilityHintText`、`accessibilityInputLabels` 和 `accessibilityTraits`。
   - 模式按钮使用当前 `engine.mode` 生成“当前模式”或“切换到某模式”的标签，并在当前模式上加入 selected trait。
   - 专注时长按钮使用当前 `store.settings.focusMinutes` 生成“当前已选”或“设置专注时长为某分钟”的标签，并在当前分钟上加入 selected trait。
   - 铃声按钮读出当前到点音色，提示会切换到点铃声。
   - 试听按钮读出当前到点音色，并提示 Pro 音色未解锁时不可试听。
   - 日程、统计、设置入口读出“打开日程/统计/设置详情”，并补齐 Voice Control input labels。

2. 更新 `scripts/verify_project.sh`：
   - 增加源码契约检查，覆盖上述按钮语义。
   - 输出稳定 marker：`Mac mini quick panel accessibility contracts verified.`
   - 在 validator 自检 greps、成功 fixture 和负向 fixture 中加入该 marker。
   - 增加缺失 marker 负向 fixture：`negative_mac_mini_quick_panel_marker_fixture`，预期 validator 输出 `FAIL verify_project mac mini quick panel accessibility contracts`。

3. 更新 `scripts/validate_ci_artifact.rb`：
   - 新增 `verify_project mac mini quick panel accessibility contracts` 检查，要求 `verify_project.log` 包含上述 marker。

4. 同步文档：
   - `README.md` 记录 Mac 小窗快捷面板按钮语义和 validator 范围。
   - `md/flow/flow.md` 与 `md/flow/flowchart.md` 同步当前真实核心流程和测试入口。
   - `md/test/test.md` 同步 Smoke 基线和 Agent C artifact 验收 PASS 清单。
   - `update_log.md` 追加 v0.67 记录。
